## 镜头逻辑（最高优先级）

**三个合法切理由（必须满足一个才能切）**:
```
信息: 有新信息需要不同镜头来展示
反应: 需要展示某人对事件的回应
强调: 需要改变情感距离传达重要性
都不是 → 不要切，延长当前镜头
```

**每个镜头必须标注**:
```
REQUIRES: 观众需要已知什么？
PROVIDES: 给观众什么新信息？ (不能为空!)
RAISES:   让观众产生什么新疑问？
```

**认知规律（神经科学验证）** → 详见 `cognitive-perception.md`:
```
scale-in (宽→窄): 观众几乎不注意 → 用于沉浸
scale-out (窄→宽): 观众明确感知 → 用于冲击/呼吸
角度变化: 不触发事件感知 → 微妙的权力转移
运动→运动: 最流畅的切   静止→静止: 最明显的切
动作中切: 同一事件       动作完成后切: 新事件
认知负荷: ECU<=2, CU<=3, MCU<=4, MS<=5, LS<=8 信息单元
情感启动: 对比切前 >=1.5s启动; 深度颠覆 >=5s
注意力: 面部>运动>对比>中心>惯性 (P1-P5优先级)
```

```
5 = 全剧转折  → ECU/CU, 延长时长
```

## Eisenstein 五级蒙太奇

```
1. 度量 → 镜头绝对时长 → 脉搏节奏
2. 节奏 → 画面内运动 → 动作流畅
3. 调性 → 情绪基调 → 氛围沉浸
4. 泛音 → 1-3综合 → 多层体验
5. 理性 → 符号碰撞 → 思想触发
```

## 连贯性铁律

```
□ 180度轴线: 不越轴（除非故意）
□ 30度规则: 角度变化≥30°
□ 视线匹配: A看右 → B从左看
□ 动作匹配: 切在动作中（非完成后）
□ 方向一致: 角色运动方向跨镜头不变
```

## 镜头序列惯用语（Idiom）— 18种

```
 1 建立:      ELS → LS → MS            (先空间后角色)
 2 正反打:    Master → OTS-A↔OTS-B → CU (升级=关键时刻)
 3 揭示渐进:  LS → MS → MCU → CU → INSERT (scale-in到真相)
   揭示突然:  MS(平静) → 硬切ECU(冲击)
 4 追逐:      A(MS)→B(MS)→环境→循环加速   (景别收窄+时长递减)
 5 窥视:      窥视者POV(有框) → 被窥视者 → 窥视者CU反应
 6 反应链:    事件 → CU-A → CU-B → CU-C   (比较不同反应)
 7 告白:      MS→MCU→CU→ECU(核心)→倾听者ECU(冲击) (景别=秘密深度)
 8 独自调查:  POV扫描 → INSERT线索 → CU反应 → 循环  (INSERT标点法)
 9 内心独白:  CU(失焦) → VO+空镜 → INSERT联想 → CU(结论) (画面承载思考)
10 电话:      INSERT手机 → MCU单侧 → CU反应(推断对方) (单侧=悬念)
11 平行剪辑:  A(MS)↔B(MS) → 加速交替 → 汇合   (切换递减=紧迫)
12 时间蒙太奇: 片段*N(递减时长) → 结果CU         (时间标记+变化积累)
13 群体扫描:  LS全组 → PAN扫描 → CU隔离 → 权力中心 (空间=权力)
14 门槛跨越:  MS门前 → CU犹豫 → 跨越 → LS新空间   (物理=心理边界)
15 异常发现:  MS日常 → 微停 → INSERT异常 → CU困惑 → 合理化/警觉
16 沉默对峙:  MASTER → CU-A↔CU-B(递增时长) → 环境INSERT → 打破
17 镜像:      MS接近镜子 → CU镜中 → 差异 → CU真实面 → 质疑
18 尾随:      MS行走 → SFX信号 → POV回头(模糊) → 加速 → 结果
```

复合记法: A->B 衔接 | A[B] 嵌套 | A||B 并行 | A>>B 中断 | A*N 循环 | !A 反idiom

## 常见逻辑错误

```
无因之切: 没理由就切 → 标注切理由/删除切
信息缺口: 镜头缺少前置信息 → 补建立镜头
空转镜头: 不提供新信息 → 删除或加内容
反应先因: 反应在原因之前 → 调整顺序
景别跳崖: ECU直切ELS → 加过渡或确认意图
全知泄露: 摄影机知道太多 → 检查视角一致性
```

## 质量指标（自审计）

```
信息效率 > 0.8    切理由覆盖 = 1.0
依赖完整 = 1.0    节奏方差σ 1.5-4.0s
Idiom覆盖 0.6-0.9  景别熵 > 2.0 bits
```

## 悬疑专项

**线索镜头**: LS中隐藏，揭示时CU/INSERT
**悬念节奏**: CU-INSERT-CU-INSERT-ECU（递增停留）
**揭示蒙太奇**: ECU快闪(0.5s×N) → 静止CU(长)
**Hitchcock式**: 观众看到角色看不到的 → INSERT威胁源

## LLM 执行优化

**提示词模板** (prompt-templates.md):
```
Template 1: /decompose → 全流程8步分解
Template 2: /shot N    → 单镜头精调+一致性验证
Template 3: /rhythm    → 节奏分析+ASCII可视化+处方
Template 4: /visual-audit → 独立VERIFY(P01-P20谓词)
Template 5: /score     → 六维度加权评分
```

**输出验证** (JSON):
```
Schema: V01-V15 (字段类型+enum)
Logic:  L01-L07 (因果+DAG+方向)
Quality: Q01-Q06 (6项可计算指标)
```

**自修正协议**:
```
VERIFY失败 → 定位失败谓词 → 仅重设计受影响镜头
→ 重验证 → 最多3轮 → 仍失败则标记 [HUMAN_REVIEW]
```

**上下文预算** (每步):
```
PARSE:   500-2000 tok  (原文+角色+空间)
BEAT:    1000-3000     (PARSE输出+原文)
BLOCK:   500-1500      (节拍+空间图)
SHOOT:   2000-4000     (节拍+调度+规则) ←最大
VERIFY:  1500-3000     (序列+谓词)
CONNECT: 1000-2000     (序列+转场规则)
RHYTHM:  500-1500      (序列+模式库)
ANNOTATE:1000-2000     (序列+参考)
全流程:  8000-18000 tok
```

**5大陷阱速查**:
```
表面模仿 → 强制标注切理由+依赖图
均匀化   → 至少1个2x长或0.5x短的镜头
过度切   → 先节拍化再设计镜头
惯性     → 每5镜头问"要打破模式吗?"
忽略负空间 → 每场至少1个SIL
```
