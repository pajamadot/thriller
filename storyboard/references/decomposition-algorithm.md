## 二、Step 1: PARSE（解析）

> **交叉引用**: `shot-grammar.md` §一 定义了完整的景别谱系（EWS→ECU 共12级），
> 其中每个景别的画面内容、情感效果、叙事用途均有详细说明。
> 本步骤的功能标注（DESC/ACT/DIA/THINK/EXPO/TRANS）应与 `shot-grammar.md`
> 中的景别-功能对应表对照使用——描写型文本通常映射为宽景别，
> 内心型文本通常映射为特写景别。

### 目标
将连续文本切分为可处理的「文本单元」，并提取隐含的视觉/听觉信息。

### 输入
原始叙事文本。

### 过程

```
1.1 段落切分
  按空行、场景分隔线、对话切换进行物理分割
  每个段落 = 一个初始文本单元

1.2 功能标注
  为每个文本单元标注叙事功能类型:
    [DESC] 描述：环境、外貌、物品描写
    [ACT]  动作：角色的物理行为
    [DIA]  对话：角色说的话
    [THINK]内心：角色的内在思维
    [EXPO] 说明：叙事者的补充信息
    [TRANS]过渡：时间/空间的跳转标记

1.3 隐含信息提取
  从文本中提取没有明确写出但隐含的视觉/听觉元素:

  示例（《镜中访客》ch1）:

  文本: "我又没有睡好。"
  显性: 角色内心独白
  隐含:
    - 视觉: 角色的状态（疲惫的面容、凌乱的头发）
    - 空间: 角色醒来的环境（卧室、床）
    - 时间: 早晨
    - 声音: 闹钟？安静？

  文本: "林小曼准时推门进来。"
  显性: 角色行动（进入）
  隐含:
    - 视觉: 门的开合动作、林小曼的全貌
    - 空间: 咨询室的布局
    - 声音: 门声
    - 调度: 从门口到沙发的路径
```

### 输出
```
文本单元列表:
  Unit 1: [THINK] "我又没有睡好。" → 隐含: 疲惫面容、卧室环境、早晨
  Unit 2: [DESC] 冲澡、脚底有灰 → 隐含: 浴室空间、水声、细节特写
  Unit 3: [EXPO] 上午取消来访 → 隐含: 时间推移、咨询室空间
  ...
```

### 质量检查
- [ ] 每个文本单元有且只有一个主要功能
- [ ] 隐含信息不与文本矛盾
- [ ] 没有遗漏关键的视觉/听觉信息

## 三、Step 2: BEAT（节拍化）

> **交叉引用**: `evaluation.md` §一 定义了六维度评分体系，其中 NARR（叙事性）维度
> 直接评估节拍覆盖率和弧线对齐。本步骤的叙事重力评分（1-5）应与 `evaluation.md`
> 的 NARR 评分标准保持一致——重力 5 的节拍必须在六维度评估中对应最高叙事冲击，
> 重力 1 的节拍是叙事胶水，不应在 NARR 维度中被要求产生弧线转折。

### 目标
将文本单元合并/拆分为叙事节拍——每个节拍代表一次价值变化，对应一个（或一组）镜头。

### 输入
Step 1 的文本单元列表。

### 过程

```
2.1 价值变化识别
  对每个文本单元问: "这里有没有价值状态的改变？"

  价值维度:
    - 权力关系: 谁占上风？
    - 情感状态: 角色的情绪？
    - 信息量: 角色/观众知道了什么新东西？
    - 威胁等级: 危险增加还是减少？
    - 角色距离: 角色之间更近还是更远？

  如果有变化 → 这是一个节拍边界
  如果没有变化 → 合并到前一个节拍

2.2 节拍分类
  行动节拍 [ACT-B]: 物理动作推进情节
  反应节拍 [REACT-B]: 角色对事件的内在回应
  揭示节拍 [REVEAL-B]: 新信息呈现
  转折节拍 [TURN-B]: 场景方向转变
  张力节拍 [TENSION-B]: 悬念施压但无释放
  释放节拍 [RELEASE-B]: 张力暂时释放
  过渡节拍 [TRANS-B]: 时空桥接

2.3 叙事重力标注
  5 = 故事转折点（全局影响）
  4 = 场景转折点（本场核心）
  3 = 重要推进（实质进展）
  2 = 辅助建设（次要信息）
  1 = 过渡连接（叙事胶水）

2.4 弧线验证
  将节拍按顺序排列，检查:
    - 是否形成有效的戏剧弧线？
    - 最高重力的节拍是否在合理的位置（通常在后半段）？
    - 是否有足够的上升（不能全是平的节拍）？
    - 是否有呼吸点（不能全是高重力节拍）？
```

### 示例应用

```

Beat 1 [TRANS-B, 重力1]:
  叶知秋没有睡好，空白记忆
  价值变化: 无（建立初始状态）
  文本单元: U01-U02

  脚底有灰 → 自我解释
```

## 四、Step 3: BLOCK（场面调度）

> **交叉引用**: `composition-staging.md` §一 定义了 Katz A-I-L 调度模型的三种基本模式:
> - **A型（Apex）**: 三角形顶点——面对面，角色视线交叉，适合对话/对峙
> - **I型（In-line）**: 直线——角色沿同一方向排列，适合行进/追踪/进入
> - **L型（L-shape）**: 直角——角色呈90度关系，适合并肩/观察/回避
>
> 该文件还定义了 Hall 人际距离学的四区间（亲密/个人/社交/公共），
> v2.0 新增的色温函数 T_color()、景深决策规则、Arnheim 视觉重量法则。
> 每个节拍的调度模式应从 A-I-L 中选择，距离应标注所属区间。

### 目标
确定每个节拍中角色在空间中的位置和运动。

### 输入
节拍序列 + 场景空间信息。

### 过程

```
3.1 空间建模
  绘制场景的简易平面图:
    - 门/窗的位置
    - 家具/关键物品的位置
    - 光源位置
    - 摄影机可能的位置

3.2 角色起始位置
  每个角色在场景开始时在哪里？
  这决定了初始的调度模式（A/I/L）

3.3 运动轨迹设计
  为每个有运动的节拍设计角色移动:
    - 从哪到哪？
    - 速度？
    - 动机（为什么移动？）
    - 移动如何改变角色间的距离关系（Proxemics）？

3.4 关键位置标注
  标注以下关键时刻的精确位置:
    - 场景的最高张力点 → 角色在哪里？面向哪里？
    - 揭示时刻 → 被揭示的物品/信息在空间中的位置？
    - 进入/离开 → 角色如何出入？
```

### 示例

```
《镜中访客》ch1 — 咨询室场景

空间:
  ┌─────────────────────────┐
  │          窗户             │
  │                          │
  │  ■ 叶知秋的椅子          │
  │      [桌子]              │
  │                          │
  │      ■ 沙发（林小曼）     │
  │                          │
  │                    门 ←──│
  └─────────────────────────┘

调度模式: A型（面对面，隔桌对坐）
距离: 社交区（~200cm）→ 不变（两人都不移动 = 职业距离维持）

角色运动:
  Beat 4: 林小曼从门口走到沙发 → I型(进入) → A型(落座)
  Beat 5-10: 无大幅移动 → A型维持
  Beat 11: 叶知秋独坐 → 无对象的A型（对面空了）= 孤独
```

### 输出
- 场景平面图
- 每个节拍的角色位置
- 调度模式标注（A/I/L + Proxemics 区间）
- 距离变化轨迹

## 五、Step 4: SHOOT（镜头设计）

> **交叉引用**: `cognitive-perception.md` §零-§一 阐述了认知规则的神经科学基础，
> 包括 Posner (1980) 注意力聚光灯、Mital (2011) 显著性层级、
> Miller (1956) 认知负荷限制（7±2 通道容量）。
> 本步骤的景别/角度/运动选择应遵守以下认知限制:
> - **INFO_DENSITY ≤ 5**（每个非建立镜头的信息通道总量不超过5）
> - **scale-in 切**用于沉浸段（观众几乎不感知切的存在）
> - **scale-out 切**用于冲击/呼吸（观众明确感知转换）
> - **运动→运动**是最流畅的切（视线追踪延续）
> - **静止→静止**是最明显的切（每个切是一次"击打"）

### 目标
为每个节拍选择具体的镜头参数。

### 输入
节拍序列 + 调度设计。

### 决策规则

```
4.1 景别选择
  规则: 情感距离 = 物理距离

  重力 5 (故事转折) → ECU / CU
  重力 4 (场景转折) → CU / MCU
  重力 3 (重要推进) → MCU / MS
  重力 2 (辅助建设) → MS / MLS
  重力 1 (过渡连接) → LS / ELS

  但需要检查:
    - 是否需要看到环境？→ 宽景
    - 是否需要看到动作？→ 全景/中景
    - 是否需要看到情感？→ 特写
    - 是否需要看到线索？→ INSERT

4.2 角度选择
  规则: 权力关系 = 垂直关系

  问: 此刻谁占优势？
    说话者占优 → 仰拍说话者
    听者占优（沉默的力量）→ 俯拍说话者
    平等 → 平视
    世界失衡 → 荷兰角
    代入体验 → POV

4.3 运动选择
  规则: 摄影机的运动 = 叙事的能量流

  问: 这个节拍的能量方向是什么？
    聚焦/发现 → 推近
    释放/揭示全貌 → 拉远
    跟随行动 → 横移/稳定器
    不稳定/恐慌 → 手持
    观察/审视 → 静止
    认知失调 → Dolly Zoom

4.4 声音设计
  规则: 声画关系三选一

  同步: 声音 = 画面（强化）
  对位: 声音 ≠ 画面（反讽/不安）
  先行: 声音 > 画面（预示）
```

### 反直觉规则

```
以下情况应违反"重力=景别"的默认规则:

1. 用远景拍高重力节拍:
   当角色的渺小本身就是故事的重点
   例: 角色在空荡的街道上奔跑 → ELS表达无助

2. 用特写拍低重力节拍:
   当角色的微表情包含关键线索
   例: 路人甲的ECU → 揭示章节后回想才发现是凶手

3. 用静止拍高能量节拍:
   当"不动"比"动"更有力量
   例: 角色被告知亲人死亡 → 15秒的静止CU（比任何运动更震撼）

4. 用运动拍低能量节拍:
   当摄影机运动制造不安
   例: 角色睡觉 → 缓慢环绕拍摄（谁在看他？）
```

### 输出
```
每个节拍的镜头参数:
  Beat 5: CU | HIGH→EYE | STATIC→PUSH |
          声音: DIA+SIL | 时长: 8s
```

## 五-b、Step 4.5: VERIFY（逻辑验证）[关键步骤]

### 目标
在继续蒙太奇设计之前，验证镜头序列的信息逻辑。
**这是防止"看起来专业但逻辑不通"的核心步骤。**

> **交叉引用 [主]**: `shot-logic.md` v2.1 §五 定义了形式化谓词 P01-P20。
> 本步骤应按照该谓词表逐条执行，不得用散文描述替代。
> 每个谓词返回 PASS / WARN / FAIL，结果直接映射到 `evaluation.md` 的 LOGIC 分数。
>
> **交叉引用 [辅]**: `llm-guidance.md` §四 定义了 LLM 执行 VERIFY 时的自审计协议，
> 包括强制思维链、常见退化模式诊断、修正循环模板。
> LLM 在执行本步骤时应遵循 `llm-guidance.md` 的 CORRECTION CYCLE 模板，
> 最多迭代 3 次修复，每次修复后重新验证所有受影响的谓词。

### 输入
Step 4 的镜头参数序列。

### 过程

```
4.5.1 多通道信息依赖图构建 [引用 shot-logic.md §二]
  对每个镜头标注三通道属性:

  V-REQUIRES / V-PROVIDES / V-RAISES (视觉通道)
  A-REQUIRES / A-PROVIDES / A-RAISES (听觉通道)
  N-REQUIRES / N-PROVIDES / N-RAISES (叙事潜文本通道)

  计算 INFO_DENSITY = |V-PROVIDES| + |A-PROVIDES| + |N-PROVIDES|

4.5.2 切理由验证 — 执行谓词 P00-P04 [引用 shot-logic.md §五 Level 1]
  逐个切检查:
    □ P01: INFORMATION — 新信息需要不同取景？
    □ P02: REACTION — 需要展示反应？
    □ P03: EMPHASIS — 需要改变情感距离？
    □ P04: ORIENTATION — 需要重新建立空间认知？ [v2.1 新增]
    □ P00: 至少一个谓词 PASS？（否则 = 无因之切 → FAIL）
  切理由可复合标注: REASON = INFORMATION + EMPHASIS

4.5.3 依赖验证 — 执行谓词 P05-P10 [引用 shot-logic.md §五 Level 2]
    □ P05: 每通道 REQUIRES 被前置镜头满足？(含跨通道满足规则)
    □ P06: 至少一个通道 PROVIDES 非空？
    □ P07: RAISES 链持续活跃？(任意时刻 active_questions ≥ 1)
    □ P08: DAG 无循环？
    □ P09: INFO_DENSITY ≤ 5 (建立镜头 ≤ 7)？
    □ P10: 三通道汇聚点对齐叙事高潮？

4.5.4 位置验证 — 执行谓词 P11-P14 [引用 shot-logic.md §五 Level 3]
    □ P11: 原因在反应之前？(延迟揭示需标注)
    □ P12: 相邻镜头顺序不可互换？
    □ P13: 无冗余镜头？(或冗余有标注理由)
    □ P14: canonical provider 已标注？

4.5.5 参数验证 — 执行谓词 P15-P18 [引用 shot-logic.md §五 Level 4]
    □ P15: 景别选择有叙事理由？
    □ P16: 景别跳跃 ≤ 2 级 (或有理由)？(INSERT 豁免)
    □ P17: 角度反映权力/心理关系？
    □ P18: 时长与信息密度匹配？

4.5.6 序列验证 — 执行谓词 P19-P20 [引用 shot-logic.md §五 Level 5]
    □ P19: 所有 idiom 完整？（建立/正反打/揭示/追逐/反应链/窥视）
    □ P20: 5-镜头窗口内始终有活跃悬念？(场景末3镜头豁免)
```

### 输出
```
验证报告:
  PASS: N 个谓词
  WARN: M 个谓词
  FAIL: K 个谓词

  通过标准:
    FAIL = 0 且 WARN ≤ 3  → 序列通过验证 → 继续 Step 5

    WARN 项的观察记录

  → evaluation.md LOGIC 分数映射:
    LOGIC = 10 × (1 - FAIL/total) × (1 - 0.1×WARN/total)
```

## 六、Step 5: CONNECT（蒙太奇设计）

> **交叉引用**: `montage-theory.md` §一-§三 定义了 Eisenstein 五级蒙太奇体系:
> 1. **度量蒙太奇**: 时长决定节奏（等长切 = 机械节拍）
> 2. **节奏蒙太奇**: 画面内运动决定切（运动高峰 = 切点）
> 3. **调性蒙太奇**: 情绪基调统一（色调/光线/速度一致 = 情感流）
> 4. **泛音蒙太奇**: 多维度同时叠加（视觉+听觉+叙事同时变化）
> 5. **理性蒙太奇**: 概念碰撞产生新意义（A+B = C，C ≠ A 也 ≠ B）
>
> 该文件还包含 Kuleshov 效应的形式化定义、Murch 六层优先级、
> 以及与 `cognitive-perception.md` 中知觉桥接理论的交叉引用。
> 每个转场的蒙太奇类型应标注为上述五级之一（或复合标注如"调性+泛音"）。

### 目标
设计镜头间的转场和关系。

### 输入
经过验证的镜头参数序列。

### 过程

```
5.1 转场选择
  默认: CUT（90%的情况）

  需要特殊转场的信号:
    - 场景变化 → DISSOLVE / FADE / MATCH
    - 时间跳跃 → FADE / DISSOLVE / JUMP CUT
    - 回忆 → DISSOLVE
    - 声音桥接 → L-Cut / J-Cut
    - 视觉关联 → MATCH / GMATCH
    - 连续感 → INVISIBLE CUT

  切的认知效果验证（Liao et al., 2023）:
    scale-in 切: 流畅，观众几乎不注意 → 用于沉浸段
    scale-out 切: 显著，观众明确感知 → 用于冲击/呼吸
    同景别切: 需要≥30°角度变化，否则是跳切
    运动→运动: 最流畅的切（视线追踪延续）
    静止→静止: 最明显的切（每个切是一次"击打"）

5.2 蒙太奇类型标注
  问: 这组镜头为什么以这个顺序排列？
    - 时长决定节奏 → 度量蒙太奇
    - 画面内运动决定 → 节奏蒙太奇
    - 情绪基调统一 → 调性蒙太奇
    - 多维度叠加 → 泛音蒙太奇
    - 概念碰撞 → 理性蒙太奇

5.3 连贯性检查
  逐对检查相邻镜头:
    □ 180度规则: 摄影机在轴线同侧？
    □ 30度规则: 角度变化≥30°？
    □ 视线匹配: 视线方向正确？
    □ 动作匹配: 动作连贯？
    □ 银幕方向: 运动方向一致？
    □ 动作连贯: 前镜有运动→后镜接运动？(Magliano, 2011)
```

## 七、Step 6: RHYTHM（节奏校准）

> **交叉引用**: `visual-rhythm.md` 是本步骤的核心参考文件。
> - §八 定义了 Duration Function D() 的完整公式和参数表
> - §九 定义了时间记号（Time Signature）类型
> - §十 定义了基础节奏模式和 Composition Algebra
> - §十一 定义了短语划分规则（3-5镜头/短语，边界条件）
> - §十二 定义了 Tempo Map 绘制方法
>
> 本步骤的所有数学计算（D_base、G、B、S 系数表）均以 `visual-rhythm.md` 为准。

### 目标
确保镜头序列的时间节奏符合叙事需要。

### 输入
完整的分镜序列（含时长标注）。

### 过程

```
6.1 时长分配 — Duration Function（参见 visual-rhythm.md §八）
  使用数学化的时长函数计算每个镜头的初始时长:

  D(shot) = D_base(size) * G(gravity) * B(beat_type) * S(style)

  参数表（完整值见 visual-rhythm.md）:
    D_base: ELS=5.0, LS=4.0, FS=3.5, MS=3.0, MCU=2.5, CU=3.0, ECU=2.0, INSERT=2.0
    G: g1=0.70, g2=0.90, g3=1.00, g4=1.30, g5=1.80
    B: action=0.80, reaction=1.20, reveal=1.50, tension=1.80, transition=0.60, ...
    S: neutral=1.00, hitchcock=1.20, fincher=0.85, ...

  附加修正:
    台词: D_speech = max(D, T_read + 0.5), T_read = 字数/语速
    开场: 场景首镜 × 2.0-2.5
    沉默: D × 1.3-1.5
    长镜头: 豁免30s上限, D = T_content + T_padding

6.2 时间记号确定（参见 visual-rhythm.md §九）
  为场景选择时间记号:
    审讯 → 4/4 synco
    追逐 → 6/8 accel
    发现 → 3/4 rubato
    对话 → 4/4 rubato
    蒙太奇 → var metric
    冥想 → free

6.3 短语划分（参见 visual-rhythm.md §十一）
  将镜头序列按短语边界分组:
    每个短语 3-5 镜头（最多7）
    边界位于: 话题转换/能量转变/空间切换/焦点转移/时间间断
    短语按段落组织: Building → Climax → Resolution

6.4 节奏模式选择与组合（参见 visual-rhythm.md §十）
  为每个段落/短语选择节奏模式:
    使用 Composition Algebra 组合基础模式:
    CONCAT, NEST, INVERT, SCALE, ACCENT, INTERLEAVE

  示例:
    interrogation = CONCAT(SCALE(Breathe,1.0), NEST(Accelerando,Pulse), Fermata)

6.5 Tempo Map 绘制（参见 visual-rhythm.md §十二）
  绘制场景的BPM曲线:
    BPM(shot_n) = 60 / D(shot_n)
  检查:
    - BPM曲线是否有明确方向？（随机波动=无节奏意图）
    - BPM峰/谷是否与叙事高潮/低点对齐？
    - 是否有呼吸（BPM下降点）？

6.6 数学验证
  计算节奏指标:
    σ (时长标准差): 目标 1.5-4.0s（快闪蒙太奇豁免）
    ASL (平均镜头时长): 对照类型目标（悬疑3.5-6.0s）
    r(1) (lag-1自相关): 目标 0.3-0.7（接近1/f分布）
    1/f指数 α: 目标 0.8-1.2（N>15时可靠）

6.7 总时长估算
  Σ D(shot_i) = 场景总时长
  检查: 是否与场景的叙事权重匹配？
  一般参考: 一页剧本 ≈ 1分钟银幕时间
```

## 九、Step 8: SELF-AUDIT（自审计）[强制步骤]

> **交叉引用 [主]**: `llm-guidance.md` §五 定义了 Q01-Q12 全部 12 项质量指标的
> 精确计算公式、PASS/WARN/FAIL 阈值、以及质量指标修复策略。
>
> **交叉引用 [辅]**: `evaluation.md` §一 定义了 6 维度加权评分体系
> （LOGIC / NARR / VIS / RHYTHM / SOUND / CRAFT），每维度 10 分满分。
>
> 本步骤是算法的**质量闭环**——前 7 步产出分镜序列，本步骤验证产出是否达标。
> 如果任何指标 FAIL，必须回溯到对应步骤修复后重新审计。
> **这不是可选步骤。每次 `/decompose` 执行必须以 SELF-AUDIT 结束。**

### 目标
对完成的分镜序列执行独立质量审计，计算全部 12 项可量化指标，
输出 6 维度评分，识别需要修复的问题并驱动迭代修正。

### 输入
Step 7 完成的带注释分镜序列（含时长、切理由、信息依赖图、Idiom标注、节奏指标）。

### 过程

```
8.1 存在性审计
  □ 每个镜头至少一个通道 PROVIDES 非空？
  □ 每个节拍有至少一个对应镜头？
  □ 无遗漏的文本单元（所有原文信息在分镜中有视觉/听觉表达）？

8.2 依赖审计
  □ 信息依赖图 (DAG) 有效？
    - 无循环
    - 无孤岛（每个镜头至少有一条入边或出边，首镜除外）
    - 所有 REQUIRES 被前置镜头满足
    - RAISES 链持续活跃

8.3 Idiom审计
  □ 所有识别到的 Idiom 是否完整？
    - 建立序列: 至少 ELS/LS → MS → MCU
    - 正反打: 含建立镜头 + 至少一组 A↔B
    - 揭示序列: 铺垫→触发→揭示→反应
    - 追逐序列: 猎→猎物交替 + 环境
    - 反应链: 刺激→反应 1→反应 2...
  □ 不完整的 Idiom 是否有标注理由（有意打断 = 可接受）？

8.4 认知审计
  □ scale-in 切用于沉浸段？
  □ scale-out 切用于冲击/呼吸？
  □ INFO_DENSITY ≤ 5 (建立镜头 ≤ 7)？
  □ 认知负荷在连续高密度段后有恢复点？

8.5 节奏审计
  □ 时长方差 σ 在目标范围？
  □ 最长/最短倍率 ≥ 3？
  □ 高潮有节奏对应（加速或停顿）？
  □ 每 5-8 个短镜头后有呼吸点？
  □ 无连续 > 4 个相同景别？

8.6 叙事审计
  □ 线索（C-type）在画面中有足够停留时间（每线索 ≥ 3s）？
  □ 红鲱鱼（H-type）有视觉强化？
  □ 不可靠叙述信号有足够数量的视觉提示？
  □ 弧线形状与叙事结构匹配？
```

### 12 项质量指标 (Q01-Q12)

全部指标的精确计算公式见 `llm-guidance.md` §五。以下是摘要:

```
Q01 信息效率
  公式: unique_info_units / total_shots
  PASS: > 0.8 | WARN: 0.5-0.8 | FAIL: < 0.5

Q02 切理由覆盖率
  公式: justified_cuts / (total_shots - 1)
  PASS: = 1.0 | WARN: 0.8-0.99 | FAIL: < 0.8

Q03 依赖完整度
  公式: satisfied_requires / total_requires
  PASS: = 1.0 | WARN: N/A | FAIL: < 1.0

Q04 节奏方差 σ
  公式: std_dev(all duration_s)
  PASS: 1.5 ≤ σ ≤ 4.0 | WARN: 1.0-1.5 or 4.0-5.0 | FAIL: < 1.0 or > 5.0

Q05 Idiom 覆盖率
  公式: shots_in_idioms / total_shots
  PASS: 0.6-0.9 | WARN: 0.4-0.6 or > 0.9 | FAIL: < 0.4

Q06 景别分布熵
  公式: -SUM(freq * log2(freq)) for all shot sizes
  PASS: > 2.0 bits | WARN: 1.5-2.0 | FAIL: < 1.5

Q07 ASL (平均镜头时长)
  公式: total_duration / total_shots
  PASS: 在类型目标区间内（悬疑 3.5-6.0s）| WARN: 偏离 ≤ 1.0s | FAIL: 偏离 > 1.0s

Q08 r(1) lag-1 自相关
  公式: autocorrelation(duration_sequence, lag=1)
  PASS: 0.3-0.7 | WARN: 0.2-0.3 or 0.7-0.8 | FAIL: < 0.2 or > 0.8

Q09 1/f 指数 α
  公式: spectral_slope(duration_sequence)  (仅 N>15 时可靠)
  PASS: 0.8-1.2 | WARN: 0.6-0.8 or 1.2-1.5 | FAIL: < 0.6 or > 1.5

Q10 D_MAE (时长函数偏差)
  公式: mean(|D_actual - D_calculated|) across all shots
  PASS: < 1.0s | WARN: 1.0-2.0s | FAIL: > 2.0s

Q11 短语 CV (变异系数)
  公式: std_dev(phrase_lengths) / mean(phrase_lengths)
  PASS: < 0.3 | WARN: 0.3-0.5 | FAIL: > 0.5

Q12 高潮对齐
  公式: max_gravity_beat 是否对应 BPM 极值（最低BPM=停顿 或 最高BPM=加速）
  PASS: true | FAIL: false
```

### 6 维度评分

基于审计结果，为分镜序列打分:

```
| 维度 | 代码 | 评估内容 | 满分 |
|------|------|---------|------|
| 逻辑性 | LOGIC | P00-P20 谓词通过率 | 10 |
| 叙事性 | NARR | 节拍覆盖、弧线对齐、线索植入 | 10 |
| 视觉性 | VIS | 景别/角度/运动的叙事合理性 | 10 |
| 节奏性 | RHYTHM | 时长分布、张力曲线、呼吸点 | 10 |
| 声音性 | SOUND | 声画关系设计、沉默使用 | 10 |
| 创造性 | CRAFT | 超越模板的创新、视觉隐喻、风格感 | 10 |

评级:
  50-60 = 大师级（可直接作为拍摄参考）
  40-49 = 优秀（少量可改进）
  30-39 = 良好（基本功扎实但缺亮点）
  20-29 = 及格（多处逻辑或节奏问题）
  < 20  = 不合格（需要重做）
```

### 迭代修正流程

```
IF any Q01-Q12 = FAIL:
  1. 识别失败指标对应的步骤:
     Q01-Q03 FAIL → 返回 Step 4 (SHOOT) 或 Step 4.5 (VERIFY)
     Q04-Q09 FAIL → 返回 Step 6 (RHYTHM)
     Q10 FAIL     → 返回 Step 6 (RHYTHM) 调整 D_actual
     Q11 FAIL     → 返回 Step 6.3 (短语划分)
     Q12 FAIL     → 返回 Step 6.5 (Tempo Map) 或 Step 4 (调整高潮镜头时长)

  2. 执行修复（最多 3 轮迭代）

  3. 重新验证所有受影响的指标

  4. 如果 3 轮后仍有 FAIL → 标记为已知缺陷 + 改进建议

IF all Q01-Q12 = PASS 或 WARN:
  → 输出审计报告 → 继续 Step 9 (SDL)
```

### 输出

## 十、Step 9: SDL（场景描述语言生成）

> **交叉引用**: `image-prompt-gen.md` 是本步骤的核心参考文件。
> - §一 定义了 SDL 架构（模型无关设计）和从分镜参数到 SDL 的转换规则
> - §二 定义了资产引用系统（@人物 / #非人物 双符号设计）
> - §三+ 定义了各模型的 Adapter 实现
>
> SDL 是分镜序列通往可视化的桥梁——每个镜头被编码为一个结构化 JSON 对象，
> 随后由 Adapter 层翻译为具体图像生成模型的 API 参数。
> 本步骤的产出直接被 `/visualize` 命令消费。

### 目标
将完成并通过审计的分镜序列转化为模型无关的场景描述语言 (SDL) 中间表征，
使其可以被任意图像生成模型（fal.ai / SD / DALL-E / MJ 等）消费。

### 输入
Step 8 通过审计的分镜序列（含全部标注）。

### 过程

```
9.1 逐镜头 SDL 对象构建
  对每个镜头，生成以下 SDL JSON 结构:

  {
    "id": "S{nn}",
    "beat": "B{nn}",
    "beat_type": "{类型}",
    "narrative_gravity": {1-5},

    // 摄影参数 → 画面构成
    "shot_size": "{景别}",           // → sdl.framing
    "camera_angle": "{角度}",        // → sdl.perspective
    "camera_movement": "{运动}",     // → sdl.motion
    "duration_seconds": {N},

    // 构图 → 画面布局
    "composition": {
      "foreground": "{FG描述 或 null}",
      "midground": "{MG描述}",
      "background": "{BG描述}",
      "rule_of_thirds_position": "{主体位置}",
      "leading_lines": "{引导线描述}",
      "framing": "{框架描述}"
    },

    // 声音 → 情绪辅助（不直接生成图像，但影响氛围描述）
    "sound": {
      "dialogue": "{台词 或 null}",
      "voiceover": "{旁白 或 null}",
      "ambient": "{环境音 或 null}",
      "sfx": "{音效 或 null}",
      "music": "{音乐 或 null}",
      "silence_seconds": {N}
    },

    // 逻辑 → 元数据（供验证和追溯）
    "logic": {
      "cut_reason": "{切理由}",
      "v_requires": ["{依赖}"],
      "v_provides": ["{提供}"],
      "n_raises": ["{悬念}"],
      "scale_direction": "scale_in | scale_out | neutral",
      "idiom": "{所属idiom 或 null}",
      "viewpoint": "{视角类型}"
    },

    // 转场
    "transition_in": "{CUT | DISSOLVE | FADE | MATCH | ...}",
    "montage_type": "{度量 | 节奏 | 调性 | 泛音 | 理性}",

    // 资产引用 → 角色/场景一致性
    "asset_references": {
      "characters": [
        {"ref": "@{角色名}", "variant": "{表情/姿态}", "resolved_url": null}
      ],
      "assets": [
        {"ref": "#{场景/物件}", "variant": "{变体}", "resolved_url": null}
      ]
    },

    // 注释
    "blocking_notes": "{调度说明}",
    "director_notes": "{导演注释}",
    "reference_films": ["{参考电影}"]
  }

9.2 SDL 到画面描述的映射
  参照 image-prompt-gen.md §一 的转换规则:

  景别 (CU/MS/LS...)     → sdl.framing（主体占比 + 可见身体范围）
  角度 (HIGH/LOW/DUTCH...)→ sdl.perspective（视角 + 透视变形）
  构图 (三分法/引导线...) → sdl.composition（主体位置 + 背景布局）
  色温 T_color()          → sdl.palette（色彩 + 光源）
  深度 (FG/MG/BG)        → sdl.depth（景深 + 焦点 + 前后景）
  隐喻 (M01-M28)         → sdl.symbolism（象征元素）
  运动 (PUSH/TRACK...)   → sdl.motion（运动模糊 + 动态构图）

9.3 资产引用解析
  @ 符号: 人物角色 → 需要面部一致性（reference portrait / IP-Adapter）
  # 符号: 非人物 → 需要环境一致性（background reference / ControlNet）

  语法:
    @角色名           → 默认参考图
    @角色名.表情       → 特定表情参考图
    #场景名.变体       → 特定场景变体

9.4 Adapter 选择与输出
  SDL 对象不直接包含模型指令——Adapter 层负责翻译:

  SDL → fal Adapter  → fal.ai API payload
  SDL → SD Adapter   → Stable Diffusion / ComfyUI workflow
  SDL → DALLE Adapter → DALL-E API payload
  SDL → MJ Adapter   → Midjourney prompt

  Adapter 只是薄层翻译，不改变画面语义。
  换模型 = 换 Adapter，不需要重新生成 SDL。
```

### 输出

```
SDL 对象数组:
  [
    { "id": "S01", "beat": "B01", ... },
    { "id": "S02", "beat": "B01", ... },
    ...
    { "id": "S36", "beat": "B15", ... }
  ]

可直接传入 /visualize 命令生成图像。
```

## 十一、质量检查清单（完成后总览）

以下清单汇总了全部步骤中的质量检查点，用于最终确认。
Step 8 SELF-AUDIT 的 Q01-Q12 覆盖了其中的可计算部分；
以下分层清单用于定性审查。

### A. 逻辑层（最高优先级）— 引用 shot-logic.md v2.1 谓词 P00-P20

```
  □ [P00] 每个切有合法理由（信息/反应/强调/重定向）？
  □ [P06] 每个镜头至少一个通道 PROVIDES 非空？
  □ [P05] 每通道 REQUIRES 被前置镜头满足？(含跨通道)
  □ [P12] 相邻镜头顺序不可互换？
  □ [P08] 信息依赖图是有效DAG（无循环、无孤岛）？
  □ [P07] RAISES 链持续活跃（任意时刻 ≥ 1 活跃悬念）？
  □ [P11] 因果关系正确（原因在前、反应在后）？
  □ [P10] 三通道汇聚点对齐叙事高潮？
  □ 视角逻辑一致（谁在看？摄影机位置物理上可行？）？
```

### B. 叙事层

```
  □ 每个节拍是否有对应的镜头？
  □ 叙事弧线是否有对应的视觉弧线？
  □ 线索是否在画面中存在（Fair-play）？
  □ 误导是否通过视觉引导实现？
  □ Idiom 是否完整（建立/正反打/揭示不是半截的）？
```

### C. 视觉层

```
  □ 景别变化有叙事理由（不是"为了变化而变化"）？
  □ 角度变化反映权力/心理变化？
  □ 运动与能量方向一致？
  □ 构图利用了深度分层（FG/MG/BG）？
  □ scale-in 用于沉浸、scale-out 用于冲击？
```

### D. 蒙太奇层

```
  □ 转场服务于叙事？
  □ 连贯性规则被遵守（或有意违反并标注理由）？
  □ 180度轴线保持？
  □ 30度规则遵守？
  □ 运动跨切连贯（运动→运动 或 有意的运动→静止）？
```

### E. 节奏层

```
  □ 时长方差 σ > 1.5s？
  □ 最长/最短镜头倍率 ≥ 3？
  □ 高潮有节奏上的对应（加速或停顿）？
  □ 每 5-8 个短镜头后有呼吸点？
  □ 无连续 > 4 个相同景别的镜头？
  □ 总时长与叙事权重匹配？
```

### F. 声音层

```
  □ 每个镜头的声音设计已标注？
  □ 声画关系有意识地选择（同步/对位/先行）？
  □ 沉默的使用 > 0（至少有一处有意的沉默）？
  □ J-Cut / L-Cut 在场景转换处被考虑？
```

### G. 可计算指标（= Q01-Q12 的简表）

```
  Q01 信息效率 = 非重复信息单元 / 总镜头数  → 目标 > 0.8
  Q02 切理由覆盖率 = 有理由的切 / 总切数    → 目标 = 1.0
  Q03 依赖完整度 = REQUIRES满足 / 总镜头    → 目标 = 1.0
  Q04 节奏方差 σ                           → 目标 1.5-4.0s
  Q05 Idiom覆盖率                          → 目标 0.6-0.9
  Q06 景别分布熵                            → 目标 > 2.0 bits
  Q07 ASL                                  → 目标 3.5-6.0s (悬疑)
  Q08 r(1) lag-1自相关                      → 目标 0.3-0.7
  Q09 1/f指数 α                            → 目标 0.8-1.2
  Q10 D_MAE                                → 目标 < 1.0s
  Q11 短语CV                               → 目标 < 0.3
  Q12 高潮对齐                              → 目标 = true
```
