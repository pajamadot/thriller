# 分镜拆解核心算法

> 从一段叙事文本到一组镜头序列的完整推理过程
> 这是 `/decompose` 命令的底层思维框架——每一步的推理逻辑

---

## 一、总览

```
输入: 一段叙事文本（场景/章节/段落）
输出: 带完整标注的分镜序列

8步流程:
  PARSE → BEAT → BLOCK → SHOOT → VERIFY → CONNECT → RHYTHM → ANNOTATE
                                    ↑
                            [关键步骤: 逻辑验证]
                            信息依赖图 + 切理由检验
                            如果不通过 → 返回SHOOT修复
```

每一步都有明确的输入、输出、决策规则和质量检查。
VERIFY 步骤是 v2 新增的核心步骤——确保镜头序列具有因果逻辑而非仅仅"看起来专业"。

---

## 二、Step 1: PARSE（解析）

### 目标
将连续文本切分为可处理的「文本单元」，并提取隐含的视觉/听觉信息。

### 输入
原始叙事文本。

### 过程

```
1.1 段落切分
  按空行、场景分隔线、对话切换进行物理分割
  每个段落 = 一个初始文本单元

1.2 功能标注
  为每个文本单元标注叙事功能类型:
    [DESC] 描述：环境、外貌、物品描写
    [ACT]  动作：角色的物理行为
    [DIA]  对话：角色说的话
    [THINK]内心：角色的内在思维
    [EXPO] 说明：叙事者的补充信息
    [TRANS]过渡：时间/空间的跳转标记

1.3 隐含信息提取
  从文本中提取没有明确写出但隐含的视觉/听觉元素:

  示例（《镜中访客》ch1）:

  文本: "我又没有睡好。"
  显性: 角色内心独白
  隐含:
    - 视觉: 角色的状态（疲惫的面容、凌乱的头发）
    - 空间: 角色醒来的环境（卧室、床）
    - 时间: 早晨
    - 声音: 闹钟？安静？

  文本: "林小曼准时推门进来。"
  显性: 角色行动（进入）
  隐含:
    - 视觉: 门的开合动作、林小曼的全貌
    - 空间: 咨询室的布局
    - 声音: 门声
    - 调度: 从门口到沙发的路径
```

### 输出
```
文本单元列表:
  Unit 1: [THINK] "我又没有睡好。" → 隐含: 疲惫面容、卧室环境、早晨
  Unit 2: [DESC] 冲澡、脚底有灰 → 隐含: 浴室空间、水声、细节特写
  Unit 3: [EXPO] 上午取消来访 → 隐含: 时间推移、咨询室空间
  ...
```

### 质量检查
- [ ] 每个文本单元有且只有一个主要功能
- [ ] 隐含信息不与文本矛盾
- [ ] 没有遗漏关键的视觉/听觉信息

## 三、Step 2: BEAT（节拍化）

### 目标
将文本单元合并/拆分为叙事节拍——每个节拍代表一次价值变化，对应一个（或一组）镜头。

### 输入
Step 1 的文本单元列表。

### 过程

```
2.1 价值变化识别
  对每个文本单元问: "这里有没有价值状态的改变？"

  价值维度:
    - 权力关系: 谁占上风？
    - 情感状态: 角色的情绪？
    - 信息量: 角色/观众知道了什么新东西？
    - 威胁等级: 危险增加还是减少？
    - 角色距离: 角色之间更近还是更远？

  如果有变化 → 这是一个节拍边界
  如果没有变化 → 合并到前一个节拍

2.2 节拍分类
  行动节拍 [ACT-B]: 物理动作推进情节
  反应节拍 [REACT-B]: 角色对事件的内在回应
  揭示节拍 [REVEAL-B]: 新信息呈现
  转折节拍 [TURN-B]: 场景方向转变
  张力节拍 [TENSION-B]: 悬念施压但无释放
  释放节拍 [RELEASE-B]: 张力暂时释放
  过渡节拍 [TRANS-B]: 时空桥接

2.3 叙事重力标注
  5 = 故事转折点（全局影响）
  4 = 场景转折点（本场核心）
  3 = 重要推进（实质进展）
  2 = 辅助建设（次要信息）
  1 = 过渡连接（叙事胶水）

2.4 弧线验证
  将节拍按顺序排列，检查:
    - 是否形成有效的戏剧弧线？
    - 最高重力的节拍是否在合理的位置（通常在后半段）？
    - 是否有足够的上升（不能全是平的节拍）？
    - 是否有呼吸点（不能全是高重力节拍）？
```

### 示例应用

```
《镜中访客》ch1 的节拍化:

Beat 1 [TRANS-B, 重力1]:
  叶知秋没有睡好，空白记忆
  价值变化: 无（建立初始状态）

Beat 2 [REVEAL-B, 重力2]:
  脚底有灰 → 自我解释
  价值变化: 信息+1（异常信号，被角色否认）

Beat 3 [TRANS-B, 重力1]:
  上午工作，钢笔没墨，百乐笔（不记得买）
  价值变化: 信息+1（再次异常，被角色忽略）

Beat 4 [ACT-B, 重力2]:
  林小曼到来，今天化了妆
  价值变化: 场景变化（进入咨询模式）

Beat 5 [REVEAL-B, 重力4]:
  "有人在跟踪我"→ 那个人说出了咨询中的原话
  价值变化: 威胁+3, 信息+3（核心悬念建立）

Beat 6 [REACT-B, 重力3]:
  叶知秋的内心反应 → "有人泄露了咨询记录"
  价值变化: 威胁+1（职业危机）

Beat 7 [ACT-B, 重力2]:
  询问周明哲的信息
  价值变化: 信息+1（嫌疑人浮现）

Beat 8 [TENSION-B, 重力3]:
  送走林小曼后独坐 → 内心焦虑
  价值变化: 威胁持续, 情感压力

Beat 9 [TRANS-B, 重力1]:
  回到公寓

Beat 10 [REVEAL-B, 重力3]:
  帽衫 → 自我解释
  价值变化: 信息+1（又一个异常被否认）

Beat 11 [ACT-B, 重力2]:
  分析周明哲的动机
  价值变化: 信息+1（推理推进）

Beat 12 [REVEAL-B, 重力3]:
  鞋子上的新鲜泥点
  价值变化: 信息+1（最强的异常信号）

弧线: 1→2→1→2→4→3→2→3→1→3→2→3
      ─────╱──╱╲──╲──╱──╱──
      建立→上升→高峰→缓和→上升→上升
```

### 输出
```
节拍序列:
  [Beat 1] TRANS-B (1) → [Beat 2] REVEAL-B (2) → ... → [Beat 12] REVEAL-B (3)

弧线形状: 双峰式（Beat 5 第一峰, Beat 12 第二峰）
```

## 四、Step 3: BLOCK（场面调度）

### 目标
确定每个节拍中角色在空间中的位置和运动。

### 输入
节拍序列 + 场景空间信息。

### 过程

```
3.1 空间建模
  绘制场景的简易平面图:
    - 门/窗的位置
    - 家具/关键物品的位置
    - 光源位置
    - 摄影机可能的位置

3.2 角色起始位置
  每个角色在场景开始时在哪里？
  这决定了初始的调度模式（A/I/L）

3.3 运动轨迹设计
  为每个有运动的节拍设计角色移动:
    - 从哪到哪？
    - 速度？
    - 动机（为什么移动？）
    - 移动如何改变角色间的距离关系（Proxemics）？

3.4 关键位置标注
  标注以下关键时刻的精确位置:
    - 场景的最高张力点 → 角色在哪里？面向哪里？
    - 揭示时刻 → 被揭示的物品/信息在空间中的位置？
    - 进入/离开 → 角色如何出入？
```

### 示例

```
《镜中访客》ch1 — 咨询室场景

空间:
  ┌─────────────────────────┐
  │          窗户             │
  │                          │
  │  ■ 叶知秋的椅子          │
  │      [桌子]              │
  │                          │
  │      ■ 沙发（林小曼）     │
  │                          │
  │                    门 ←──│
  └─────────────────────────┘

调度模式: A型（面对面，隔桌对坐）
距离: 社交区（~200cm）→ 不变（两人都不移动 = 职业距离维持）

角色运动:
  Beat 4: 林小曼从门口走到沙发 → I型(进入) → A型(落座)
  Beat 5-7: 无大幅移动 → A型维持
  Beat 8: 叶知秋独坐 → 无对象的A型（对面空了）= 孤独
```

### 输出
- 场景平面图
- 每个节拍的角色位置
- 调度模式标注
- 距离变化轨迹

## 五、Step 4: SHOOT（镜头设计）

### 目标
为每个节拍选择具体的镜头参数。

### 输入
节拍序列 + 调度设计。

### 决策规则

```
4.1 景别选择
  规则: 情感距离 = 物理距离

  重力 5 (故事转折) → ECU / CU
  重力 4 (场景转折) → CU / MCU
  重力 3 (重要推进) → MCU / MS
  重力 2 (辅助建设) → MS / MLS
  重力 1 (过渡连接) → LS / ELS

  但需要检查:
    - 是否需要看到环境？→ 宽景
    - 是否需要看到动作？→ 全景/中景
    - 是否需要看到情感？→ 特写
    - 是否需要看到线索？→ INSERT

4.2 角度选择
  规则: 权力关系 = 垂直关系

  问: 此刻谁占优势？
    说话者占优 → 仰拍说话者
    听者占优（沉默的力量）→ 俯拍说话者
    平等 → 平视
    世界失衡 → 荷兰角
    代入体验 → POV

4.3 运动选择
  规则: 摄影机的运动 = 叙事的能量流

  问: 这个节拍的能量方向是什么？
    聚焦/发现 → 推近
    释放/揭示全貌 → 拉远
    跟随行动 → 横移/稳定器
    不稳定/恐慌 → 手持
    观察/审视 → 静止
    认知失调 → Dolly Zoom

4.4 声音设计
  规则: 声画关系三选一

  同步: 声音 = 画面（强化）
  对位: 声音 ≠ 画面（反讽/不安）
  先行: 声音 > 画面（预示）
```

### 反直觉规则

```
以下情况应违反"重力=景别"的默认规则:

1. 用远景拍高重力节拍:
   当角色的渺小本身就是故事的重点
   例: 角色在空荡的街道上奔跑 → ELS表达无助

2. 用特写拍低重力节拍:
   当角色的微表情包含关键线索
   例: 路人甲的ECU → 揭示章节后回想才发现是凶手

3. 用静止拍高能量节拍:
   当"不动"比"动"更有力量
   例: 角色被告知亲人死亡 → 15秒的静止CU（比任何运动更震撼）

4. 用运动拍低能量节拍:
   当摄影机运动制造不安
   例: 角色睡觉 → 缓慢环绕拍摄（谁在看他？）
```

### 输出
```
每个节拍的镜头参数:
  Beat 5: CU | HIGH→EYE | STATIC→PUSH |
          声音: DIA+SIL | 时长: 8s
```

## 五-b、Step 4.5: VERIFY（逻辑验证）[关键步骤]

### 目标
在继续蒙太奇设计之前，验证镜头序列的信息逻辑。
**这是防止"看起来专业但逻辑不通"的核心步骤。**

> **交叉引用**: shot-logic.md v2.1 第五章定义了形式化谓词 P01-P20。
> 本步骤应按照该谓词表逐条执行，不得用散文描述替代。
> 每个谓词返回 PASS / WARN / FAIL，结果直接映射到 evaluation.md 的 LOGIC 分数。

### 输入
Step 4 的镜头参数序列。

### 过程

```
4.5.1 多通道信息依赖图构建 [引用 shot-logic.md §二]
  对每个镜头标注三通道属性:

  V-REQUIRES / V-PROVIDES / V-RAISES (视觉通道)
  A-REQUIRES / A-PROVIDES / A-RAISES (听觉通道)
  N-REQUIRES / N-PROVIDES / N-RAISES (叙事潜文本通道)

  计算 INFO_DENSITY = |V-PROVIDES| + |A-PROVIDES| + |N-PROVIDES|

4.5.2 切理由验证 — 执行谓词 P00-P04 [引用 shot-logic.md §五 Level 1]
  逐个切检查:
    □ P01: INFORMATION — 新信息需要不同取景？
    □ P02: REACTION — 需要展示反应？
    □ P03: EMPHASIS — 需要改变情感距离？
    □ P04: ORIENTATION — 需要重新建立空间认知？ [v2.1 新增]
    □ P00: 至少一个谓词 PASS？（否则 = 无因之切 → FAIL）
  切理由可复合标注: REASON = INFORMATION + EMPHASIS

4.5.3 依赖验证 — 执行谓词 P05-P10 [引用 shot-logic.md §五 Level 2]
    □ P05: 每通道 REQUIRES 被前置镜头满足？(含跨通道满足规则)
    □ P06: 至少一个通道 PROVIDES 非空？
    □ P07: RAISES 链持续活跃？(任意时刻 active_questions ≥ 1)
    □ P08: DAG 无循环？
    □ P09: INFO_DENSITY ≤ 5 (建立镜头 ≤ 7)？
    □ P10: 三通道汇聚点对齐叙事高潮？

4.5.4 位置验证 — 执行谓词 P11-P14 [引用 shot-logic.md §五 Level 3]
    □ P11: 原因在反应之前？(延迟揭示需标注)
    □ P12: 相邻镜头顺序不可互换？
    □ P13: 无冗余镜头？(或冗余有标注理由)
    □ P14: canonical provider 已标注？

4.5.5 参数验证 — 执行谓词 P15-P18 [引用 shot-logic.md §五 Level 4]
    □ P15: 景别选择有叙事理由？
    □ P16: 景别跳跃 ≤ 2 级 (或有理由)？(INSERT 豁免)
    □ P17: 角度反映权力/心理关系？
    □ P18: 时长与信息密度匹配？

4.5.6 序列验证 — 执行谓词 P19-P20 [引用 shot-logic.md §五 Level 5]
    □ P19: 所有 idiom 完整？（建立/正反打/揭示/追逐/反应链/窥视）
    □ P20: 5-镜头窗口内始终有活跃悬念？(场景末3镜头豁免)
```

### 输出
```
验证报告:
  PASS: N 个谓词
  WARN: M 个谓词
  FAIL: K 个谓词

  通过标准:
    FAIL = 0 且 WARN ≤ 3  → 序列通过验证 → 继续 Step 5
    FAIL = 0 且 WARN > 3   → 需要优化但可继续（记录 WARN）
    FAIL ≥ 1               → 必须修复 → 返回 Step 4

  附带输出:
    多通道信息依赖图（三通道 DAG）
    每个 FAIL 的具体镜头 + 修复建议
    WARN 项的观察记录

  → evaluation.md LOGIC 分数映射:
    LOGIC = 10 × (1 - FAIL/total) × (1 - 0.1×WARN/total)
```

## 六、Step 5: CONNECT（蒙太奇设计）

### 目标
设计镜头间的转场和关系。

### 输入
经过验证的镜头参数序列。

### 过程

```
5.1 转场选择
  默认: CUT（90%的情况）

  需要特殊转场的信号:
    - 场景变化 → DISSOLVE / FADE / MATCH
    - 时间跳跃 → FADE / DISSOLVE / JUMP CUT
    - 回忆 → DISSOLVE
    - 声音桥接 → L-Cut / J-Cut
    - 视觉关联 → MATCH / GMATCH
    - 连续感 → INVISIBLE CUT

  切的认知效果验证（Liao et al., 2023）:
    scale-in 切: 流畅，观众几乎不注意 → 用于沉浸段
    scale-out 切: 显著，观众明确感知 → 用于冲击/呼吸
    同景别切: 需要≥30°角度变化，否则是跳切
    运动→运动: 最流畅的切（视线追踪延续）
    静止→静止: 最明显的切（每个切是一次"击打"）

5.2 蒙太奇类型标注
  问: 这组镜头为什么以这个顺序排列？
    - 时长决定节奏 → 度量蒙太奇
    - 画面内运动决定 → 节奏蒙太奇
    - 情绪基调统一 → 调性蒙太奇
    - 多维度叠加 → 泛音蒙太奇
    - 概念碰撞 → 理性蒙太奇

5.3 连贯性检查
  逐对检查相邻镜头:
    □ 180度规则: 摄影机在轴线同侧？
    □ 30度规则: 角度变化≥30°？
    □ 视线匹配: 视线方向正确？
    □ 动作匹配: 动作连贯？
    □ 银幕方向: 运动方向一致？
    □ 动作连贯: 前镜有运动→后镜接运动？(Magliano, 2011)
```

## 七、Step 6: RHYTHM（节奏校准）

### 目标
确保镜头序列的时间节奏符合叙事需要。

### 输入
完整的分镜序列（含时长标注）。

### 过程

```
6.1 时长分配 — Duration Function（参见 visual-rhythm.md §八）
  使用数学化的时长函数计算每个镜头的初始时长:

  D(shot) = D_base(size) * G(gravity) * B(beat_type) * S(style)

  参数表（完整值见 visual-rhythm.md）:
    D_base: ELS=5.0, LS=4.0, FS=3.5, MS=3.0, MCU=2.5, CU=3.0, ECU=2.0, INSERT=2.0
    G: g1=0.70, g2=0.90, g3=1.00, g4=1.30, g5=1.80
    B: action=0.80, reaction=1.20, reveal=1.50, tension=1.80, transition=0.60, ...
    S: neutral=1.00, hitchcock=1.20, fincher=0.85, ...

  附加修正:
    台词: D_speech = max(D, T_read + 0.5), T_read = 字数/语速
    开场: 场景首镜 × 2.0-2.5
    沉默: D × 1.3-1.5
    长镜头: 豁免30s上限, D = T_content + T_padding

6.2 时间记号确定（参见 visual-rhythm.md §九）
  为场景选择时间记号:
    审讯 → 4/4 synco
    追逐 → 6/8 accel
    发现 → 3/4 rubato
    对话 → 4/4 rubato
    蒙太奇 → var metric
    冥想 → free

6.3 短语划分（参见 visual-rhythm.md §十一）
  将镜头序列按短语边界分组:
    每个短语 3-5 镜头（最多7）
    边界位于: 话题转换/能量转变/空间切换/焦点转移/时间间断
    短语按段落组织: Building → Climax → Resolution

6.4 节奏模式选择与组合（参见 visual-rhythm.md §十）
  为每个段落/短语选择节奏模式:
    使用 Composition Algebra 组合基础模式:
    CONCAT, NEST, INVERT, SCALE, ACCENT, INTERLEAVE

  示例:
    interrogation = CONCAT(SCALE(Breathe,1.0), NEST(Accelerando,Pulse), Fermata)

6.5 Tempo Map 绘制（参见 visual-rhythm.md §十二）
  绘制场景的BPM曲线:
    BPM(shot_n) = 60 / D(shot_n)
  检查:
    - BPM曲线是否有明确方向？（随机波动=无节奏意图）
    - BPM峰/谷是否与叙事高潮/低点对齐？
    - 是否有呼吸（BPM下降点）？

6.6 数学验证
  计算节奏指标:
    σ (时长标准差): 目标 1.5-4.0s（快闪蒙太奇豁免）
    ASL (平均镜头时长): 对照类型目标（悬疑3.5-6.0s）
    r(1) (lag-1自相关): 目标 0.3-0.7（接近1/f分布）
    1/f指数 α: 目标 0.8-1.2（N>15时可靠）

6.7 总时长估算
  Σ D(shot_i) = 场景总时长
  检查: 是否与场景的叙事权重匹配？
  一般参考: 一页剧本 ≈ 1分钟银幕时间
```

## 八、Step 7: ANNOTATE（导演注释）

### 目标
为关键镜头添加意图说明和参考信息。

### 输入
完成的分镜序列。

### 过程

```
7.1 意图说明
  对重力≥3的镜头，写明:
    - 为什么选择这个景别/角度/运动？
    - 观众在这个镜头应该感受到什么？
    - 这个镜头与前后镜头的关系是什么？

7.2 参考标注
  如果某个镜头设计参考了已知电影:
    - 注明电影名、场景、时间码（如可能）
    - 说明参考了什么（构图？节奏？色调？）

7.3 替代方案
  对有多种可能处理的镜头:
    - 列出 1-2 个替代设计
    - 说明替代方案的不同效果

7.4 技术注意事项
  特殊镜头的技术要求:
    - Dolly Zoom: 需要变焦镜头 + 轨道
    - 长镜头: 演员需要完整排练

    - Steadicam: 设备/操作员
```

## 九、质量检查清单（完成后）

### A. 逻辑层（最高优先级）— 引用 shot-logic.md v2.1 谓词 P00-P20

```
  □ [P00] 每个切有合法理由（信息/反应/强调/重定向）？
  □ [P06] 每个镜头至少一个通道 PROVIDES 非空？
  □ [P05] 每通道 REQUIRES 被前置镜头满足？(含跨通道)
  □ [P12] 相邻镜头顺序不可互换？
  □ [P08] 信息依赖图是有效DAG（无循环、无孤岛）？
  □ [P07] RAISES 链持续活跃（任意时刻 ≥ 1 活跃悬念）？
  □ [P11] 因果关系正确（原因在前、反应在后）？
  □ [P10] 三通道汇聚点对齐叙事高潮？
  □ 视角逻辑一致（谁在看？摄影机位置物理上可行？）？
```

### B. 叙事层

```
  □ 每个节拍是否有对应的镜头？
  □ 叙事弧线是否有对应的视觉弧线？
  □ 线索是否在画面中存在（Fair-play）？
  □ 误导是否通过视觉引导实现？
  □ Idiom 是否完整（建立/正反打/揭示不是半截的）？
```

### C. 视觉层

```
  □ 景别变化有叙事理由（不是"为了变化而变化"）？
  □ 角度变化反映权力/心理变化？
  □ 运动与能量方向一致？
  □ 构图利用了深度分层（FG/MG/BG）？
  □ scale-in 用于沉浸、scale-out 用于冲击？
```

### D. 蒙太奇层

```
  □ 转场服务于叙事？
  □ 连贯性规则被遵守（或有意违反并标注理由）？
  □ 180度轴线保持？
  □ 30度规则遵守？
  □ 运动跨切连贯（运动→运动 或 有意的运动→静止）？
```

### E. 节奏层

```
  □ 时长方差 σ > 1.5s？
  □ 最长/最短镜头倍率 ≥ 3？
  □ 高潮有节奏上的对应（加速或停顿）？
  □ 每 5-8 个短镜头后有呼吸点？
  □ 无连续 > 4 个相同景别的镜头？
  □ 总时长与叙事权重匹配？
```

### F. 声音层

```
  □ 每个镜头的声音设计已标注？
  □ 声画关系有意识地选择（同步/对位/先行）？
  □ 沉默的使用 > 0（至少有一处有意的沉默）？
  □ J-Cut / L-Cut 在场景转换处被考虑？
```

### G. 可计算指标

```
  信息效率 = 非重复信息单元 / 总镜头数  → 目标 > 0.8
  切理由覆盖率 = 有理由的切 / 总切数    → 目标 = 1.0
  依赖完整度 = REQUIRES满足 / 总镜头    → 目标 = 1.0
  节奏方差 σ                           → 目标 1.5-4.0s
  Idiom覆盖率                          → 目标 0.6-0.9
  景别分布熵                            → 目标 > 2.0 bits
```
