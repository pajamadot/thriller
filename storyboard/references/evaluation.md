> 量化与定性相结合的分镜评估框架：6维度加权评分 + 导演解剖法
> 评估标准随类型适配（悬疑/动作/文艺/恐怖权重不同）

## 1.5、十二项量化指标体系（12 Quality Metrics）

> Band 4-7 进化产物。前 6 项（信息效率、切理由覆盖、依赖完整、Idiom覆盖、景别熵、时长方差σ）
> 为 v1 原始指标；后 6 项（ASL、r(1)、D_MAE、phrase_CV、climax_alignment、
> cognitive_load_compliance）为 100-round 进化中新增的量化指标。

### 指标定义表

| # | 指标 | 代码 | 计算公式 | 目标值 | 来源 |
|---|------|------|---------|--------|------|

| 2 | 切理由覆盖 | cut_cov | 有明确因果理由的切数 / 总切数 | = 1.0 | shot-logic.md P00-P04 |

| 5 | 景别熵 | size_H | -Σ p(size)·log2(p(size)) | ≥ 2.0 bits | shot-grammar.md |
| 6 | 时长方差 | sigma | std(durations) | 1.5-4.0s | visual-rhythm.md §七 |

### r(1) 和 1/f 指数 α 的解读

```
r(1) = lag-1 自相关系数, 测量相邻镜头时长的相关性

r(1) ≈ 0    → 白噪声: 时长完全随机，无节奏模式
r(1) ∈ [0.3, 0.7] → 粉噪声(1/f): 最佳参与度（Cutting 2010 发现）
r(1) > 0.85  → 棕噪声: 过度可预测，观众走神
r(1) < 0     → 反相关: 刻意的长-短交替（Breathe模式——若有意则可接受）

α = 1/f 指数 (仅 N>15 时计算):
  α ∈ [0.8, 1.2] → 最优 (粉噪声)
  α < 0.5 → 白噪声倾向 (过度随机)
  α > 1.5 → 棕噪声倾向 (过度规律)

计算方法:
  1. 对 duration 序列做功率谱密度 (PSD)
  2. 双对数坐标上做线性回归
  3. 斜率的绝对值 = α
```

### D_MAE 计算示例

```
Duration Function: D(shot) = D_base(size) × G(gravity) × B(beat_type) × S(style)

对每个镜头:
  D_function = D_base × G × B × S (visual-rhythm.md §八)
  D_actual = 分镜标注的实际时长
  error = |D_actual - D_function|

D_MAE = mean(errors)

  < 1.0s: PASS — 直觉与公式高度吻合
  1.0-1.5s: WARN — 部分镜头偏离函数输出，需检查理由
  > 1.5s: FAIL — 时长分配与叙事重力/节拍类型脱节
```

### cognitive_load_compliance 检查项

```
来源: cognitive-perception.md §三

逐镜头检查:
  (1) info_units ≤ max_for_size(shot_size) ?
      ECU: ≤2, CU: ≤3, MCU: ≤4, MS: ≤5, FS: ≤5, LS: ≤8, ELS: ≤3(叙事信息)
  (2) actual_duration ≥ min_duration(info_units, shot_size) ?
      min_duration = info_units × 0.8 + base_duration(size)
  (3) 不存在连续 3+ 高负荷镜头无呼吸点 (SEQ-LOAD-1)?

cognitive_load_compliance = all checks pass → true
  任何检查失败 → false + 列出违规镜头
```

## 二、逐维度评分细则

### LOGIC（逻辑性）0-10

> **交叉引用**: shot-logic.md v2.1 定义了 20 个形式化谓词 (P01-P20)。
> LOGIC 分数可通过谓词执行结果直接计算:
> `LOGIC = 10 × (1 - FAIL/total_predicates) × (1 - 0.1×WARN/total_predicates)`
> 其中 total_predicates = 实际执行的谓词数（P00-P20 中适用于当前序列的）。

```
10分: (FAIL=0, WARN≤1)
  - 每个切有明确的因果理由（信息/反应/强调/重定向, P00-P04）
  - 多通道信息DAG完整无缺口 (P05 全PASS)
  - 三通道汇聚点对齐叙事高潮 (P10 PASS)
  - 所有Idiom完整且正确使用 (P19 PASS)
  - 无任何10种逻辑错误（错误词典 §七）
  - 相邻镜头顺序不可互换 (P12 全PASS)

7-9分: (FAIL=0, WARN=2-4)
  - 95%以上的切有理由
  - DAG基本完整，1-2个小缺口或跨通道满足
  - Idiom基本完整
  - 1-2个轻微逻辑 WARN

4-6分: (FAIL=1-2, WARN=3-5)
  - 70-90%的切有理由
  - DAG有明显缺口
  - 有不完整的Idiom
  - 3-5个逻辑问题

1-3分: (FAIL≥3)
  - <70%的切有理由
  - DAG不连贯
  - 多处逻辑错误

0分:
  - 镜头序列是随机排列的（P00 几乎全 FAIL）
```

### NARR（叙事性）0-10

```
10分:
  - 每个节拍都有对应的镜头覆盖
  - 叙事弧线与视觉弧线完美对齐
  - 线索在画面中以Fair-play方式植入
  - 误导通过视觉引导精确实现
  - 视角选择一致且有叙事意图

7-9分:
  - 节拍覆盖完整
  - 弧线基本对齐
  - 线索植入存在但可以更精妙

4-6分:
  - 节拍有遗漏
  - 弧线偶尔脱节
  - 线索植入方式过于直白或缺失

1-3分:
  - 大量节拍未覆盖
  - 视觉与叙事脱节

0分:
  - 镜头与叙事无关
```

### VIS（视觉性）0-10

```
10分:
  - 景别选择始终服务于情感距离法则
  - 角度变化精确反映权力/心理关系
  - 摄影机运动与叙事能量方向一致
  - 构图利用深度分层、三分法、负空间
  - 每个镜头的视觉设计有叙事意图

7-9分:
  - 视觉选择大部分有叙事理由
  - 构图设计合理
  - 偶尔有景别选择不够精确

4-6分:
  - 视觉选择部分缺乏理由
  - 构图设计平淡
  - 多处景别选择可以更好

1-3分:
  - 视觉选择随意
  - 缺乏构图意识

0分:
  - 全部默认MS + EYE + STATIC
```

### RHYTHM（节奏性）0-10

> 数学化评分体系（参见 visual-rhythm.md §八-§十七）

```
量化指标（必须计算）:
  σ = 时长标准差 (秒)
  ASL = 平均镜头时长 (秒)
  r(1) = lag-1自相关系数
  α = 1/f指数 (N>15时)
  phrase_regularity = 短语长度的变异系数 (CV)
  climax_alignment = 最高重力镜头是否对应BPM极值 (bool)
  D_MAE = Duration Function计算值与最终取值的MAE (秒)

10分:
  - σ ∈ [1.5, 4.0]s (快闪蒙太奇豁免)
  - ASL 在类型目标区间内 (悬疑: 3.5-6.0s)
  - r(1) ∈ [0.3, 0.7] (接近1/f, N>15时)
  - α ∈ [0.8, 1.2] (N>15时)
  - phrase_regularity: 所有短语3-5拍, CV < 0.3
  - climax_alignment = true
  - D_MAE < 1.0s
  - 节奏模式可用Composition Algebra表达
  - Time Signature标注正确且与场景类型匹配
  - Tempo Map显示有方向性的BPM曲线
  - 至少一个"建立-打破"策略（Surprise > 1.0）

7-9分:
  - σ ∈ [1.0, 5.0]s (略超标准区间)
  - ASL 在类型目标±1s内
  - r(1) ∈ [0.15, 0.8]
  - phrase_regularity: 大部分短语3-7拍
  - climax_alignment = true
  - D_MAE < 1.5s
  - Tempo Map有方向性但不完全平滑
  - 1-2处节奏不够精确

4-6分:
  - σ < 1.0 (偏均匀) 或 σ > 5.0 (偏混乱)
  - ASL 超出类型目标区间
  - r(1) < 0.1 (过于随机) 或 r(1) > 0.85 (过于相关)
  - 高潮处缺乏节奏支撑 (climax_alignment = false)
  - 无明确的短语结构
  - 无Time Signature标注

1-3分:
  - σ < 0.5 (全程近似等时长)
  - r(1) ≈ 0 (白噪声——完全随机切换)
  - 无节奏模式可识别
  - 有明显节奏病症（见 visual-rhythm.md §七）

0分:
  - 所有镜头相同时长 (σ = 0)
  - 或全部时长随机生成，无任何叙事逻辑
```

### SOUND（声音性）0-10

```
10分:
  - 每个镜头的声音设计都有标注
  - 声画关系在同步/对位/先行间有意识选择
  - 沉默被作为叙事工具使用
  - J-Cut/L-Cut在场景过渡中被恰当使用
  - 声音设计增强而非仅仅伴随画面

7-9分:
  - 声音标注完整
  - 声画关系大部分合理
  - 有沉默的使用

4-6分:
  - 声音标注不完整
  - 声画关系全部是同步（无对位/先行）
  - 缺少沉默

1-3分:
  - 声音标注缺失
  - 未考虑声画关系

0分:
  - 无声音设计
```

### CRAFT（创造性）0-10

```
10分:
  - 有至少2个超越模板的创新镜头设计
  - 有独特的视觉隐喻（空间=叙事、镜子=分裂等）
  - 有统一的导演风格感
  - 有意识地打破规则来达到效果（并能解释为什么）
  - 有至少1个参考经典电影但做了自己的变体

7-9分:
  - 有1-2个创新镜头
  - 有视觉隐喻但不够深入
  - 风格基本一致

4-6分:
  - 基本按模板执行
  - 缺少创新和隐喻
  - 风格不明确

1-3分:
  - 完全模板化
  - 无个人风格

0分:
  - 机械的景别堆叠
```

### 认知保真度检查（Cognitive Fidelity Sub-checks）

> 来源: `cognitive-perception.md` — Band 3 认知与知觉深化产物

以下认知保真度检查项嵌入 LOGIC 和 VIS 维度的评分中，作为加分/扣分项:

```
嵌入 LOGIC 维度:
  +1 如果每个镜头的信息单元数在景别限制内 (cognitive-perception.md 三)
  +1 如果所有相邻镜头之间有至少一个桥接线索 (cognitive-perception.md 九)
  -1 如果存在连续 3+ 个高负荷镜头无呼吸点 (SEQ-LOAD-1)
  -1 如果对比切前启动时长不足 (<1.5s) (PRIME-2/3)

嵌入 VIS 维度:
  +1 如果注意力引导与叙事意图对齐 (P1-P5 优先级正确使用)
  +1 如果 Fair-play 线索在低注意力优先级位置 (DESYNC 规则)
  -1 如果关键信息在低同步条件下呈现 (DESYNC-1/2)

嵌入 RHYTHM 维度:
  +1 如果镜头时长满足最低呈现时长公式
  +0.5 如果时长序列近似 1/f 模式 (RHYTHM-1F)
  -1 如果存在低于认知处理阈值的时长 (<0.3s 非故意闪现)

认知保真度总分: 上述加减项的净值
  纳入方式: 分散到 LOGIC/VIS/RHYTHM 三个维度的评分中
  最大影响: +/- 2 分 (per dimension)
```

## 三、`/score` 命令

```
/score [分镜序列]

输入: 已完成的分镜序列（表格或JSON）
输出:
  1. 六维度评分（含细节说明）
  2. 十二项量化指标
  3. 加权总分（按类型权重）
  4. 强项/弱项诊断
  5. 改进建议（最多3条，按优先级）
  6. Tempo Map 可视化

可选参数:
  --genre <psych_thriller|noir|procedural|horror_adj|k_thriller|art_auteur>
  --compare <另一个分镜序列>  对比模式
  --benchmark  与3个基准场景对比
  --verbose  包含逐镜头的谓词执行结果
```

```
  4. 与当前方法论的对比（"我们的框架能解释吗？"）
  5. 方法论盲点标注（"这个技巧我们还没有"）
```

### 解剖协议（5步）

```
Step 1: 镜头清单重建
  列出场景中的每个镜头:
    编号、景别、角度、运动、时长（估计）、内容描述

Step 2: 切理由逆推
  对每个切，推断导演的理由:
    为什么从镜头A切到镜头B？
    是信息/反应/强调中的哪一个？
    有没有超出三种理由的第四种？

Step 3: 信息DAG逆推
  为每个镜头标注 REQUIRES / PROVIDES / RAISES
  画出信息流图
  检查: 这个导演的信息管理策略是什么？

Step 4: 模式识别
  识别导演使用的Idiom:
    标准Idiom（建立、正反打、揭示、追逐）？
    变体Idiom（标准模式的创新变化）？
    全新Idiom（我们框架中没有的模式）？

Step 5: 方法论对比
  我们的框架能完全解释这个场景吗？
  如果不能:
    哪个维度/规则无法覆盖？
    这是一个[Experiment]级别的新发现？
    应该加入哪个参考文件？
```

### 工作示例 1: 《沉默的羔羊》— Clarice 初见 Lecter

```markdown
## 视觉解剖: 《沉默的羔羊》(Demme, 1991) — 地下室走廊 + 初次对话

### 场景概要
Clarice Starling 穿过巴尔的摩精神病院的地下走廊，走向 Hannibal Lecter 的牢房。
约2分钟，15个镜头。Demme 使用了正面直视摄影机的对话拍摄法。

### 镜头清单 (关键镜头)
| # | 景别 | 角度 | 运动 | 时长 | 内容 | 切理由 |
|---|------|------|------|------|------|--------|
| 1 | LS | EYE | TRACK-IN | 8s | Clarice走过走廊, 两侧牢房 | 建立:空间+威胁渐近 |
| 2 | MS | EYE | STEADY | 4s | 经过第一个囚犯 | 信息:威胁环境 |
| 3 | MCU | EYE→LOW | STATIC | 3s | 囚犯1低语 | 信息:危险确认 |
| 4 | CU | EYE | TRACK-IN | 6s | Clarice面部(正面直视) | 反应:紧张但坚定 |
| 5 | MS | EYE | STATIC | 5s | 到达Lecter牢房 | 信息:目标到达 |
| 6 | CU | EYE | STATIC | 3s | Lecter正面直视(第一次) | 强调:正面凝视=审视 |
| 7 | CU | EYE | STATIC | 4s | Clarice正面直视 | 反应:被审视 |
| ... | | | | | | |

### 导演策略提取

1. **Demme的正面凝视法**: Clarice和Lecter的对话CU都是面向摄影机的正面拍摄,
   而非标准OTS。效果: 观众被迫成为对话的接受者。当Lecter说"你好,Clarice",
   他在对观众说。
   → 我们的框架: shot-grammar.md 的角度体系中没有"正面直视"(frontal)的独立类别
   → 建议: 在OTS/Single分类中增加 "Frontal Single" 作为子类型

2. **走廊的信息递进**: 走廊长镜头中每个牢房是一个"层级"——
   第一个囚犯(低威胁) → 第二个(中威胁) → 空牢房(恐惧:什么都没有更可怕)
   → Lecter(最高威胁但最文明) = 反直觉递进
   → 我们的框架: shot-logic.md P05 的 PROVIDES 链可以描述,
     但缺少"反直觉递进"(anti-climactic escalation)的命名

3. **景别与权力的反转**: 按常规, Lecter(威胁源)应被LOW ANGLE拍摄。
   但Demme始终用EYE LEVEL → Lecter的威胁来自智识,不是物理 → 平视更恐怖
   → 我们的框架: 完全可解释(shot-grammar.md的"平视=平等"法则)
     但需要补充: "刻意平视本应仰拍的威胁源 = 通过违反期望制造更深的不安"

### 量化指标 (场景)
  ASL = 4.2s, σ = 1.8s, r(1) = 0.62
  → 全部落在心理悬疑目标区间内 ✓

### 对当前框架的贡献
  → [Gap] "正面直视对话"(Frontal SRS) 应纳入 shot-grammar.md 角度体系
  → [Gap] "反直觉递进"模式应命名并加入 shot-logic.md Idiom 3 变体
  → [Validation] Demme的指标完美落在评分系统目标区间 = 验证通过
```

### 工作示例 2: 《记忆碎片》— 开场逆序蒙太奇

```markdown
## 视觉解剖: 《记忆碎片》(Nolan, 2000) — 开场宝丽来照片 + 倒叙第一序列

### 场景概要
电影以一张宝丽来照片"褪色"开场(实际是倒放)，然后展示一场谋杀的逆序过程。
约90秒, 12个镜头, 全部倒放或逆序剪辑。

### 镜头清单 (关键镜头)
| # | 景别 | 角度 | 运动 | 时长 | 内容 | 切理由 |
|---|------|------|------|------|------|--------|
| 1 | ECU | EYE | STATIC | 8s | 宝丽来照片(倒放——图像褪色) | 建立:时间倒流 |
| 2 | INSERT | BIRD | STATIC | 3s | 照片从手中飞回相机 | 信息:倒叙规则 |
| 3 | MS | EYE | STATIC | 4s | 手握枪(倒放) | 信息:暴力+倒叙 |
| 4 | CU | LOW | STATIC | 2s | 子弹壳从地面飞回枪膛 | 强调:物理法则颠覆 |
| 5 | LS | EYE | STATIC | 5s | 血液缩回尸体(倒放) | 信息:死亡被撤销 |
| ... | | | | | | |

### 导演策略提取

1. **倒序作为认知隐喻**: 物理世界的倒放=主角的记忆状态。
   观众从结果回推原因 = Leonard 的日常体验。
   → 我们的框架: montage-theory.md 的5级蒙太奇均基于正向时间
   → [Gap] 缺少"逆向蒙太奇"(Reverse Montage)的理论框架
   → 建议: 加入第六种蒙太奇模式——时间操控蒙太奇

2. **ECU的时间扭曲**: 开场8秒的宝丽来ECU是异常长(ECU基准=2s,
   D_function ≈ 2.0×0.7×1.5×0.95 = 2.0s, 实际8s → 偏差6s → D_MAE爆表)。
   但这正是Nolan的意图: 观众需要8秒才能理解"这是倒放"。
   → 我们的框架: D_MAE 会给出 FAIL, 但这是有效的规则打破
   → 建议: D_MAE 的评分应包含"有理由的偏离豁免"(justified deviation exemption)

3. **认知负荷的武器化**: 每个镜头同时传递叙事信息 + 倒叙规则 → 认知双重负荷。
   Nolan 故意违反 SEQ-LOAD-1(连续高负荷)——因为混乱IS主角的体验。
   → 我们的框架: cognitive-perception.md §三 的负荷规则正确识别了问题,
     但需要补充: "有意违反负荷规则以模拟角色的认知状态 = 有效策略"
   → 这在 CRAFT 维度应给加分而非在 LOGIC 维度扣分

### 对当前框架的贡献
  → [Gap] "逆向/非线性蒙太奇" 不在 montage-theory.md 的5级体系中
  → [Gap] D_MAE 需要 justified_deviation 豁免机制
  → [Insight] 认知负荷规则可以被有意违反——应在 evaluation.md 增加
    "有理由的规则打破在 CRAFT 维度加分"的明确条款
```
