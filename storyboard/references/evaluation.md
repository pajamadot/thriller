# 分镜质量评估系统

> 量化与定性相结合的分镜评估框架：6维度加权评分 + 导演解剖法
> 评估标准随类型适配（悬疑/动作/文艺/恐怖权重不同）

---

## 一、六维度评分体系

### 维度定义

| 维度 | 代码 | 评估内容 | 满分 |
|------|------|---------|------|
| 逻辑性 | LOGIC | 每个切的因果理由、信息DAG完整性、Idiom完整性 | 10 |
| 叙事性 | NARR | 节拍覆盖、弧线对齐、线索植入、视角一致性 | 10 |
| 视觉性 | VIS | 景别/角度/运动选择的叙事合理性、构图设计 | 10 |
| 节奏性 | RHYTHM | 时长分布、张力曲线、呼吸点、类型适配 | 10 |
| 声音性 | SOUND | 声画关系设计、沉默使用、声音桥接 | 10 |
| 创造性 | CRAFT | 超越模板的创新镜头、独特的视觉隐喻、导演风格感 | 10 |

**总分: 60分**

### 评级

| 分数 | 评级 | 含义 |
|------|------|------|
| 50-60 | 大师 | 可以直接作为拍摄参考 |
| 40-49 | 优秀 | 逻辑完整、有创意、少量可改进 |
| 30-39 | 良好 | 基本功扎实、但缺乏亮点或有逻辑瑕疵 |
| 20-29 | 及格 | 框架正确但多处逻辑问题或节奏失调 |
| <20 | 不合格 | 需要根本性重构 |

### 类型权重适配

| 维度 | 悬疑/惊悚 | 动作 | 文艺/剧情 | 恐怖 |
|------|----------|------|----------|------|
| LOGIC | **1.3** | 1.0 | 0.9 | 1.1 |
| NARR | **1.2** | 0.9 | **1.3** | 1.0 |
| VIS | 1.0 | 1.1 | **1.2** | 1.1 |
| RHYTHM | 1.0 | **1.3** | 0.8 | **1.2** |
| SOUND | **1.1** | 0.8 | 1.0 | **1.3** |
| CRAFT | 0.9 | 0.9 | **1.2** | 0.9 |

**加权总分 = Σ(维度分 × 类型权重)**

悬疑/惊悚强调：逻辑 > 叙事 > 声音（每个切必须有理由、线索植入精确、沉默是武器）
动作强调：节奏 > 视觉（快切节奏、运动连贯）
文艺强调：叙事 > 视觉 > 创造性（情感弧线、构图美学、独特视角）
恐怖强调：声音 > 节奏 > 逻辑（声音先行制造恐惧、节奏控制惊吓点）

## 二、逐维度评分细则

### LOGIC（逻辑性）0-10

> **交叉引用**: shot-logic.md v2.1 定义了 20 个形式化谓词 (P01-P20)。
> LOGIC 分数可通过谓词执行结果直接计算:
> `LOGIC = 10 × (1 - FAIL/total_predicates) × (1 - 0.1×WARN/total_predicates)`
> 其中 total_predicates = 实际执行的谓词数（P00-P20 中适用于当前序列的）。

```
10分: (FAIL=0, WARN≤1)
  - 每个切有明确的因果理由（信息/反应/强调/重定向, P00-P04）
  - 多通道信息DAG完整无缺口 (P05 全PASS)
  - 三通道汇聚点对齐叙事高潮 (P10 PASS)
  - 所有Idiom完整且正确使用 (P19 PASS)
  - 无任何10种逻辑错误（错误词典 §七）
  - 相邻镜头顺序不可互换 (P12 全PASS)

7-9分: (FAIL=0, WARN=2-4)
  - 95%以上的切有理由
  - DAG基本完整，1-2个小缺口或跨通道满足
  - Idiom基本完整
  - 1-2个轻微逻辑 WARN

4-6分: (FAIL=1-2, WARN=3-5)
  - 70-90%的切有理由
  - DAG有明显缺口
  - 有不完整的Idiom
  - 3-5个逻辑问题

1-3分: (FAIL≥3)
  - <70%的切有理由
  - DAG不连贯
  - 多处逻辑错误

0分:
  - 镜头序列是随机排列的（P00 几乎全 FAIL）
```

### NARR（叙事性）0-10

```
10分:
  - 每个节拍都有对应的镜头覆盖
  - 叙事弧线与视觉弧线完美对齐
  - 线索在画面中以Fair-play方式植入
  - 误导通过视觉引导精确实现
  - 视角选择一致且有叙事意图

7-9分:
  - 节拍覆盖完整
  - 弧线基本对齐
  - 线索植入存在但可以更精妙

4-6分:
  - 节拍有遗漏
  - 弧线偶尔脱节
  - 线索植入方式过于直白或缺失

1-3分:
  - 大量节拍未覆盖
  - 视觉与叙事脱节

0分:
  - 镜头与叙事无关
```

### VIS（视觉性）0-10

```
10分:
  - 景别选择始终服务于情感距离法则
  - 角度变化精确反映权力/心理关系
  - 摄影机运动与叙事能量方向一致
  - 构图利用深度分层、三分法、负空间
  - 每个镜头的视觉设计有叙事意图

7-9分:
  - 视觉选择大部分有叙事理由
  - 构图设计合理
  - 偶尔有景别选择不够精确

4-6分:
  - 视觉选择部分缺乏理由
  - 构图设计平淡
  - 多处景别选择可以更好

1-3分:
  - 视觉选择随意
  - 缺乏构图意识

0分:
  - 全部默认MS + EYE + STATIC
```

### RHYTHM（节奏性）0-10

> 数学化评分体系（参见 visual-rhythm.md §八-§十七）

```
量化指标（必须计算）:
  σ = 时长标准差 (秒)
  ASL = 平均镜头时长 (秒)
  r(1) = lag-1自相关系数
  α = 1/f指数 (N>15时)
  phrase_regularity = 短语长度的变异系数 (CV)
  climax_alignment = 最高重力镜头是否对应BPM极值 (bool)
  D_MAE = Duration Function计算值与最终取值的MAE (秒)

10分:
  - σ ∈ [1.5, 4.0]s (快闪蒙太奇豁免)
  - ASL 在类型目标区间内 (悬疑: 3.5-6.0s)
  - r(1) ∈ [0.3, 0.7] (接近1/f, N>15时)
  - α ∈ [0.8, 1.2] (N>15时)
  - phrase_regularity: 所有短语3-5拍, CV < 0.3
  - climax_alignment = true
  - D_MAE < 1.0s
  - 节奏模式可用Composition Algebra表达
  - Time Signature标注正确且与场景类型匹配
  - Tempo Map显示有方向性的BPM曲线
  - 至少一个"建立-打破"策略（Surprise > 1.0）

7-9分:
  - σ ∈ [1.0, 5.0]s (略超标准区间)
  - ASL 在类型目标±1s内
  - r(1) ∈ [0.15, 0.8]
  - phrase_regularity: 大部分短语3-7拍
  - climax_alignment = true
  - D_MAE < 1.5s
  - Tempo Map有方向性但不完全平滑
  - 1-2处节奏不够精确

4-6分:
  - σ < 1.0 (偏均匀) 或 σ > 5.0 (偏混乱)
  - ASL 超出类型目标区间
  - r(1) < 0.1 (过于随机) 或 r(1) > 0.85 (过于相关)
  - 高潮处缺乏节奏支撑 (climax_alignment = false)
  - 无明确的短语结构
  - 无Time Signature标注

1-3分:
  - σ < 0.5 (全程近似等时长)
  - r(1) ≈ 0 (白噪声——完全随机切换)
  - 无节奏模式可识别
  - 有明显节奏病症（见 visual-rhythm.md §七）

0分:
  - 所有镜头相同时长 (σ = 0)
  - 或全部时长随机生成，无任何叙事逻辑
```

### SOUND（声音性）0-10

```
10分:
  - 每个镜头的声音设计都有标注
  - 声画关系在同步/对位/先行间有意识选择
  - 沉默被作为叙事工具使用
  - J-Cut/L-Cut在场景过渡中被恰当使用
  - 声音设计增强而非仅仅伴随画面

7-9分:
  - 声音标注完整
  - 声画关系大部分合理
  - 有沉默的使用

4-6分:
  - 声音标注不完整
  - 声画关系全部是同步（无对位/先行）
  - 缺少沉默

1-3分:
  - 声音标注缺失
  - 未考虑声画关系

0分:
  - 无声音设计
```

### CRAFT（创造性）0-10

```
10分:
  - 有至少2个超越模板的创新镜头设计
  - 有独特的视觉隐喻（空间=叙事、镜子=分裂等）
  - 有统一的导演风格感
  - 有意识地打破规则来达到效果（并能解释为什么）
  - 有至少1个参考经典电影但做了自己的变体

7-9分:
  - 有1-2个创新镜头
  - 有视觉隐喻但不够深入
  - 风格基本一致

4-6分:
  - 基本按模板执行
  - 缺少创新和隐喻
  - 风格不明确

1-3分:
  - 完全模板化
  - 无个人风格

0分:
  - 机械的景别堆叠
```

### 认知保真度检查（Cognitive Fidelity Sub-checks）

> 来源: `cognitive-perception.md` — Band 3 认知与知觉深化产物

以下认知保真度检查项嵌入 LOGIC 和 VIS 维度的评分中，作为加分/扣分项:

```
嵌入 LOGIC 维度:
  +1 如果每个镜头的信息单元数在景别限制内 (cognitive-perception.md 三)
  +1 如果所有相邻镜头之间有至少一个桥接线索 (cognitive-perception.md 九)
  -1 如果存在连续 3+ 个高负荷镜头无呼吸点 (SEQ-LOAD-1)
  -1 如果对比切前启动时长不足 (<1.5s) (PRIME-2/3)

嵌入 VIS 维度:
  +1 如果注意力引导与叙事意图对齐 (P1-P5 优先级正确使用)
  +1 如果 Fair-play 线索在低注意力优先级位置 (DESYNC 规则)
  -1 如果关键信息在低同步条件下呈现 (DESYNC-1/2)

嵌入 RHYTHM 维度:
  +1 如果镜头时长满足最低呈现时长公式
  +0.5 如果时长序列近似 1/f 模式 (RHYTHM-1F)
  -1 如果存在低于认知处理阈值的时长 (<0.3s 非故意闪现)

认知保真度总分: 上述加减项的净值
  纳入方式: 分散到 LOGIC/VIS/RHYTHM 三个维度的评分中
  最大影响: +/- 2 分 (per dimension)
```

## 三、`/score` 命令

```
/score [分镜序列]

输入: 已完成的分镜序列（表格或JSON）
输出:
  1. 六维度评分（含细节说明）
  2. 加权总分（按类型权重）
  3. 强项/弱项诊断
  4. 改进建议（最多3条，按优先级）
  5. 可计算指标对比（6项）

可选参数:
  --genre <thriller|action|drama|horror>  类型（影响权重）
  --compare <另一个分镜序列>  对比模式
```

### 评分报告格式

```markdown
## 分镜评分报告: [场景名]

### 总分: 42/60 (优秀) [加权: 45.6/60 - 悬疑权重]

| 维度 | 分数 | 权重 | 加权 | 评语 |
|------|------|------|------|------|
| LOGIC | 8 | 1.3 | 10.4 | DAG完整，1处Idiom不完整 |
| NARR | 7 | 1.2 | 8.4 | 节拍覆盖好，线索植入可更精妙 |
| VIS | 7 | 1.0 | 7.0 | 景别选择合理，构图可加深度 |
| RHYTHM | 6 | 1.0 | 6.0 | 方差偏低(σ=1.3)，需更多变化 |
| SOUND | 7 | 1.1 | 7.7 | 有沉默设计，L-Cut未使用 |
| CRAFT | 7 | 0.9 | 6.3 | 帽衫揭示镜头设计出色 |

### 可计算指标
| 指标 | 值 | 目标 | 状态 |
|------|-----|------|------|
| 信息效率 | 0.85 | >0.8 | PASS |
| 切理由覆盖 | 0.92 | =1.0 | WARN |
| 依赖完整 | 1.0 | =1.0 | PASS |
| 节奏方差σ | 1.3s | 1.5-4.0 | FAIL |
| ASL | 4.2s | 3.5-6.0 (悬疑) | PASS |
| r(1) 自相关 | 0.22 | 0.3-0.7 | WARN |
| 1/f指数α | — | 0.8-1.2 (N<15不适用) | N/A |
| D_MAE | 0.35s | <1.0 | PASS |
| 短语CV | 0.15 | <0.3 | PASS |
| 高潮对齐 | true | true | PASS |
| Idiom覆盖 | 0.75 | 0.6-0.9 | PASS |
| 景别熵 | 2.4 bits | >2.0 | PASS |

### 改进建议（优先级排序）
1. **节奏方差**: σ=1.3低于1.5阈值。在Beat 8处加入一个8-10s的长镜头呼吸点，预计σ→2.1
2. **自相关偏低**: r(1)=0.22偏向白噪声。检查是否存在刻意的长-短交替(Breathe模式)——若是，可接受
3. **声音桥接**: Beat 9→Beat 10的场景过渡适合使用L-Cut
```

## 四、`/autopsy-visual` 命令 — 导演解剖法

从已完成的电影场景中逆向工程镜头选择的逻辑。

```
/autopsy-visual <电影名> <场景描述>

输入:
  1. 电影名 + 具体场景（如 "《七宗罪》开场调查场景"）
  2. 或 提供场景的镜头描述列表

输出:
  1. 镜头逻辑逆向分析（每个切的理由推断）
  2. 导演的视觉策略提取
  3. 可复用的技巧模式
  4. 与当前方法论的对比（"我们的框架能解释吗？"）
  5. 方法论盲点标注（"这个技巧我们还没有"）
```

### 解剖协议（5步）

```
Step 1: 镜头清单重建
  列出场景中的每个镜头:
    编号、景别、角度、运动、时长（估计）、内容描述

Step 2: 切理由逆推
  对每个切，推断导演的理由:
    为什么从镜头A切到镜头B？
    是信息/反应/强调中的哪一个？
    有没有超出三种理由的第四种？

Step 3: 信息DAG逆推
  为每个镜头标注 REQUIRES / PROVIDES / RAISES
  画出信息流图
  检查: 这个导演的信息管理策略是什么？

Step 4: 模式识别
  识别导演使用的Idiom:
    标准Idiom（建立、正反打、揭示、追逐）？
    变体Idiom（标准模式的创新变化）？
    全新Idiom（我们框架中没有的模式）？

Step 5: 方法论对比
  我们的框架能完全解释这个场景吗？
  如果不能:
    哪个维度/规则无法覆盖？
    这是一个[Experiment]级别的新发现？
    应该加入哪个参考文件？
```

### 解剖报告格式

```markdown
## 视觉解剖: 《七宗罪》开场犯罪现场调查

### 场景概要
[场景描述]

### 镜头清单
| # | 景别 | 角度 | 运动 | 时长 | 内容 | 切理由 |
|---|------|------|------|------|------|--------|
| 1 | ELS | BIRD | STATIC | 5s | 犯罪现场鸟瞰 | — (建立) |
| 2 | MS | EYE | TRACK | 4s | Somerset走过走廊 | 信息:角色进入 |
| ... | | | | | | |

### 信息DAG
[图示]

### 导演策略提取
1. **Fincher的空间压迫**: 用低饱和色调+紧凑框架制造...
2. **信息过载策略**: 每个镜头塞入3+视觉线索...
3. ...

### 可复用模式
- [新Idiom] "信息过载揭示": LS(充满细节)→INSERT×5(快切)→CU(反应)
  → 与标准揭示序列不同: 不是逐步聚焦，而是信息轰炸后聚焦

### 方法论盲点
- [Gap] 我们的框架没有"信息密度"概念（每个镜头的视觉信息量）
  → 建议: 在shot-logic.md增加 INFO_DENSITY 属性

### 对当前框架的贡献
  → 新增1个Idiom到shot-logic.md?
  → 新增1个评估维度到evaluation.md?
  → 记录到 meta/autopsy-visual-*.md
```

## 五、跨项目评估对比

当完成多个分镜项目后，可以进行跨项目对比:

```
/score --compare-projects

输出:
  各项目的六维度雷达图叠加
  维度趋势（是否在进步？哪个维度停滞？）
  方法论进化信号（哪些改进最有效？）
  → 触发 gene_sb_evaluation_scoring 的进化信号
```

## 六、与 /retro 和 /evolve 的集成

```
分镜项目完成后:
  /score → 产生评分报告
  /retro → 分析过程偏差
    评分中哪些维度低？为什么？
    是方法论问题还是执行问题？

两个项目后:
  /evolve → 跨项目模式识别
    共同的弱点维度？
    → 触发对应的GEP Gene
    → 进化对应的reference文件

/autopsy-visual 的发现:
  方法论盲点 → /benchmark 整合
  新Idiom → 加入shot-logic.md
  新评估维度 → 加入evaluation.md
```
