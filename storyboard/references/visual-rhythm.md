## 一、节奏的本质

> "剪辑就是呼吸。" —— Walter Murch

视觉节奏是观众的**情感呼吸**。镜头时长控制观众的心理节奏，正如音乐的拍号控制听众的身体节奏。

```
核心公式:

镜头时长 ∝ 信息密度 × 情感重量 / 叙事紧迫度

  信息密度高（复杂场景）→ 需要更长时间消化
  情感重量大（关键时刻）→ 需要更长停留
  叙事紧迫度高（追逐/倒计时）→ 需要更短时长

三者的平衡决定每个镜头的"呼吸空间"。
```

## 二、镜头时长参考体系

### 基础时长参照

| 景别 | 基础时长 | 最短 | 最长 | 说明 |
|------|---------|------|------|------|
| ELS | 4-8s | 2s | 15s | 需要时间让观众"扫描"环境 |
| LS | 3-6s | 2s | 12s | 需要看到完整的身体行动 |
| FS | 3-5s | 2s | 10s | 看完整的肢体语言 |
| MS | 2-4s | 1s | 8s | 对话的标准节奏 |
| MCU | 2-4s | 1s | 8s | 面部+上身 |
| CU | 2-5s | 0.5s | 15s | 情感停留——可以很长 |
| ECU | 1-3s | 0.3s | 8s | 冲击力来自短暂或异常长 |
| INSERT | 1-3s | 0.5s | 5s | 足够识别信息 |

### 时长与情感的关系

```
极短（<1s）:
  效果: 闪现、冲击、潜意识植入
  用途: 恐怖闪回、爆炸碎片、记忆碎片
  注意: 连续使用>5个会导致观众疲劳

短（1-2s）:
  效果: 紧迫、快速信息传递
  用途: 追逐、战斗、紧张对话
  注意: 需要简单构图，复杂画面来不及消化

中（3-5s）:
  效果: 正常叙事节奏、舒适的信息消化
  用途: 大部分叙事场景的工作时长
  注意: 过多中等时长 → 节奏单调

长（6-10s）:
  效果: 停留、沉浸、情感积累
  用途: 关键反应、氛围建立、长对白
  注意: 需要画面内部有足够的视觉兴趣

极长（>10s / 长镜头）:
  效果: 真实感、不可逃避、张力累积
  用途: 审讯、跟踪、重要对话、空间展示
  注意: 需要摄影机运动或画面内运动来维持兴趣
```

## 三、节奏模式库

### 模式 1：脉搏型（Pulse）

```
时长: ■■ ■■ ■■ ■■ ■■ ■■
      2s 2s 2s 2s 2s 2s

特征: 均匀节拍，像心跳
效果: 稳定的紧张、机械感、仪式感
适用: 追逐、搜索、倒计时
悬疑场景: 角色逐个房间搜索，每个房间一个镜头
```

### 模式 2：加速型（Accelerando）

```
时长: ■■■■■ ■■■■ ■■■ ■■ ■ ■ ■
      5s    4s  3s  2s 1s 1s 1s

特征: 时长递减，节奏加速
效果: 紧迫感递增，不可阻挡
适用: 高潮前的建设、追逐的加剧
悬疑场景: 真相逐步显露，每个线索的呈现越来越快
```

### 模式 3：减速型（Ritardando）

```
时长: ■ ■ ■■ ■■■ ■■■■ ■■■■■■■
      1s 1s 2s 3s  4s    7s

特征: 时长递增，节奏减速
效果: 从激烈到沉静，从外在到内在
适用: 高潮后的余波、顿悟时刻
悬疑场景: 真相揭示后，角色消化信息的过程
```

### 模式 4：呼吸型（Breathe）

```
时长: ■■■ ■ ■■■■ ■ ■■■ ■ ■■■■■
      3s  1s 4s   1s 3s  1s 5s

特征: 长-短交替，像呼吸的吸-呼
效果: 自然节奏、松紧有度
适用: 大部分叙事场景的默认模式
悬疑场景: 审讯——问题(短) + 回答(长) + 追问(短) + 犹豫(长)
```

### 模式 5：爆发型（Burst）

```
时长: ■■■■■ ■■■■■ ■■■■■ ■ ■ ■ ■ ■ ■■■■■■■■■■
      5s    5s    5s    1 1 1 1 1  10s

特征: 长段稳定 → 突然快切爆发 → 延长停留
效果: 积蓄-释放-沉淀
适用: 高潮时刻、情感爆发
悬疑场景: 平静对话 → 突然发现真相（快切回溯） → 角色凝固在真相中
```

### 模式 6：停顿型（Fermata）

```
时长: ■■ ■■ ■■ ■■■■■■■■■■■■■■■ ■■ ■■
      2s 2s 2s      15s         2s 2s

特征: 正常节奏中的异常长镜头
效果: 时间暂停感、强调、不可逃避
适用: 关键揭示、重要决定、情感高峰
悬疑场景: 角色看到监控录像——15秒的静止特写，角色的世界在崩塌
```

## 四、张力弧线设计

### 张力的构成要素

```
张力 = 威胁 × 关切 × 不确定性 / 控制感

威胁(Threat): 坏事可能发生
关切(Care): 观众在意角色
不确定性(Uncertainty): 不知道结果如何
控制感(Agency): 角色能否影响结果

镜头语言如何操控这四个变量:
  威胁 ↑ : 低角度拍摄威胁源 / 推近 / 暗调
  关切 ↑ : 特写表情 / 延长反应时长
  不确定 ↑ : 不完整的画面 / 负空间 / 声音先行
  控制感 ↓ : 手持晃动 / 荷兰角 / POV受限
```

### 场景级张力弧线模板

#### 模板A：渐进式（Escalation）

```
张力: ╱──────────────╱────────╱──╲
      低→中→中高→高→高峰→释放

镜头策略:
  开始: LS/MS + 稳定 + 正常色调
  中段: MCU/CU + 运动增加 + 色调变化
  高峰: ECU + 手持/推近 + 极端色调
  释放: LS/ELS + 拉远 + 色调回归
```

#### 模板B：伏击式（Ambush）

```
张力: ──────────────────────╱╲──
      低→低→低→低→突然高峰→回落

镜头策略:
  长段平静: LS/MS + 稳定 + 暖色调
  伏击瞬间: 硬切到ECU/CU + 惊吓SFX
  回落: 手持抖动 → 稳定器
```

#### 模板C：锯齿式（Sawtooth）

```
张力: ╱╲╱╲╱╲╱──╲
      起伏→起伏→起伏→更高起伏→释放

镜头策略:
  每个峰: CU + 推近 + SFX
  每个谷: MS + 稳定 + 对白
  整体趋势: 峰值越来越高，谷值也在抬升
```

#### 模板D：压力锅式（Pressure Cooker）

```
张力: ──╱────────────────────╲──
      低→升→持续高压→持续→持续→释放

镜头策略:
  长段高压: 长镜头 + 静止 + 特写
  不给观众释放的机会 → 张力来自"没有切出去"
  最终释放: 切到ELS或完全不同的场景
```

## 五、景别节奏分析

### 景别序列的情感效果

```
递进式（由远到近）:
  ELS → LS → MS → MCU → CU → ECU
  效果: 逐步接近、深入、聚焦
  情感: 从客观观察到情感投入
  适用: 进入场景、认识角色、接近真相

递退式（由近到远）:
  ECU → CU → MS → LS → ELS
  效果: 逐步远离、释放、超脱
  情感: 从沉浸到抽离
  适用: 离开场景、结束、释放

跳跃式:
  MS → ECU → LS → CU → ELS → INSERT
  效果: 不安定、碎片化、多角度
  情感: 焦虑、混乱、主观体验
  适用: 恐慌、精神崩溃、时间碎片

锁定式:
  CU → CU → CU → CU → CU (同一景别)
  效果: 困住、无法逃脱、窒息
  情感: 压迫、关注、不放过
  适用: 审讯、独白、内心戏
```

## 六、节奏与类型的关系

### 悬疑/惊悚的节奏特征

```
整体特征:
  - 平均镜头时长 3-6 秒（比动作片长，比文艺片短）
  - 静止镜头占比 40-60%（比动作片高）
  - 特写镜头占比 30-40%（面部表演是核心）
  - 长镜头常出现在关键场景（审讯、跟踪、揭示）

节奏禁忌:
  ✗ 全程快切（悬疑需要呼吸和停留）
  ✗ 全程慢节奏（需要突然加速的能力）
  ✗ 均匀节拍（可预测的节奏=没有惊喜）
  ✗ 高潮前减速（应该加速到高潮点）
```

### 类型节奏对比

| 类型 | 平均镜头时长 | 静止占比 | 特写占比 | 长镜头使用 |
|------|------------|---------|---------|----------|
| 动作片 | 1.5-3s | 15% | 20% | 罕见 |
| 恐怖片 | 3-5s | 50% | 35% | 偶尔（制造不安） |
| 悬疑片 | 3-6s | 45% | 35% | 常见（审讯/跟踪） |
| 文艺片 | 5-15s | 60% | 25% | 频繁 |
| 喜剧片 | 2-4s | 30% | 30% | 罕见 |

## 七、节奏诊断工具

### 节奏评分卡

对一组分镜序列进行以下检查：

```
□ 变化度 (Variance)
  镜头时长的标准差是否 > 1.5s？
  太低 → 节奏单调（所有镜头差不多长）
  太高 → 节奏混乱（无模式可循）

□ 弧线性 (Arc)
  时长变化是否形成有意义的弧线？
  随机波动 → 缺乏叙事逻辑
  有方向性（加速/减速/爆发）→ 有节奏意图

□ 呼吸点 (Breathing)
  每 5-8 个短镜头后是否有一个长镜头？
  没有呼吸 → 观众疲劳
  呼吸太多 → 缺乏紧迫感

□ 高潮对齐 (Climax Alignment)
  叙事高潮是否对应节奏的最快点或最慢点？
  不对齐 → 情感和形式脱节

□ 景别配合 (Size Harmony)
  长镜头配大景别？短镜头配小景别？
  长+ECU → 有效（情感停留）
  短+ELS → 无效（看不清）
```

### 节奏修复处方

| 症状 | 处方 |
|------|------|
| 节奏太快，无法消化 | 在快切序列后加一个6-10s的静止CU |
| 节奏太慢，观众走神 | 用INSERT或ECU快切打断长镜头 |
| 节奏无起伏 | 找到场景的情感最高点，在那里制造节奏突变 |
| 高潮来得太突然 | 在高潮前加3-5个递减时长的镜头（加速铺垫） |
| 高潮后无余韵 | 最后一个镜头延长3-5秒，让观众消化 |
| 对话场景无聊 | 在"关键台词"时切到异常景别或角度 |

## 八、Duration Function（时长函数）

> Band 4 (r031-r040) 引入的数学形式化。将直觉性的时长表格提升为可计算的函数。
>
> 此函数被 decomposition-algorithm.md Step 6 直接调用。完整计算示例见 examples/mirror-visitor-ch1-storyboard.md §Step 6.

### 核心公式

```
D(shot) = D_base(size) * G(gravity) * B(beat_type) * S(style_modifier)

单位: 秒
约束: D(shot) ∈ [0.3, 30.0] （硬限——低于0.3s人眼无法解析，超过30s需要特殊理由）
```

### D_base：景别基准时长

基于观众消化该景别视觉信息所需的最小认知时间（Cutting et al., 2010 数据校准）。

```
D_base(ELS)    = 5.0s   # 大远景：需要扫描环境、建立空间感
D_base(LS)     = 4.0s   # 远景：看清完整人体和动作
D_base(FS)     = 3.5s   # 全景：身体语言阅读
D_base(MS)     = 3.0s   # 中景：对话的标准框架
D_base(MLS)    = 3.2s   # 中远景：介于MS和FS之间
D_base(MCU)    = 2.5s   # 中近景：面部+手势
D_base(CU)     = 3.0s   # 特写：情感阅读（注意：比MCU长，因为情感停留）
D_base(ECU)    = 2.0s   # 极特写：冲击力来自短暂或异常长
D_base(INSERT) = 2.0s   # 插入：物件/细节识别
D_base(POV)    = 2.5s   # 主观镜头：等同于角色扫视时间
```

### G：重力修正因子

叙事重力（gravity）反映节拍的重要程度。重要时刻需要更长的停留，过渡时刻快速通过。

```
G(1) = 0.70   # 过渡/连接——快速通过
G(2) = 0.90   # 辅助建设——略低于基准
G(3) = 1.00   # 重要推进——基准时长（锚点）
G(4) = 1.30   # 场景转折——停留更久，让观众消化
G(5) = 1.80   # 故事转折——最长停留，这是改变一切的时刻
```

**设计原理**: G(3)=1.0 作为锚点。G(5)/G(1) = 2.57, 意味着故事转折镜头的时长是过渡镜头的约2.6倍——这与Murch的剪辑实践一致（关键反应镜头通常是过渡镜头的2-3倍长）。

### B：节拍类型修正因子

不同的叙事功能需要不同的时间分配。

```
B(action)     = 0.80   # 动作：紧凑执行，观众看"做了什么"
B(reaction)   = 1.20   # 反应：需要时间阅读面部，理解内心
B(reveal)     = 1.50   # 揭示：信息炸弹，观众需要消化
B(turn)       = 1.00   # 转折：与基准同步（转折的力量在内容不在时长）
B(tension)    = 1.80   # 张力保持：Fermata效果，越长越不安
B(release)    = 1.30   # 释放：余韵，情感着陆
B(transition) = 0.60   # 过渡：快速桥接，不让观众脱离
B(dialogue)   = 1.00   # 对话：与台词时长联动（见台词修正）
B(setup)      = 0.90   # 铺垫：比基准略快，保持前进感
B(payoff)     = 1.40   # 回收：铺垫的兑现需要停留
```

### S：风格修正因子

不同导演风格对节奏有系统性偏好。S因子对整个场景统一生效。

```
S(neutral)    = 1.00   # 无风格偏好（默认）
S(hitchcock)  = 1.20   # Hitchcock: 更长停留，静止中积累不安
S(fincher)    = 0.85   # Fincher: 精确快切，信息密度高
S(wong)       = 1.50   # Wong Kar-Wai: 时间拉长，感受浸泡
S(nolan)      = 0.95   # Nolan: 略快于基准，叙事驱动
S(kubrick)    = 1.35   # Kubrick: 冷静凝视，不舒适的长镜头
S(park)       = 1.10   # Park Chan-wook: 精致但微快，张力内蕴
S(lynch)      = 1.40   # Lynch: 时间感异常，日常变诡异
S(thriller)   = 1.00   # 悬疑/惊悚通用（默认等于neutral）
```

### 台词长度修正（附加因子）

当镜头包含台词时，时长受台词朗读时间约束：

```
D_speech(shot) = max(D(shot), T_read(dialogue) + 0.5)

其中:
  T_read = 字数 / 语速
  中文语速 ≈ 3.5字/秒（正常）, 2.5字/秒（沉重/犹豫）, 5字/秒（急促）
  英文语速 ≈ 2.5词/秒（正常）, 1.8词/秒（沉重）, 4词/秒（急促）
  +0.5s 是台词结束后的反应缓冲
```

### 计算示例

```
Shot: CU of 叶知秋, gravity=4 (场景转折), beat_type=reaction, style=hitchcock

D = D_base(CU) * G(4) * B(reaction) * S(hitchcock)
  = 3.0 * 1.30 * 1.20 * 1.20
  = 5.62s

直觉估计: 5-6s ✓ （在±1s范围内）
```

```
Shot: ELS establishing, gravity=1 (过渡), beat_type=transition, style=fincher

D = D_base(ELS) * G(1) * B(transition) * S(fincher)
  = 5.0 * 0.70 * 0.60 * 0.85
  = 1.79s

直觉估计: 2s ✓ （Fincher的建立镜头极短——Se7en的建立镜头平均1.5-2s）
```

## 九、Time Signature Notation（时间记号系统）

> 借用音乐的拍号概念，为场景类型定义结构化的时间框架。
>
> 各类型的默认时间记号见 genre-libraries.md 各库的 rhythm_override 字段。

### 核心概念

```
时间记号 = X/Y [修饰语]

X = 每小节的拍数（一个"phrase"中的镜头数量）
Y = 基本拍的单位值（4=正常，8=密集/复合拍）
修饰语 = 速度/弹性指示

一个"小节"(bar) = 一个叙事短语 (phrase)
一个"拍"(beat) = 一个镜头
```

### 场景类型的时间记号

```
┌──────────────┬────────┬──────────────────────────────────────────┐
│ 场景类型       │ 时间记号  │ 说明                                      │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 审讯           │ 4/4    │ 正拍强调：问(1) 答(2) 追问(3) 反应(4)     │
│ (Interrogation)│ synco  │ 切分音——强拍落在"反应"而非"台词"上       │
│               │        │ 重音在1和3：问题和追问是结构骨架          │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 追逐           │ 6/8    │ 复合拍：每phrase 6个镜头分为2组×3        │
│ (Chase)        │ accel  │ 加速：tempo从每拍2s递减至0.8s             │
│               │        │ 重音在1和4：两个追逐主体交替             │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 发现           │ 3/4    │ 华尔兹式：铺垫(1) 铺垫(2) 揭示(3)       │
│ (Discovery)    │ rubato │ 弹性速度：铺垫快，揭示慢                 │
│               │        │ 重音在3：第三拍是信息炸弹                │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 对话           │ 4/4    │ 正拍：A说(1) B反应(2) B说(3) A反应(4)    │
│ (Dialogue)     │ rubato │ 弹性速度：语速和情感决定弹性              │
│               │        │ 无固定重音：重音跟随叙事重力              │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 蒙太奇         │ var    │ 变拍号：拍号本身就是内容                  │
│ (Montage)      │ metric │ 度量式：精确的时长递减/递增构成意义       │
│               │        │ 从 4/4 → 3/4 → 2/4 = 收紧               │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 冥想/内心       │ free   │ 自由拍：无固定拍号                       │
│ (Meditation)   │        │ 镜头时长由内容密度和情感重量决定          │
│               │        │ 类似音乐的华彩段(cadenza)                │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 跟踪/尾随      │ 5/4    │ 不对称拍：制造持续的不安感               │
│ (Surveillance) │ ostina │ 固执低音：同一构图反复出现(跟踪者视角)    │
│               │        │ 观众始终无法"落地"在舒适的节拍上         │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 倒计时         │ 2/4    │ 进行曲式：紧凑、压迫、无弹性             │
│ (Countdown)    │ strict │ 严格节拍：每拍等长，机械的精确感          │
│               │        │ 越接近零点，SCALE因子越小                │
├──────────────┼────────┼──────────────────────────────────────────┤
│ 梦境/闪回      │ 3/4    │ 华尔兹式：但tempo极慢或极快              │
│ (Dream)        │ rubato │ 时间被拉伸或压缩——非正常的时间流逝感     │
│               │        │ 镜头时长方差极大（0.5s到10s混合）         │
└──────────────┴────────┴──────────────────────────────────────────┘
```

### 时间记号的标注语法

```
场景标头:
  SCENE: 咨询室对话
  TIME_SIG: 4/4 rubato
  TEMPO: ♩=15 BPM → ♩=20 BPM (accel)
  KEY: Breathe → Accelerando → Fermata

分镜序列中的小节线:
  | Shot1 Shot2 Shot3 Shot4 | Shot5 Shot6 Shot7 Shot8 | ...
  |______ bar 1 ____________|_______ bar 2 ___________|

  其中每个 | 是phrase边界（见下文"十一、Phrase Model"）
```

### BPM与镜头时长的换算

```
BPM (Beats Per Minute) = 60 / D_avg(shot)

其中 D_avg 是该段的平均镜头时长（秒）

换算表:
  BPM 10  → 平均 6.0s/镜头 (极慢——冥想/审讯高压段)
  BPM 15  → 平均 4.0s/镜头 (正常偏慢——悬疑对话)
  BPM 20  → 平均 3.0s/镜头 (标准叙事节奏)
  BPM 30  → 平均 2.0s/镜头 (紧迫)
  BPM 40  → 平均 1.5s/镜头 (快切)
  BPM 60  → 平均 1.0s/镜头 (极快——追逐高潮/恐怖快闪)
  BPM 120 → 平均 0.5s/镜头 (闪切蒙太奇极限)
```

## 十、Rhythm Composition Algebra（节奏组合代数）

> 将六种基础节奏模式视为代数元素，定义组合运算符，使复杂场景的节奏可以被精确描述和计算。

### 基础元素

```
P = Pulse(d, n)         # 脉搏型: d=每拍时长, n=拍数
A = Accelerando(d0, r, n)  # 加速型: d0=起始时长, r=衰减率, n=拍数
R = Ritardando(d0, r, n)   # 减速型: d0=起始时长, r=增长率, n=拍数
Br = Breathe(d_long, d_short, n)  # 呼吸型: 长短交替
Bu = Burst(d_calm, d_fast, d_hold, n_calm, n_fast)  # 爆发型
F = Fermata(d_normal, d_held, pos)  # 停顿型: pos=长镜头的位置

每个元素产生一个时长序列:
  P(2, 6) → [2, 2, 2, 2, 2, 2]
  A(5, 0.7, 6) → [5, 3.5, 2.45, 1.72, 1.20, 0.84]
  R(1, 1.5, 5) → [1, 1.5, 2.25, 3.38, 5.06]
  Br(4, 1.5, 6) → [4, 1.5, 4, 1.5, 4, 1.5]
  Bu(5, 1, 10, 3, 5) → [5, 5, 5, 1, 1, 1, 1, 1, 10]
  F(2, 15, 4) → [2, 2, 2, 15, 2, 2]
```

### 组合运算符

#### CONCAT(A, B) — 序列连接

```
定义: 将B的时长序列接在A的末尾
语义: 先执行A的节奏，然后执行B的节奏
结果: len(CONCAT) = len(A) + len(B)

CONCAT(P(2,3), A(5,0.7,3))
  = [2, 2, 2] ++ [5, 3.5, 2.45]
  = [2, 2, 2, 5, 3.5, 2.45]

连接律: CONCAT(CONCAT(A,B), C) = CONCAT(A, CONCAT(B,C))
```

#### NEST(outer, inner) — 嵌套

```
定义: 在outer的每个拍内部，用inner的微结构替代单一时长
语义: 宏观节奏由outer控制，微观脉动由inner提供
约束: inner的总时长被缩放到outer的每拍时长

NEST(Br(6, 2, 4), P(1, 3))
  outer = [6, 2, 6, 2]
  inner_for_6s = P(2, 3) = [2, 2, 2]  # 总和=6 ✓
  inner_for_2s = P(0.67, 3) = [0.67, 0.67, 0.67]  # 总和≈2 ✓
  result = [2, 2, 2, 0.67, 0.67, 0.67, 2, 2, 2, 0.67, 0.67, 0.67]

应用: 审讯场景——宏观上Breathe(问答交替)，每个"问"内部是Pulse(紧逼感)
```

#### INVERT(A) — 反转

```
定义: 将A的时长序列逆序排列
语义: 将加速变为减速，减速变为加速
恒等式: INVERT(INVERT(A)) = A
关系: INVERT(Accelerando) ≈ Ritardando

INVERT(A(5, 0.7, 5))
  = INVERT([5, 3.5, 2.45, 1.72, 1.20])
  = [1.20, 1.72, 2.45, 3.5, 5]
```

#### SCALE(A, factor) — 缩放

```
定义: 将A的每个时长乘以factor
语义: 整体加速(factor<1)或减速(factor>1)，保持内部比例
约束: factor > 0

SCALE(P(2, 4), 1.5)
  = [3, 3, 3, 3]

SCALE(A(5, 0.7, 4), 0.6)
  = [3, 2.1, 1.47, 1.03]

分配律: SCALE(CONCAT(A,B), f) = CONCAT(SCALE(A,f), SCALE(B,f))
```

#### ACCENT(A, positions, factor) — 重音

```
定义: 在指定位置的拍上乘以accent_factor
语义: 强调某些拍（延长），弱化其他（保持或缩短）
positions: 从0开始的索引列表
factor: 默认1.5（延长50%）

ACCENT(P(2, 6), [0, 3], 1.5)
  = [3, 2, 2, 3, 2, 2]

应用: 4/4拍的重音结构 → ACCENT(P(d, 4), [0, 2]) 强调第1和第3拍
```

#### INTERLEAVE(A, B) — 交织

```
定义: 交替取A和B的元素
语义: 两个节奏模式交替出现（如追逐中追与逃的交替）
约束: len(A) = len(B)（否则短序列末尾补B的最后一个值）

INTERLEAVE(P(2, 3), P(4, 3))
  = [2, 4, 2, 4, 2, 4]

应用: 追逐场景——追逐者(快切P(1.5,n)) 与 逃跑者(略长P(2.5,n)) 交替
```

### 组合示例：完整场景节奏表达

```
审讯场景:
  interrogation = CONCAT(
    SCALE(Br(4, 2, 6), 1.0),              # 开场：正常呼吸节奏的问答
    NEST(A(6, 0.75, 4), P(1, 3)),          # 中段：加速逼问，内部有脉搏
    ACCENT(P(2, 4), [0, 2], 1.3),          # 快速连续追问，奇数拍加重
    F(2, 12, 1)                            # 终局：Fermata——关键证据展示
  )

  总时长序列: [4, 2, 4, 2, 4, 2,           # Breathe x6 (18s)
              2, 2, 2, 1.5, 1.5, 1.5,      # Nested Accelerando bar1-2 (9s)
              1.13, 1.13, 1.13, 0.84, 0.84, 0.84,  # bar3-4 (5.9s)
              2.6, 2, 2.6, 2,              # Accented Pulse (9.2s)
              2, 12, 2]                     # Fermata (16s)
  总计: ~58s ≈ 约1分钟的审讯段落
```

```
追逐场景:
  chase = CONCAT(
    SCALE(P(3, 4), 1.0),                   # 意识到被追：正常脉搏
    INTERLEAVE(A(2, 0.8, 6), A(2.5, 0.8, 6)), # 追与逃交替加速
    Bu(0, 0.5, 5, 0, 8)                    # 爆发：极快切 → 长镜头结果
  )
```

## 十一、Phrase Model（短语结构模型）

> 镜头序列不是无差别的流，而是分层的——镜头构成短语，短语构成段落，段落构成场景。

### 层级结构

```
场景 (Scene)
  └── 段落 (Section): 3-5个短语，对应场景的"起承转合"
       └── 短语 (Phrase): 3-5个镜头，一个完整的叙事单元
            └── 镜头 (Shot): 最小单位

类比音乐:
  乐章 (Movement) → 场景
  段 (Period)     → 段落
  乐句 (Phrase)   → 短语
  音符 (Note)     → 镜头
```

### 短语边界规则

```
短语边界应出现在以下位置:

1. 话题转换: 对话从A话题切换到B话题
2. 能量转变: 从calm到tense，或从tense到release
3. 空间切换: 镜头转向新的空间区域
4. 角色焦点转移: 从关注角色A转到关注角色B
5. 时间间断: 任何时间跳跃（即使是微小的省略）

短语长度:
  最短: 2个镜头（最小有意义单元: 动作+反应）
  标准: 3-5个镜头（认知舒适区）
  最长: 7个镜头（超过7个，观众会自行在心理上分段）

  Miller法则的应用: 工作记忆 7±2 项 → 短语不超过7个镜头
```

### 短语内部结构

```
三拍短语 (最紧凑):
  建立 → 发展 → 解决
  [setup] [develop] [resolve]
  镜头时长比: 1.0 : 1.2 : 0.8 （中间略长，结尾干脆）

四拍短语 (标准):
  建立 → 发展 → 转折 → 解决
  [setup] [develop] [turn] [resolve]
  镜头时长比: 1.0 : 1.0 : 1.3 : 0.9 （转折最长）

五拍短语 (完整):
  建立 → 发展 → 高潮 → 反应 → 解决
  [setup] [develop] [climax] [react] [resolve]
  镜头时长比: 0.8 : 1.0 : 1.5 : 1.2 : 0.7 （高潮最长，前后递减）
```

### 段落结构

```
标准三段式:
  段落1 (Building): 2-3个短语，节奏从松到紧
  段落2 (Climax): 1-2个短语，包含场景的最高张力点
  段落3 (Resolution): 1-2个短语，释放并过渡

段落间的"呼吸":
  段落边界通常由以下镜头标记:
    - 大景别重建立镜头 (ELS/LS)
    - 异常长镜头 (Fermata)
    - 空间切换
    - 沉默
```

### 短语标注语法

```
在分镜序列中标注短语和段落:

=== SECTION 1: Building (4/4 rubato, ♩=15) ===

  --- Phrase 1a: 建立空间 [setup] ---
  | Shot1(ELS, 5s) | Shot2(MS, 3s) | Shot3(MCU, 2.5s) |
  phrase_duration: 10.5s | phrase_rhythm: Ritardando_micro

  --- Phrase 1b: 角色进入 [develop] ---
  | Shot4(MS, 3s) | Shot5(MCU, 4s) | Shot6(CU, 3.5s) | Shot7(CU, 2s) |
  phrase_duration: 12.5s | phrase_rhythm: Breathe

=== SECTION 2: Climax (4/4 accel, ♩=15→25) ===
  ...
```

## 十三、Rhythmic Memory Model（节奏记忆模型）

> 观众不是逐帧感受节奏，而是在心理上建立"期望"。连续的模式建立期望，打破期望制造惊奇。

### 期望建立公式

```
E(n) = 期望强度，在第n个镜头处

E(n) = Σ_{i=1}^{n} w(i) * match(shot_i, pattern)

其中:
  w(i) = λ^(n-i)     # 指数衰减权重，近处影响大
  λ = 0.7             # 衰减率（3个镜头前的影响只有0.7^3 = 34%）
  match(shot, pattern) = 1 if shot符合已建立的模式, 0 otherwise

直觉:
  3个连续等时长镜头后，E ≈ 0.7 + 0.49 + 0.34 = 1.53 → 中等期望
  5个连续等时长镜头后，E ≈ 2.38 → 强期望
  7个连续等时长镜头后，E ≈ 2.94 → 极强期望（打破会产生强烈惊奇）
```

### 惊奇度计算

```
Surprise(n) = E(n-1) * |D(shot_n) - D_expected(n)| / D_expected(n)

其中:
  E(n-1) = 打破前的期望强度
  D(shot_n) = 实际时长
  D_expected(n) = 基于模式推测的期望时长

Surprise ∈ [0, ∞)
  0-0.5: 微弱惊奇（观众几乎不注意）
  0.5-1.5: 中等惊奇（"咦？"——注意力重新聚焦）
  1.5-3.0: 强烈惊奇（情感冲击）
  >3.0: 断裂（可能失去观众——除非有强叙事理由）
```

### 模式识别窗口

```
观众在心理上使用滑动窗口来识别模式:

窗口大小: 3-5个镜头（短期记忆容量）

识别的模式类型:
  1. 等距 (Isochronous): 连续N个镜头时长差异 < 15%
  2. 单调递减 (Accelerando): 连续N个镜头时长严格递减
  3. 单调递增 (Ritardando): 连续N个镜头时长严格递增
  4. 交替 (Alternating): 长-短-长-短 模式
  5. 分组 (Grouped): 每K个镜头形成一组，组间有间隔

识别阈值:
  N ≥ 3: 模式开始被感知
  N ≥ 5: 模式被确认，期望强烈
  N ≥ 8: 模式变得催眠/机械——打破或观众走神
```

### 叙事应用

```
建立-打破策略:

策略1: 3+1突变
  建立3个等距镜头 → 第4个突然延长3倍
  E(3) ≈ 1.53, Surprise ≈ 1.53 * 2.0 = 3.06 → 强烈冲击
  适用: 发现尸体、看到真相、角色背叛

策略2: 5+1突变
  建立5个等距镜头 → 第6个缩短到1/3
  E(5) ≈ 2.38, Surprise ≈ 2.38 * 0.67 = 1.59 → 中强冲击
  适用: 从沉思突然被打断、电话铃声、枪响

策略3: 渐进渗透
  模式A建立4个 → 模式B的元素混入（先1个，再2个）→ 完全替换
  适用: 从正常到异常的渐变（梦境侵入、精神失控）
  惊奇度: 低但持续，制造"不对劲"的不安感

策略4: 伪模式
  建立看似固定的模式（实际有微小变化）→ 观众的期望从未完全稳定
  适用: 心理惊悚——始终有"哪里不对"的感觉
  Kubrick常用: 《The Shining》的steadicam跟拍，看似规律实际不规律
```

## 十四、1/f Distribution Compliance（1/f分布合规检测）

> Cutting, DeLong & Nothelfer (2010) 发现好莱坞电影的镜头时长遵循1/f噪声（粉红噪声）分布。这不是巧合——它映射了人类注意力的自然波动。
>
> 1/f分布的认知科学基础见 cognitive-perception.md Source 3 (Cutting et al. 2010)。

### 1/f噪声的本质

```
1/f噪声（Pink Noise）:
  功率谱密度 S(f) ∝ 1/f^α, 其中 α ≈ 1

  白噪声 (α=0): 完全随机，无结构 → 混乱的剪辑
  粉红噪声 (α=1): 分形结构，大尺度和小尺度都有相关性 → 自然节奏
  棕色噪声 (α=2): 过度相关，变化缓慢 → 单调的剪辑

  电影数据 (Cutting et al.):
    α=0.0-0.5: 早期电影（1930s），较随机
    α=0.7-1.0: 经典好莱坞（1960s-1990s），接近1/f
    α=1.0-1.3: 现代电影（2000s+），更强相关性

  悬疑/惊悚最佳区间: α ∈ [0.8, 1.2]
```

### 1/f合规性检测算法

```
输入: 镜头时长序列 D = [d_1, d_2, ..., d_N]
输出: α值和合规判定

Step 1: 去趋势
  移除整体趋势（如果场景有系统性加速/减速）:
  D_detrended = D - linear_fit(D)

Step 2: 计算功率谱
  使用离散傅里叶变换(DFT):
  F = FFT(D_detrended)
  S(f) = |F(f)|^2  (功率谱密度)

Step 3: 对数-对数拟合
  在 log(f) vs log(S(f)) 空间做线性回归:
  log(S) = -α * log(f) + c
  α = 回归斜率的绝对值

Step 4: 判定
  α ∈ [0.8, 1.2] → PASS: 符合1/f分布
  α ∈ [0.5, 0.8) → WARN: 偏随机，缺乏长程相关
  α ∈ (1.2, 1.5] → WARN: 偏平缓，节奏变化不够
  α < 0.5 → FAIL: 过于随机
  α > 1.5 → FAIL: 过于单调
```

### 简化版检测（无需FFT）

```
对于分镜实践中的快速检测，使用autocorrelation代替完整功率谱:

Step 1: 计算lag-1自相关 r(1)
  r(1) = Σ(d_i - μ)(d_{i+1} - μ) / Σ(d_i - μ)^2

Step 2: 计算lag-2自相关 r(2)
  r(2) = Σ(d_i - μ)(d_{i+2} - μ) / Σ(d_i - μ)^2

Step 3: 判定
  r(1) ∈ [0.3, 0.7] 且 r(2) ∈ [0.1, 0.5] → 接近1/f
  r(1) < 0.1 → 过于随机（白噪声）
  r(1) > 0.85 → 过于相关（棕色噪声）
  r(1) < 0 → 反相关（不自然的刻意交替）

直觉: 前一个镜头的时长应该对下一个有中等程度的影响（不是完全无关，也不是完全决定）
```

## 十八、节奏与视觉隐喻的关系

> 节奏不仅是时间结构，也是隐喻的"引擎"。特定的节奏模式强化特定的视觉隐喻，反之亦然。
> 详细的隐喻定义见 visual-metaphor.md。

### 核心原理

```
节奏-隐喻耦合法则:

  隐喻的情感效果 = 视觉符号 × 节奏模式

  同一个视觉符号（如楼梯），配上不同节奏，传递完全不同的含义:
    楼梯 + Accelerando = 急切的攀登/追逐
    楼梯 + Ritardando  = 疲惫的下行/堕落
    楼梯 + Fermata     = 在某一级台阶上的顿悟

  节奏是隐喻的时间维度——它告诉观众"以什么速度感受这个符号"。
```

### 节奏-隐喻组合示例

```
示例1: M09(楼梯=旅程) + Accelerando

  《寄生虫》楼梯下降场景 (参见 technique-library.md T07):
    节奏设计: A(4, 0.85, 6) — 从4s递减到1.8s
    每级楼梯=一个社会阶层，时长递减=坠落加速
    Composition: CONCAT(A(4, 0.85, 6), F(1.5, 8, 1))
    → 加速下降后，在被淹没的家中Fermata停顿

示例2: M12(走廊=不可逃避) + Ritardando

  《闪灵》走廊跟拍 (参见 technique-library.md T06):
    节奏设计: R(2, 1.3, 5) — 从2s递增到5.7s
    每个转角后的走廊段落越来越长=时间在减速
    主观体验: "我走不到尽头，时间越来越慢"
    Composition: NEST(R(6, 1.3, 4), P(2, 3))
    → 宏观减速，微观保持Danny三轮车的脉搏

示例3: M10(时钟=压迫) + Pulse strict → Accelerando

  倒计时场景:
    Phase 1: P(4, 8) — 8个等距4s镜头，模拟时钟滴答
    Phase 2: A(4, 0.8, 10) — 加速，时钟在"加快"
    Phase 3: Bu(0, 0.5, 6, 0, 6) — 爆发→Fermata结果
    Composition: CONCAT(P(4, 8), A(4, 0.8, 10), Bu(0, 0.5, 6, 0, 6))

示例4: M04(镜像=分裂) + 双层Breathe

  角色面对镜子的场景:
    Layer A (角色): Br(4, 2, 6) — 正常呼吸
    Layer B (镜像): Br(4.3, 1.8, 6) — 略有偏移的呼吸
    INTERLEAVE(A, B) → 观众感受到"同步但不同步"的分裂感
    越到场景后段，两个Breathe的偏差越大=分裂加深
```

### 反模式：节奏-隐喻冲突

```
冲突会削弱隐喻效果:

  ✗ M12(走廊=不可逃避) + Accelerando
    → 加速暗示"可以逃跑"，与走廊的宿命感矛盾
    → 除非: 走廊用于追逐场景（此时隐喻语义从"不可逃避"转为"无处可逃"）

  ✗ M10(时钟=压迫) + Ritardando
    → 减速暗示"时间充裕"，与时钟的压迫感矛盾
    → 除非: 故意制造"时间停止"的超现实效果

  ✗ M21(沉默=恐惧) + 快切蒙太奇
    → 快切破坏沉默，两者无法共存
    → 除非: 无声快切（视觉碎片化+听觉沉默=极端不安，参见T01变体）

  ✗ M09(楼梯=权力转移) + Pulse uniform
    → 等距节拍暗示无变化，与"转移"的动态性矛盾
    → 应该使用有方向性的节奏（Accelerando或Ritardando）

诊断规则:
  如果选定的隐喻暗示"变化/运动" → 节奏必须有方向性（非Pulse/Uniform）
  如果选定的隐喻暗示"停滞/困境" → 节奏应该是重复/锁定型
  如果选定的隐喻暗示"分裂/双重" → 节奏应该有两层或交替结构
```
