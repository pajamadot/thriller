### v2.1 审计发现的三个结构性缺口

```
GAP-1: CONTINUITY CUT 未被覆盖
  原系统的三种切理由 (INFORMATION / REACTION / EMPHASIS) 无法解释
  "维持空间连贯的重定向切" —— 即当角色移动或新角色进入时，
  为了保持观众空间认知而必须切到包含新空间关系的镜头。
  这既不是新信息（空间已知），也不是反应（没有事件），
  也不是强调（情感距离未变）。
  → 解决: 增加第四理由 ORIENTATION

GAP-2: 单通道 PROVIDES 模型不足
  原系统的 PROVIDES 是一个扁平列表，无法区分同一镜头同时
  在视觉通道（如：角色表情）和听觉通道（如：环境音变化）
  和叙事潜文本通道（如：隐含的权力转移）上提供的不同信息。
  这导致:
    - 无法检测纯听觉镜头（暗画面+声音）的合法性
    - 无法建模内心独白场景的信息流
    - 无法量化多通道信息密度
  → 解决: 三通道 PROVIDES 模型 (V/A/N)

GAP-3: VERIFY 谓词是散文，不可计算
  原系统的验证规则用自然语言描述，存在歧义，
  LLM 执行时结果不一致（同一序列不同 session 可能给出不同判断）。
  → 解决: 编号谓词 P01-P20，每个谓词有 IF/THEN 形式，
     返回 PASS/FAIL/WARN，可被 LLM 确定性执行
```

## 一、核心问题：为什么要切？

> "The single most important thing about editing is: every cut must have a reason."
> —— Walter Murch

> "Information theory tells us: a message has value only if it reduces uncertainty."
> —— Claude Shannon, applied to visual narrative by Bordwell (1985)

每一次剪辑（从镜头A切到镜头B）都必须回答一个问题：**为什么现在要切？**

如果你无法给出理由，那就不要切。延长当前镜头。

### 信息论基础

从 Shannon 信息论的视角看，每个镜头是一个**信息源**，每次切是一个**信道切换**。
切的合法性 = 新信道提供的信息增益 > 信道切换的认知成本。

Bordwell (1985) 在《Narration in the Fiction Film》中指出，电影叙事的信息分配策略有三种模式:
- **限知叙事**: 观众 ≤ 角色的信息量 → 悬念
- **全知叙事**: 观众 > 角色的信息量 → 悬疑 (Hitchcock)
- **交替叙事**: 在限知与全知之间切换 → 节奏控制

每次切都隐含一个信息分配决策：**切给观众看什么，不切让观众不看什么。**

Salt (2009) 在《Film Style and Technology》的镜头统计中发现:
- 对话场景平均镜头时长 (ASL) = 4-6s
- 动作场景 ASL = 2-3s
- 建立场景 ASL = 6-10s
- 现代电影 ASL 整体趋势: 从 1930s 的 10s 下降到 2010s 的 4s

这些统计不是规则，而是**基线**——偏离基线必须有叙事理由。

### 四个合法的切的理由

```
理由 1: 新信息（INFORMATION）
  画面中出现了当前镜头无法展示的新信息
  → 切到能展示这个信息的新镜头

  形式化谓词 [参见 P01]:
    IF 存在信息单元 I，使得:
      (1) I 对叙事推进是必要的，且
      (2) 当前镜头的取景范围/角度无法展示 I
    THEN 切到能展示 I 的新镜头
    REASON = INFORMATION

  示例:
    角色A说"你看那边" → 切到A所指的方向（POV/INSERT）
    电话铃响 → 切到电话的特写
    观众需要看到角色B的存在 → 切到包含B的镜头

  边缘案例:
    信息已在 A-channel（听觉）提供，V-channel（视觉）是否仍需切？
    → 取决于信息重要性: 如果是关键线索，V-channel 确认是必要的
    → 如果是环境信息（如背景音乐变化），不需要视觉切

理由 2: 反应（REACTION）
  一个事件发生后，叙事需要展示某人的反应
  → 切到反应者的面部或肢体

  形式化谓词 [参见 P02]:
    IF 镜头 S[n] 中发生了事件 E，且
      存在角色 C 使得:
      (1) C 对 E 的反应携带叙事信息（情感/线索/性格），且
      (2) C 不在当前镜头的可见范围内，或
      (3) C 在当前镜头中但景别不足以展示反应细节
    THEN 切到 C 的反应镜头
    REASON = REACTION

  示例:
    角色A说出关键台词 → 切到角色B听到后的面部反应
    爆炸发生 → 切到目击者的表情
    线索出现 → 切到侦探的眼神变化

  边缘案例（反应链）:
    当多个角色需要展示反应时（如宣布消息后），
    按叙事重要性排序反应镜头。
    最重要的反应放最后（延迟满足 = 更大冲击）。

  Hitchcock: "对话场景中，更重要的不是说话的人，
  而是听话的人的反应。"

理由 3: 强调（EMPHASIS）
  叙事需要改变观众与画面的情感距离
  → 切到不同景别/角度来改变情感权重

  形式化谓词 [参见 P03]:
    IF 叙事节拍的重力值发生变化 ΔG ≥ 2，或
      叙事需要观众注意到当前镜头中的特定元素 E，
      但 E 在当前景别中不够突出
    THEN 切到不同景别/角度的镜头
    REASON = EMPHASIS
    约束: 新景别与旧景别差 ≥ 1 级

  示例:
    角色走进房间（MS）→ 角色发现异常时切到CU（强调发现的重要性）
    对话正常进行（MS）→ 关键台词时切到CU（这句话很重要）
    角色孤独地站着（CU）→ 拉到ELS（强调渺小/孤独）

  边缘案例:
    强调 vs 信息的区分: 如果切到新景别同时引入了新信息，
    主要理由是 INFORMATION（新信息是主因），
    强调是附带效果。用 INFORMATION + EMPHASIS 双标签。

理由 4: 重定向（ORIENTATION） [v2.1 新增]
  空间关系发生变化，需要重新建立观众的空间认知
  → 切到包含新空间关系的镜头

  形式化谓词 [参见 P04]:
    IF 以下任一条件成立:
      (1) 角色移动导致原有空间格局改变（A型→L型，有人站起/走动），且
          当前镜头的取景范围无法展示新格局
      (2) 新角色进入场景，需要展示其与现有角色的空间关系
      (3) 连续 ≥ 3 个 CU/ECU 镜头后，观众可能丧失空间定位
      (4) 时间跳跃后同一空间的状态发生了变化
    THEN 切到更宽的景别（通常 MS 或以上）展示新的空间关系
    REASON = ORIENTATION

  与其他三种理由的区分:
    ORIENTATION ≠ INFORMATION: 不是引入新的叙事信息，
      而是重新确认已知空间的当前状态
    ORIENTATION ≠ REACTION: 不是展示对事件的回应
    ORIENTATION ≠ EMPHASIS: 不是改变情感距离，
      而是恢复空间认知的可靠性

  示例:
    正反打对话持续了 6 个 CU 后 → 切回双人 MS 重新定位空间关系
    角色从沙发走到窗边 → 切到 MS/MLS 展示新位置
    第三人进入房间 → 切到包含三人的 MLS 重新建立空间

  Bordwell 的理论依据:
    在《Film Art》中，Bordwell 指出 continuity editing 的首要目标
    是维护"空间连贯性" (spatial coherence)。当空间关系改变而
    镜头未更新时，观众会经历"空间漂移" (spatial drift) ——
    即失去对角色在场景中相对位置的确定感。
    重定向切是预防空间漂移的主动措施。

  反面案例:
    对话场景中每 2 个正反打就插入一个 master 重定向 → 过度
    （观众的空间记忆在 3-5 个同场景切内是稳定的）
    → ORIENTATION 的频率应该低于其他三种理由
```

### 四种切理由的优先级与共现规则

```
当一个切同时满足多个理由时，按以下优先级标注主要理由:

  1. INFORMATION（最高）— 新信息是最强的切理由
  2. REACTION — 反应紧随事件，时效性强
  3. EMPHASIS — 情感权重调整
  4. ORIENTATION（最低）— 空间维护是最"安静"的切理由

共现标注格式:
  REASON = INFORMATION + EMPHASIS  (主: 新信息, 副: 景别变化强调)
  REASON = REACTION               (单一理由)
  REASON = ORIENTATION             (纯空间重定向)
```

### 反面：不切的理由（保持当前镜头）

```
以下情况不应该切:

  1. 当前镜头内的动作还没有完成
     → 等动作完成或到达自然节点再切

  2. 没有新信息、没有需要展示的反应、不需要改变情感距离、
     空间关系没有改变
     → 延长当前镜头（长镜头本身就有叙事力量）

  3. 刚切过来不到1.5秒
     → 观众还没有消化上一个镜头的信息
     → 除非是故意的快切蒙太奇
     → Salt (2009) 统计: 低于 1s 的镜头仅占总镜头的 3-5%

  4. 切过去之后没有新东西给观众看
     → 新镜头必须比旧镜头多至少一个信息单元（任意通道）

  5. [v2.1 新增] 切会中断正在建立的多通道信息流
     → 如果一个镜头同时在 V 和 A 通道构建信息，
        在两个通道都完成之前不要切
```

## 二、多通道信息依赖图（Multi-Channel Shot Information DAG）

每个镜头不是孤立的——它依赖于前面的镜头提供的信息，并且为后面的镜头提供必要信息。

**v2.1 关键升级: 三通道模型**

原 v1/v2 的 PROVIDES 是单一列表，无法建模以下常见情况:
- 一个镜头同时通过画面（角色表情）和声音（声音颤抖）提供信息
- 纯声音场景（黑画面 + 声音）的合法性
- 内心独白场景中叙事潜文本是唯一信息源
- 多通道信息冲突制造的不安感（画面平静 + 声音紧张 = 对位）

### 三通道信息模型

```
每个镜头在三个独立通道上携带信息:

  V-Channel（视觉通道）:
    画面直接展示的信息 — 角色、空间、物体、动作、表情、构图
    这是传统 PROVIDES 的主要内容
    量化: 每个可辨识的视觉信息单元 = 1V

  A-Channel（听觉通道）:
    声音携带的信息 — 对话、环境音、音乐、沉默
    包括: 台词内容、语气、声音变化、声音缺失
    量化: 每个可辨识的听觉信息单元 = 1A

  N-Channel（叙事潜文本通道）:
    画面和声音都没有直接表达，但通过构图、并置、
    节奏暗示传达的潜在信息 — 权力关系、隐含情感、
    角色心理、叙事反讽
    量化: 每个可推断的叙事潜文本单元 = 1N

通道间关系:
  同步: V 和 A 传达相同信息 → 强化
  互补: V 和 A 传达不同但相容的信息 → 丰富
  对位: V 和 A 传达矛盾的信息 → 不安/反讽
  单通道: 只有一个通道有信息 → 特殊效果

信息密度 (INFO_DENSITY):
  ID = V_count + A_count + N_count
  单镜头 ID 建议范围: 1-5
  ID > 5: 信息过载风险 → 考虑拆分
  ID = 0: 空转镜头 → 必须删除或赋予信息
```

### 完整信息属性（三元组 × 三通道）

```
每个镜头有 3 × 3 = 9 个信息属性:

  REQUIRES:
    V-REQUIRES: 观众需要已经看到过什么？
    A-REQUIRES: 观众需要已经听到过什么？
    N-REQUIRES: 观众需要已经理解什么潜文本？

  PROVIDES:
    V-PROVIDES: 这个镜头在视觉上提供什么新信息？
    A-PROVIDES: 这个镜头在听觉上提供什么新信息？
    N-PROVIDES: 这个镜头暗示了什么潜文本？

  RAISES:
    V-RAISES: 画面引发了什么视觉上的疑问？
    A-RAISES: 声音引发了什么听觉上的疑问？
    N-RAISES: 潜文本引发了什么深层疑问？

简化规则:
  当某通道无信息时，可省略该通道的标注。
  至少一个通道必须有 PROVIDES（否则空转）。
  大多数场景只需标注 2 个主通道。
```

### 示例：《镜中访客》ch1 咨询室场景（多通道标注）

```
Shot 1: 咨询室建立镜头 (ELS→MS)
  CUT-REASON: — (开场镜头)
  V-REQUIRES: 无
  V-PROVIDES: 空间定位（咨询室布局）、时间（下午光线）、叶知秋独坐
  A-PROVIDES: 环境音（空调嗡鸣、时钟滴答）→ 建立安静、密闭的氛围
  N-PROVIDES: 叶知秋的孤独状态（一个人在等待 → 职业日常）
  V-RAISES: 无
  A-RAISES: 无
  N-RAISES: 无
  INFO_DENSITY: 3V + 2A + 1N = 6 → 接近上限，但开场镜头允许

Shot 2: 林小曼进门 (MS)
  CUT-REASON: INFORMATION (新角色进入)
  V-REQUIRES: Shot 1 的空间定位
  V-PROVIDES: 林小曼的存在、她化了妆（粉底下的青黑）
  A-PROVIDES: 门声、脚步声
  N-PROVIDES: 化妆 = 异常（熟客突然在意外表 → 有事要说）
  V-RAISES: 为什么化妆？
  N-RAISES: 她来做什么？（与往常不同）

Shot 3: 林小曼坐下，犹豫 (MCU)
  CUT-REASON: EMPHASIS (从 MS 推近到 MCU → 聚焦情感)
  V-REQUIRES: Shot 2 已建立林小曼的存在
  V-PROVIDES: 紧张的肢体语言（绞袖子、目光回避）
  A-PROVIDES: 沉默（对话的缺失本身是信息）
  N-PROVIDES: 犹豫 = 她要说的事情很严重
  V-RAISES: 她的手在做什么？
  A-RAISES: 为什么不说话？
  N-RAISES: 她要说什么？

Shot 4: "有人在跟踪我" (CU 林小曼)
  CUT-REASON: EMPHASIS (从 MCU 推到 CU → 关键台词)
  V-REQUIRES: Shot 3 的犹豫氛围
  A-REQUIRES: Shot 3 的沉默（沉默的打破 = 更大的冲击）
  V-PROVIDES: 林小曼说话时的面部——恐惧、不确定
  A-PROVIDES: 台词"有人在跟踪我"+ 声音变小
  N-PROVIDES: 核心叙事信息——从母女关系议题突然转向被害
  V-RAISES: 无
  A-RAISES: 无
  N-RAISES: 是谁？为什么？是真的还是焦虑症状？

Shot 5: 叶知秋的反应 (CU 叶知秋)
  CUT-REASON: REACTION (对 Shot 4 的关键台词)
  V-REQUIRES: Shot 4 的台词已完成
  V-PROVIDES: 叶知秋的面部反应——专业的冷静中有一丝警觉
  A-PROVIDES: "能具体说说吗？" → 专业询问（但语气略有变化）
  N-PROVIDES: 叶知秋的第一反应是"焦虑障碍"→ 职业框架限制了判断
  N-RAISES: 她能正确判断吗？职业训练会不会成为盲点？

Shot 6: "他说了你妈妈的原话" (CU 林小曼 → INSERT 笔记本)
  CUT-REASON: INFORMATION (新信息: 跟踪者知道咨询内容)
  V-REQUIRES: Shot 4-5 建立的对话语境
  A-REQUIRES: Shot 5 的追问
  V-PROVIDES: 林小曼直视叶知秋的眼睛（求助 + 指控）→ INSERT: 笔记本上的记录
  A-PROVIDES: "你妈妈说得对，你确实不值得被爱" → 原话复述
  N-PROVIDES: 信息泄露 = 不可能事件 → 只有叶知秋知道这些内容
  V-RAISES: 笔记本上写了什么？
  A-RAISES: 无
  N-RAISES: 谁泄露了？怎么泄露的？这是否指向叶知秋自己？

Shot 7: 叶知秋的内心震动 (ECU 叶知秋眼睛 + SIL)
  CUT-REASON: REACTION + EMPHASIS (对信息升级的反应 + 从 CU→ECU 极端推近)
  V-REQUIRES: Shot 6 的信息揭示
  V-PROVIDES: 叶知秋眼中的震动——瞳孔微张、眼球微颤
  A-PROVIDES: 沉默（SIL）→ 声音的突然消失 = 内心冲击
  N-PROVIDES: 叶知秋意识到这不是焦虑——是真实威胁 + 指向自己的职业危机
  N-RAISES: 她要怎么办？如果记录真的泄露了，她的职业生涯结束了
```

### 信息依赖图（DAG）—— 多通道版

```
     V-channel         A-channel         N-channel
     ─────────         ─────────         ─────────

  S1 [空间+角色]    S1 [环境音]        S1 [孤独]
     │                  │                  │
     ↓                  ↓                  ↓
  S2 [林小曼+妆]    S2 [门声]          S2 [异常信号]
     │                  │                  │
     ↓                  ↓                  ↓
  S3 [肢体语言]     S3 [沉默]          S3 [犹豫=严重]
     │                  │                  │
     ↓                  ↓                  ↓
  S4 [面部恐惧]     S4 [关键台词]      S4 [被害?焦虑?]
     │                  │              ╱   │
     ↓                  ↓            ╱     ↓
  S5 [专业面具]     S5 [追问]       ╱   S5 [职业盲点]
     │                  │         ╱        │
     ↓                  ↓       ╱          ↓
  S6 [眼神+笔记]    S6 [原话]  ╱       S6 [不可能泄露]
     │                  │    ╱             │
     ↓                  ↓  ╱               ↓
  S7 [瞳孔震动]     S7 [沉默]          S7 [职业危机]

注意:
  - N-channel 的依赖可以跨越多个镜头（S4→S6 的 N 依赖跳过了 S5）
  - A-channel 的 S3[沉默]→S4[台词] 构成"沉默-打破"模式
  - V-channel 是最线性的（每个镜头都依赖前一个）
  - S7 是三通道汇聚点——V/A/N 同时到达高点
```

### 多通道依赖的特殊模式

```
模式 A: 通道分离 (Channel Split)
  一个镜头在 V 和 A 通道提供不同的信息
  示例: 画面是角色平静的脸 (V: 平静) + 声音是加速的心跳 (A: 恐惧)
  效果: 通道矛盾 = 角色在压抑情感 → 悬疑片的核心手法
  → N-PROVIDES 由观众自行推断: "她在隐藏什么"

模式 B: 通道延迟 (Channel Delay)
  信息先在 A 通道出现，若干镜头后才在 V 通道确认
  示例: 先听到脚步声 (A) → 3个镜头后才看到来人 (V)
  效果: 延迟 = 悬念积累
  → A-RAISES 驱动后续镜头的 V-PROVIDES

模式 C: 单通道场景 (Single-Channel Scene)
  仅有一个通道携带信息:
    纯 V: 无声场景（暗示耳聋、回忆、梦境）
    纯 A: 黑画面 + 声音（暗示失明、意识模糊、电话另一端）
    纯 N: 看似无事的画面 + 观众根据前文推断出潜文本
  验证: 单通道场景仍然合法，只要该通道有 PROVIDES

模式 D: 通道覆盖 (Channel Override)
  新镜头的某通道信息压制/替代旧镜头同通道的信息
  示例: S1 的 A-PROVIDES 是轻柔音乐，S2 的 A-PROVIDES 是尖叫
  → 尖叫覆盖音乐 → 不需要在 A-REQUIRES 中声明音乐
```

### 依赖验证规则（形式化版本）

```
以下规则已被编号为谓词 P05-P10，详见第五章形式化谓词表。

□ [P05] 每个镜头的每个通道的 REQUIRES 都被前面某个镜头
        的同通道或跨通道 PROVIDES 满足？
  → 跨通道满足: A-REQUIRES 可以被 V-PROVIDES 满足
     （观众看到了所以不需要听到），反之亦然
  → 但 N-REQUIRES 只能被 N-PROVIDES 或 (V+A)-PROVIDES 共同满足

□ [P06] 每个镜头至少有一个通道的 PROVIDES 非空？
  → 如果三通道 PROVIDES 全空 → 空转镜头 → 删除

□ [P07] RAISES 链条在至少一个通道上持续活跃？
  → 在任意时刻，三通道的 RAISES 总和 ≥ 1
  → 如果所有通道的 RAISES 都为零 → 叙事"死点"

□ [P08] 不存在循环依赖？
  → 在每个通道内和跨通道都不应存在 A→B→A 的依赖

□ [P09] 单镜头 INFO_DENSITY ≤ 5？
  → 超过时考虑拆分（开场建立镜头例外，允许 ID ≤ 7）

□ [P10] 三通道汇聚点出现在叙事高点？
  → V + A + N 同时达到高 PROVIDES 的镜头应对应场景高潮
  → 如果汇聚点在非高潮位置 → 检查是否为结构问题
```

## 三、镜头序列惯用语（Film Idioms as State Machines）

镜头序列不是自由排列的——存在经过百年验证的固定模式（idiom），
就像语言中的"固定搭配"。

### Idiom 1: 建立序列（Establishing Sequence）

```
状态机:

  [场景开始] → ELS/LS(环境) → MS/MLS(角色在环境中) → 继续

  逻辑: 先让观众知道"在哪里"，再聚焦"谁在这里"
  违反后果: 观众不知道角色在什么空间里 → 空间迷失

  变体:
    经典: ELS → LS → MS
    跳过: 直接从MS开始 → 适用于观众已知的空间（之前建立过）
    延迟: MS → 对话 → LS(走到窗边) → 通过窗户看到ELS
         → 延迟揭示空间，制造悬念

  决策规则:
    这是第一次进入这个空间？ → 需要建立镜头
    这个空间已经出现过？ → 可以跳过（直接从MS开始）
    空间的变化（时间/天气/状态）有叙事意义？ → 即使出现过也需要重新建立
```

### Idiom 2: 正反打序列（Shot-Reverse-Shot）

```
状态机:

  [对话开始] → Master(双人) → OTS-A → OTS-B → CU-A → CU-B → ...
                                 ↕         ↕
                              (按需切换，遵循180度轴线)

  逻辑:
    Master 建立空间关系 → OTS 进入对话 → CU 在关键时刻聚焦

  何时从 OTS 升级到 CU:
    1. 角色说出关键台词
    2. 角色在撒谎（观众需要看微表情）
    3. 角色情绪爆发
    4. 权力关系发生变化

  何时从 CU 退回 Master/OTS:
    1. 对话缓和
    2. 需要重新确认空间关系（有人移动了）
    3. 新角色进入场景
    4. 对话结束

  轴线规则:
    摄影机始终在两人连线的同一侧
    A在画面左侧 ↔ B在画面右侧（始终保持）
    越轴只在权力关系颠覆时使用

  认知依据:
    Magliano et al. (2011): 正反打被土耳其无电视村庄的
    居民误解为"两个人在不同时空" → 这不是天生能理解的，
    而是习得的视觉语法。观众通过轴线一致性理解"他们在对话"。
```

### Idiom 3: 揭示序列（Reveal Sequence）

```
状态机:

  [揭示准备] → 建立期望 → 延迟 → 揭示 → 反应

  三种揭示模式:

  模式A — 渐进揭示（Progressive）:
    LS(环境有异常) → MS(角色注意到) → MCU(角色靠近) →
    CU(角色表情变化) → INSERT/ECU(揭示物)
    逻辑: 景别逐步收窄 = 注意力逐步聚焦 = 真相逐步显露
    认知依据: Liao et al. (2023) 证实 scale-in(宽→窄)
    在神经层面几乎不被意识到 → 观众自然跟随

  模式B — 突然揭示（Sudden）:
    MS/LS(完全无预警的画面) → 硬切到 CU/ECU(揭示物)
    逻辑: 平静中的突然冲击
    注意: 需要至少3秒的"平静"做铺垫，否则冲击无效
    认知依据: Liao et al. (2023) 证实 scale-out(窄→宽)
    会引发更强的神经反应 → 观众意识到"刚才发生了什么"

  模式C — 延迟揭示（Delayed）:
    POV(角色看到了什么) → CU(角色反应——但观众不知道看到了什么) →
    延迟若干镜头 → 最终揭示(INSERT/CU)
    逻辑: 先给反应再给原因 → Hitchcock式悬念
```

### Idiom 4: 追逐/紧迫序列（Chase/Urgency Sequence）

```
状态机:

  [追逐开始] → A(追者)MS → B(被追者)MS → 环境LS →
               A(更近)MCU → B(更急)MCU → 环境(缩小) →
               → 循环，每次景别更紧，时长更短 →
  [追上/逃脱]

  逻辑:
    追者和被追者交替出现 → 平行蒙太奇
    景别逐渐收窄 → 距离在缩小
    镜头时长逐渐缩短 → 节奏加速
    环境镜头的比例逐渐减少 → 世界在缩小

  规则:
    追者向右走 → 被追者向右走 → 银幕方向一致
    偶尔的LS提醒观众两者的距离关系
    最后的汇合点用MS或CU，不用LS（保持紧迫）
```

### Idiom 5: 窥视/监视序列（Voyeur/Surveillance Sequence）

```
状态机:

  [监视建立] → 窥视者POV(有框架/遮挡) → 被窥视者(不知情的动作) →
               窥视者反应(CU) → 窥视者POV(更近/不同角度) → ...

  逻辑:
    POV镜头通过框架（门缝/窗户/望远镜）建立偷窥视角
    被窥视者的画面"正常"但被偷窥的语境重新定义了含义
    窥视者和被窥视者的景别差异 → 权力不对称

  Hitchcock核心技巧:
    让观众成为窥视者 → 观众的罪恶感/紧张感
    观众知道比角色多的信息 → 悬念（不是惊吓）
```

### Idiom 6: 反应链序列（Reaction Chain）

```
状态机:

  [事件发生] → 反应者A(CU) → 反应者B(CU) → 反应者C(CU) →
               → 回到事件(可能有变化) → ...

  逻辑:
    一个事件 → 连续展示多人反应 → 每个反应不同
    比较不同角色的不同反应 → Kuleshov效应的应用
    每个反应CU告诉观众这个角色的性格/秘密

  悬疑用法:
    宣布消息 → A惊恐 → B平静 → C假装惊恐 →
    观众比较三人反应 → 推理谁是凶手
```

### Idiom 7: 告白/揭露对话（Confession/Revelation Dialogue）

```
入口条件:
  一方角色持有秘密或关键信息，需要在对话中向另一方揭露。
  揭露具有情感重量——不只是传递数据，而是改变两人关系。

状态机:

  [一方持有秘密] → MASTER(双人,MS) → 倾听者OTS(MS) →
                   告白者OTS(MS) → 告白者MCU(开始吐露) →
                   倾听者CU(反应) → 告白者CU(深入) →
                   告白者ECU(核心真相) → 倾听者ECU(冲击) →
                   [MASTER(新关系)/沉默]

  状态图:
    S0 [铺垫]     → MASTER(MS), 正常对话氛围
    S1 [犹豫]     → 告白者MCU, 欲言又止, 停顿增多
    S2 [开口]     → 告白者OTS→MCU, 开始陈述
    S3 [深入]     → 告白者CU, 细节涌出, 情绪升温
    S4 [核心]     → 告白者ECU, 最痛苦/最关键的真相
    S5 [冲击]     → 倾听者ECU, 消化/崩溃/接受
    S6 [新平衡]   → MASTER(新景别), 关系已改变

  转换规则:
    S0→S1: 对话出现第一个异常停顿或欲言又止
    S1→S2: 告白者决定说出来（"我必须告诉你一件事"）
    S2→S3: 告白内容从表面进入实质
    S3→S4: 到达最核心的秘密/最痛苦的记忆
    S4→S5: 硬切到倾听者——核心真相说出的瞬间
    S5→S6: 沉默后回到双人镜头——但景别/角度已改变（关系不同了）

  景别逻辑:
    MS → MCU → CU → ECU: 随真相深度递进，景别持续收窄
    这是 scale-in 的极端应用——观众被不可抗拒地拉入核心
    收窄速度 = 秘密的严重程度

  何时从 CU 回到 MASTER:
    1. 告白被打断（新角色/电话/外部事件）
    2. 告白者反悔/收回
    3. 关系平衡需要重建（新MASTER = 新关系）

  退出条件:
    (a) 秘密完全揭露 → S6 新平衡
    (b) 告白被中断 → 切到中断源 → 悬而未决
    (c) 倾听者拒绝接受 → 倾听者起身 → 退出到行动

  变体:
    A — 渐进告白: S0→S1→S2→S3→S4→S5→S6（经典完整弧）
    B — 爆发告白: 跳过S1-S2，从S0直接硬切到S3/S4
        （角色突然爆发，无铺垫 → 冲击力更强但缺乏准备）
    C — 反向告白: 先给S5(倾听者震惊反应) → 延迟 →
        再给S4(观众最后才知道说了什么) → Hitchcock延迟揭示

  反面示例（不应该这样做）:
    整个告白过程保持MS不变 → 景别不递进 = 情感不递进
    观众感觉不到秘密的重量在增加
    → 修复: 让景别跟随情感深度收窄

  参考:
    Daniel Arijon "Grammar of the Film Language" (1976) 6.10-6.14:
      对话场景中的情感升级通过景别收窄实现
    David Bordwell "The Way Hollywood Tells It" (2006):
      现代电影中告白场景的 ASL 递减模式
```

### Idiom 8: 独自调查（Solo Investigation）

```
入口条件:
  单个角色独自面对一个空间或一组物件，目的是搜索、检查、发现线索。
  角色有目的性（不是漫无目的地闲逛）。

状态机:

  [角色独自面对线索/空间] → 建立(LS/MS角色在空间中) →
    POV(角色视角扫描) → INSERT(线索特写) →
    CU(角色反应/思考) → POV(新方向) → INSERT(新线索) →
    CU(角色连接线索) → ... → 发现(ECU线索 + CU反应)

  状态图:
    S0 [进入]     → LS/MS, 角色进入调查空间
    S1 [扫描]     → POV(SLOW PAN/TRACK), 角色环视
    S2 [注意]     → POV停在某处 → INSERT(CU/ECU物件)
    S3 [处理]     → CU角色面部, 思考/困惑/联想
    S4 [深入]     → POV新方向/角色移动到新位置
    S5 [发现]     → ECU关键线索 + PUSH → CU角色(领悟/震惊)
    S6 [总结]     → MS角色在空间中(新姿态) 或 切出

  转换规则:
    S0→S1: 角色开始有目的地观察环境
    S1→S2: POV中某物引起注意（视觉异常/声音引导）
    S2→S3: INSERT提供了需要处理的信息
    S3→S4: 角色未满足，继续搜索（循环S1-S4可重复2-4次）
    S4→S5: 最终发现关键线索
    S5→S6: 角色完成处理，决定下一步行动

  POV使用规则:
    POV镜头 = 观众成为侦探 → 最大代入感
    POV扫描速度 = 角色的心理状态（慢=仔细, 快=焦虑）
    POV的起止方向暗示角色的注意力路径
    POV中的焦点变化（rack focus）= 注意力转移

  INSERT标点法:
    INSERT是独自调查的核心"标点符号"
    每个INSERT必须PROVIDES至少一个信息单元
    INSERT之间的CU反应 = 角色的推理过程（让观众跟上）
    INSERT频率递增 → 发现在加速
    INSERT时长递减 → 紧迫感增加

  退出条件:
    (a) 关键发现 → 切到角色决断 → 离开
    (b) 被打断 → 其他角色/声音/危险介入
    (c) 死胡同 → 角色挫败 → MS/LS角色在空间中（渺小感）

  变体:
    A — 系统搜索: S0→S1→S2→S3→S1→S2→S3→S5→S6
        （有条理地逐个检查，适合专业侦探）
    B — 直觉搜索: S0→S2→S3→S5（跳过扫描，角色直觉导向）
        （适合心理敏感型角色，如叶知秋）
    C — 恐惧搜索: S0→S1(HANDHELD)→S2→S3(角色退缩)→S1→S5
        （角色害怕发现真相，犹豫反复）

  反面示例:
    连续5个INSERT没有角色CU反应 → 观众不知道角色怎么想
    变成"物品幻灯片"而非"调查过程"
    → 修复: 每1-2个INSERT之后必须有角色反应CU

  参考:
    Arijon (1976) 7.1-7.6: POV序列的标准构型
    Bordwell & Thompson "Film Art" (2012) Ch.6:
      通过限制叙事信息到角色视角创造侦探效果
```

### Idiom 9: 内心独白可视化（Internal Monologue Visualization）

```
入口条件:
  角色刚接收到重要信息或经历重要事件，需要内在处理。
  外部动作暂停，叙事焦点转入角色的思维过程。

状态机:

  [角色需要内在处理] → CU(角色凝视/失焦) → VO开始 →
    环境空镜(承载VO) → INSERT(记忆/联想物) →
    MCU(角色在日常动作中) → VO继续 →
    环境空镜(情绪转变) → CU(角色表情变化——结论到达)

  状态图:
    S0 [触发]     → CU角色, 刚接收到需要消化的信息
    S1 [内转]     → CU→MCU, 角色眼神失焦/转向窗外/低头
                    VO开始（"我的第一反应是……"）
    S2 [漫游]     → 环境空镜序列(LS/MS), VO叠加
                    角色日常动作（泡茶/走路/看窗外）承载思考
    S3 [联想]     → INSERT回忆碎片/相关物件, VO连接
    S4 [转折]     → 环境空镜调性变化(光线/天气), VO达到转折
    S5 [结论]     → CU角色, 眼神重新聚焦, VO结束或最后一句
    S6 [行动]     → MS/MLS角色起身/做出决定 → 退出

  转换规则:
    S0→S1: 外部信息停止输入，角色进入内在处理模式
    S1→S2: VO成为声音轨道的主导（对白退出）
    S2→S3: VO中出现具体回忆/联想 → 视觉匹配
    S3→S4: VO推理到达转折点（"除非——"/"但如果——"）
    S4→S5: VO到达结论或新问题
    S5→S6: 内在处理完成，角色准备行动

  VO与画面的关系:
    同步: VO说"窗外的天暗了" → 画面是暗下来的窗
    对位: VO说"一切都很正常" → 画面是不正常的细节
    先行: VO还在说上一个想法 → 画面已经切到下一个联想
          （J-Cut——声音滞后于画面 → 思维的非线性）

  空镜选择规则:
    空镜不是随机的环境画面——它是角色内心状态的外化
    焦虑 → 被风吹动的窗帘/不停滴水的龙头/晃动的灯
    沉思 → 静止的茶杯/雨滴在玻璃上/黄昏的光线移动
    恐惧 → 空走廊/关闭的门/镜中的自己（不看镜头）
    领悟 → 光线突然变化/水面清澈/焦点锐化

  退出条件:
    (a) 角色做出决定 → S6行动
    (b) 外部打断（电话/敲门/声音）→ 硬切回现实
    (c) VO自然耗尽 → 角色没有结论 → 不安的沉默

  变体:
    A — 线性思考: S0→S1→S2→S5→S6（简单推理，快速结论）
    B — 螺旋思考: S0→S1→S2→S3→S2→S3→S4→S5（反复联想，
        适合复杂推理或纠结心理）
    C — 碎片闪回: S0→S1→S3*N(快闪INSERT)→S5
        （创伤记忆涌入，每个INSERT<=1秒，无VO纯视觉）

  反面示例:
    VO说了3分钟而画面只有一个不动的角色CU
    → "广播剧"效果——视觉信息为零
    → 修复: VO期间画面必须持续提供视觉信息（空镜/日常动作/INSERT）

  参考:
    Bordwell & Thompson "Film Art" (2012):
      Voice-over narration 与 mental subjectivity 的视觉化
    Thompson "Storytelling in Film and Television" (2003) Ch.4:
      内心独白的视觉锚定技术
```

### Idiom 10: 电话/远程对话（Phone Call / Remote Dialogue）

```
入口条件:
  角色接到或拨出电话/视频通话。
  关键特征: 物理分离——两方不在同一空间中。

状态机:

  [电话响起/发起] → INSERT(手机/电话) → CU(接听者) →
    MCU(接听者说话) → SIL(听对方说——观众只听到一侧) →
    CU(接听者反应) → MCU(接听者回应) → ... →
    CU(接听者挂断) → [反应/行动]

  状态图:
    S0 [触发]     → SFX电话铃 → INSERT手机/电话(CU)
    S1 [接听]     → CU接听者拿起电话/看屏幕 → 决定是否接
    S2 [对话]     → MCU接听者, 单侧对白
                    对方声音: (a)完全不可闻 (b)模糊可闻 (c)清晰可闻
    S3 [反应]     → CU接听者, 听到关键信息时的面部变化
                    这是此idiom的信息核心——观众从反应推断内容
    S4 [回应]     → MCU接听者说话, 提供信息（观众的理解来源）
    S5 [结束]     → CU挂断动作 / INSERT手机屏幕
    S6 [余波]     → CU接听者, 消化电话内容 → 决定行动

  转换规则:
    S0→S1: 铃声→角色注意到
    S1→S2: 角色接听
    S2→S3: 对方说了重要信息（观众从反应推断）
    S3→S4: 角色需要回应/追问
    S2-S4循环: 对话进行中，按信息重要性在MCU/CU间切换
    S4→S5: 对话结束
    S5→S6: 电话结束后的处理

  单侧信息规则（悬疑核心技巧）:
    观众只听到一侧 → 必须从反应推断另一侧
    角色的台词必须间接透露对方说了什么:
      "什么？他在哪里？" → 观众推断: 对方报告了某人的位置
      "不，那不可能。"  → 观众推断: 对方说了一个惊人的事实
    CU反应镜头的微表情 = 唯一的情报来源

  对方声音的三种处理:
    (a) 完全不可闻: 最大悬念，观众完全依赖反应推断
    (b) 模糊可闻: 观众能听到语调但不清楚内容 → 中等悬念
    (c) 清晰可闻: 无悬念，但可用于效率叙事（压缩时间）

  退出条件:
    (a) 正常挂断 → S5→S6
    (b) 被切断 → INSERT手机(断线画面) → CU角色(挫败/紧张)
    (c) 角色不接 → S1停留 → 铃声消失 → CU角色(回避的理由)

  变体:
    A — 经典单侧: 只看到一方，另一方完全不可见
        （悬疑标准——Lynch/Hitchcock/Fincher常用）
    B — 分屏/交切: 两方交替出现（各自MCU/CU），
        双方都可见但物理分离（适合两方都有重要反应时）
    C — 扬声器/外放: 电话开外放 → 房间内其他角色也能听到
        → 转化为"反应链"idiom（多人听到同一信息）

  反面示例:
    电话对话中保持MS全程不变，且对方声音完全清晰
    → 变成了"双人对话但只看到一个人" → 失去电话idiom的独特价值
    → 修复: 利用单侧性制造信息不对称；用CU反应替代对方台词

  参考:
    Arijon (1976) 5.8-5.12: 电话场景的镜头语法
    Bordwell (2006): 电话场景作为信息控制工具
```

### Idiom 11: 平行剪辑/交叉剪辑（Parallel Action / Cross-Cutting）

```
入口条件:
  两条（或更多）行动线在叙事时间上同时进行，
  需要交替展示以制造紧迫感或对比。

状态机:

  [两条行动线同时进行] →
    线A(MS/场景A) → 线B(MS/场景B) → 线A(MCU) → 线B(MCU) →
    线A(CU) → 线B(CU) → ... → 汇合/冲突

  状态图:
    S0 [建立双线]  → 线A建立镜头(LS) → 线B建立镜头(LS)
                     观众理解: 两件事同时在发生
    S1 [稳定交替]  → 线A(MS,3-5s) → 线B(MS,3-5s) → 线A → 线B
                     节奏平稳，建立两条线的内容
    S2 [加速]      → 切换频率增加: A(2-3s) → B(2-3s)
                     景别收窄: MS→MCU
    S3 [紧迫]      → 切换频率再增: A(1-2s) → B(1-2s)
                     景别: MCU→CU
    S4 [高潮]      → 极速切换: A(0.5-1s) → B(0.5-1s)
                     景别: CU→ECU
    S5 [汇合/碰撞] → 两条线合并到同一个镜头
                     或: 一条线的结果影响另一条线

  转换规则:
    S0→S1: 两条线都建立完毕，观众理解时间并行
    S1→S2: 紧迫感开始上升（倒计时/危险逼近）
    S2→S3: 紧迫感达到高位（"来不及了"）
    S3→S4: 紧迫感达到顶峰（最后几秒）
    S4→S5: 两条线在叙事上碰撞（成功/失败/错过）

  节奏数学:
    切换间隔的递减模式:
      S1: 4s-4s-4s-4s     (稳定)
      S2: 3s-3s-2s-2s     (加速)
      S3: 2s-1.5s-1s-1s   (紧迫)
      S4: 0.5s-0.5s-0.5s  (冲刺)
    观众的心率跟随切换频率 → Eisenstein度量蒙太奇原理

  线索管理:
    每条线在被切走时，应该留下一个RAISES
    （观众被切走时带着"那边怎么样了？"的焦虑）
    切回时，先回答上一个RAISES，再创造新的

  退出条件:
    (a) 两条线物理汇合 → 回到单线叙事
    (b) 一条线结束 → 专注另一条线
    (c) 两条线都到达结果 → 对比性MASTER或反应链

  变体:
    A — 经典追逐: 追者(线A) vs 被追者(线B)
        → 景别收窄 = 距离缩小
    B — 救援倒计时: 救援者(线A) vs 受害者(线B)
        → 线B的危险递增 = 线A的紧迫递增
    C — 对比蒙太奇: 线A(富人的晚餐) vs 线B(穷人的晚餐)
        → 不汇合，纯粹是Eisenstein理性蒙太奇（碰撞产生意义）
    D — 三线交叉: A→B→C→A→B→C → 更复杂但更高风险
        → 每条线必须视觉风格有区别（色调/景别/节奏）

  反面示例:
    两条线的切换频率始终不变（S1节奏维持到底）
    → 没有加速 = 没有紧迫感 = 观众感觉不到时间在收缩
    → 修复: 切换频率必须递增（这是此idiom的核心力学）

  参考:
    Griffith (1916) "Intolerance": 交叉剪辑的原型
    Eisenstein (1929): 度量蒙太奇理论基础
    Bordwell (2006) Ch.5: 现代电影中的加速交叉剪辑统计
```

### Idiom 12: 时间压缩蒙太奇（Time Compression Montage）

```
入口条件:
  叙事需要跳过一段时间（数小时到数年），
  但这段时间中的变化/积累对故事有意义，不能简单省略。

状态机:

  [需要压缩一段时间] → 时间标记A → 动作片段A(MS,短) →
    时间标记B → 动作片段B(MS,短) → ... →
    时间标记N → 结果镜头(CU/MS,长)

  状态图:
    S0 [进入]     → 建立"时间将要流逝"的信号
                    （叠化/日历/时钟/天光变化）
    S1 [片段序列] → 一系列短镜头(每个2-4秒)，每个代表一个时间点
                    内容: 同一动作的不同阶段/同一空间的不同时间
    S2 [变化积累] → 片段之间的视觉差异递增
                    （角色更疲惫/季节变化/物件堆积或消失）
    S3 [高潮片段] → 最后一个片段时长加倍(5-8秒)
                    景别变化(CU) → 这是蒙太奇要传达的"结果"
    S4 [回归实时] → 正常节奏恢复 → 观众理解时间跳跃已完成

  转换规则:
    S0→S1: 第一个时间标记出现（"三天后"/"Tuesday"/天光变化）
    S1→S2: 片段数>=3，变化模式确立
    S2→S3: 最后一个片段打破节奏（更长/更近/更安静）
    S3→S4: DISSOLVE或硬切回正常叙事节奏

  时间标记的类型:
    显式: 字幕("周一""周二""三个月后")
    隐式: 天光变化/季节变化/角色外貌变化
    物件: 烟灰缸从空到满/日历翻页/花从绽放到枯萎
    声音: 不同时间点的环境音变化（鸟鸣→蝉鸣→雨声）

  蒙太奇内部节奏:
    等长片段: 时间均匀流逝（日常感）
    递减片段: 时间加速（急迫/成长加速）
    递增片段: 时间减速（最后时刻来临）

  退出条件:
    (a) 到达目标时间点 → S4回归实时
    (b) 被事件打断 → 硬切回实时（紧急事件中断时间流逝）

  变体:
    A — 日常重复蒙太奇: 同一动作重复N次但每次有细微变化
        （起床/出门/工作 → 每次更疲惫/更焦虑/更偏执）
        适用: 《镜中访客》叶知秋的"空白夜晚"累积
    B — 技能成长蒙太奇: 角色练习某技能从失败到成功
    C — 恶化蒙太奇: 情况逐步恶化的压缩
        （线索越来越多/异常事件频率增加/角色精神状态下降）
    D — 对比蒙太奇: 时间压缩+平行剪辑的混合
        （角色A的日常 vs 角色B的日常，时间加速推进）

  反面示例:
    蒙太奇中每个片段都是5秒且景别相同(MS)
    → 没有节奏变化 = 没有时间加速感 = 只是一堆并列镜头
    → 修复: 片段时长必须有变化模式；景别应随时间推移而收窄

  参考:
    Eisenstein (1929) "度量蒙太奇": 通过绝对时长控制节奏
    Bordwell & Thompson "Film Art" (2012) Ch.7:
      蒙太奇序列的叙事功能分类
    Salt (2009): 蒙太奇序列的平均镜头时长统计
```

### Idiom 13: 群体扫描/集体动态（Crowd/Group Dynamics）

```
入口条件:
  3个或更多角色同时在场。
  需要展示群体的结构、权力关系、或从群体中隔离特定个体。

状态机:

  [多人同时在场] → ELS/LS(全组) → PAN/TRACK(扫描个体) →
    CU(个体A) → CU(个体B) → CU(个体C) →
    MS(权力中心) → 回到LS(群体反应) → ...

  状态图:
    S0 [全景]     → ELS/LS, 展示群体的整体存在和空间关系
    S1 [扫描]     → TRACK/PAN在群体中移动，逐一扫过个体
                    每个人停留0.5-1.5秒（足以建立存在但不深入）
    S2 [隔离]     → CU特定个体，从群体中"提取"出来
                    这个人因为某种原因值得关注
    S3 [权力中心] → MS/MCU群体中最重要的人物
                    所有人的视线/身体朝向指向此人
    S4 [群体反应] → 回到LS/ELS，看群体对权力中心的集体反应
    S5 [裂隙]     → CU某个与群体反应不同的个体
                    （此人是重要角色/嫌疑人/异端）

  转换规则:
    S0→S1: 需要让观众认识群体中的个体
    S1→S2: 扫描中某人引起特别注意（表情/动作异常）
    S2→S3: 从个体聚焦到群体核心人物
    S3→S4: 权力中心做出行动/声明 → 看群体反应
    S4→S5: 群体反应中有一个人不同 → 隔离此人

  权力映射规则:
    空间位置 = 权力关系
      中心 → 权力者      边缘 → 边缘人
      高位 → 支配者      低位 → 服从者
      前方 → 领导者      后方 → 追随者/隐藏者
    摄影机向谁PUSH = 谁是此刻的权力焦点

  退出条件:
    (a) 群体解散 → 各自行动
    (b) 焦点锁定到1-2个人 → 转入正反打或其他双人idiom
    (c) 群体事件发生 → 转入反应链

  变体:
    A — 会议/审讯室: S0→S3→S4→S5（快速聚焦权力中心和异端）
    B — 人群中寻人: S0→S1→S1→S2（扫描→锁定目标）
    C — 集体沉默: S0→S2→S2→S2→S4（逐个隔离看每人的沉默反应）

  反面示例:
    群体场景只用一个LS拍到底，不隔离任何个体
    → 观众不知道该看谁 → 信息过载 → 谁都没看清
    → 修复: 扫描→隔离→回到全景，给观众一个注意力路径

  参考:
    Arijon (1976) Ch.8: 多人场景的镜头分配
    Bordwell (2006): 群体场景中的注意力引导统计
```

### Idiom 14: 门槛跨越（Threshold Crossing）

```
入口条件:
  角色即将进入或离开一个具有叙事意义的空间。
  跨越不仅是物理移动，更是心理/叙事状态的转变。

状态机:

  [角色即将进入/离开重要空间] →
    MS(角色在门前/边界处) → 停顿(CU角色犹豫/决心) →
    动作(角色跨越门槛) → 新空间建立(LS/ELS新环境) →
    MS(角色在新空间中——已经不同了)

  状态图:
    S0 [接近]     → MS/MLS角色走向门/边界/入口
                    TRACK跟随或STATIC等待
    S1 [门槛]     → MS角色站在门口/边界处
                    物理门槛 = 心理门槛的视觉隐喻
    S2 [犹豫]     → CU角色面部(决断/恐惧/期待)
                    可选: INSERT门把手/钥匙/门牌
    S3 [跨越]     → 角色打开门/走过边界
                    运动方式表达心理状态（推门/犹豫地推/一脚踹开）
    S4 [新世界]   → LS/ELS新空间（从角色POV或全知视角）
                    新空间的视觉特征 = 新状态的隐喻
    S5 [着陆]     → MS角色在新空间中
                    角色的姿态已经改变（更紧张/更放松/更警觉）

  转换规则:
    S0→S1: 角色到达物理/心理边界
    S1→S2: 角色停下（即使0.5秒的停顿也有效）
    S2→S3: 决定跨越
    S3→S4: 跨越完成 → 新空间展开
    S4→S5: 角色在新空间中定位

  门槛的叙事意义:
    Joseph Campbell "英雄之旅": 门槛 = 从已知世界到未知世界
    每次跨越门槛，角色都应该有微妙的变化
    进入 → 角色面对新挑战    离开 → 角色带着新信息/创伤

  门槛的视觉处理:
    门框 = 天然的画面框架（frame within frame）
    角色在门框中 = 被限制/被观察/处于过渡中
    从亮处进入暗处 = 进入未知/危险
    从暗处进入亮处 = 发现/解脱/幻灭

  退出条件:
    (a) 角色完全进入新空间 → 转入新空间的建立序列
    (b) 角色拒绝跨越 → S2停留 → 退回 → 未能改变
    (c) 门被关上/锁上 → 困境建立

  变体:
    A — 主动跨越: S0→S1→S2(决心)→S3→S4→S5（经典英雄门槛）
    B — 被动跨越: S0→S3(被推/被拉/不由自主)→S4(陌生环境冲击)
        （角色没有选择 → 被命运推入新状态）
    C — 回归跨越: 同一个门槛的反向穿越——但角色已经不同
        （第二次经过同一个门 → 景别/角度/光线都变了
         → 角色成长/退化的视觉证据）
    D — 拒绝门槛: S0→S1→S2→S1(退缩)→S0(离开)
        （角色没有准备好 → 未来必须再次面对）

  反面示例:
    角色从一个空间走到另一个空间，完全没有门槛停顿
    → 空间转换没有心理重量 → 只是换了个背景
    → 修复: 即使是短暂的停顿(0.5s CU)也能赋予跨越以意义

  参考:
    Campbell "The Hero with a Thousand Faces" (1949):
      门槛守护者原型与空间过渡
    Bordwell & Thompson (2012): 空间过渡的剪辑语法
```

### Idiom 15: 异常发现/不对劲（Anomaly Discovery）

```
入口条件:
  角色在日常环境中遇到不正常的细节。
  异常足够微妙，角色可以选择合理化（忽略）或警觉（调查）。

状态机:

  [日常场景中有异常] → MS(角色在正常活动中) →
    角色停顿(微妙的注意) → POV(角色看到异常) →
    INSERT(异常物件/细节 — CU) → CU(角色困惑) →
    角色合理化("大概是……吧") 或 角色警觉 →
    MS(继续日常/开始调查)

  状态图:
    S0 [日常]     → MS/MCU角色在日常动作中（穿鞋/泡茶/整理）
                    节奏正常，无异常信号
    S1 [微停]     → 角色的动作出现0.5-1秒的停顿
                    运动中断 → 观众潜意识注意到"怎么了？"
    S2 [注视]     → POV或OTS，角色看向异常源
                    摄影机模拟角色的注意力转移
    S3 [异常]     → INSERT/CU异常物件或细节
                    必须足够微妙以至于角色可以合理化
                    但足够异常以至于观众会记住
    S4 [处理]     → CU角色面部——困惑/微微皱眉
                    两条分支:
                      → S5a [合理化] 角色自我说服（"可能是……"）
                      → S5b [警觉] 角色开始重视（稀有，通常在后期章节）
    S5a [合理化]  → MS角色继续日常（但观众已经被种下不安种子）
    S5b [警觉]    → 角色进入"独自调查"idiom

  转换规则:
    S0→S1: 角色感官捕捉到异常（视觉/触觉/嗅觉）
    S1→S2: 注意力从日常动作转向异常源
    S2→S3: 异常被正式展示给观众
    S3→S4: 角色内在处理
    S4→S5a: 角色选择忽略（不可靠叙述者的自我保护）
    S4→S5b: 角色选择面对（通常在故事后半段才发生）

  悬疑核心技巧——"公平性植入":
    异常的呈现必须满足公平性协议:
      观众看到了异常(S3的INSERT) → 公平: 信息已展示
      角色合理化了异常(S5a) → 观众可能跟随角色而忽略
      但二次阅读/观看时 → 这个INSERT会变得"毛骨悚然"

  INSERT的微妙度梯度:
    早期章节: 异常极微妙（脚底的灰、不认识的钢笔）
    中期章节: 异常渐增（位置移动的物品、无法解释的痕迹）
    晚期章节: 异常不可忽视（直接矛盾的证据）

  退出条件:
    (a) 角色合理化 → 回到日常(S5a) → 观众不安但故事继续
    (b) 角色警觉 → 转入独自调查idiom(S5b)
    (c) 异常自行消失 → 更深的不安（是幻觉还是有人在清除证据？）

  变体:
    A — 单次异常: S0→S1→S2→S3→S4→S5a（经典植入）
    B — 累积异常: S0→S1→S3→S5a → S0→S1→S3→S5a → ...
        （同一场景中多个异常依次出现，每个都被合理化——但累积效应
        让观众越来越不安。《镜中访客》ch1的结构就是这个变体。）
    C — 双视角异常: S3时摄影机比角色看到更多
        （INSERT展示异常时，画面边缘有角色没注意到的第二个异常
        → 观众知道的比角色多 → Hitchcock悬念）

  反面示例:
    异常被呈现为 LS 中的一个微小细节且无INSERT特写
    → 观众完全没看到 → 不是"微妙的线索"而是"不存在的线索"
    → 修复: 再微妙的线索也必须有CU/INSERT让观众有机会看到
    （公平性的底线: 信息必须被呈现，即使可以被忽略）

  参考:
    Hitchcock: "让观众看到炸弹"——即使角色不知道，观众必须被给予信息
    Bordwell (1985): 限知叙事中的线索植入策略
```

### Idiom 16: 沉默对峙/张力保持（Silent Standoff / Tension Hold）

```
入口条件:
  两人或多人面对面，对话突然终止或从未开始。
  沉默本身成为叙事动作——谁先说话、谁坚持更久是核心。

状态机:

  [两人/多人面对面，无人说话] →
    MASTER(MS双人) → CU-A(等待) → CU-B(等待) →
    INSERT(环境细节: 时钟/水滴/风) →
    CU-A(微变化) → CU-B(微变化) →
    → 持续直到有人打破沉默或采取行动

  状态图:
    S0 [触发]     → 某个事件/台词导致双方沉默
                    最后一句话的回声消失 → 沉默开始
    S1 [对峙建立] → MASTER(MS或OTS), 两人的物理关系可见
                    空间距离 = 心理距离
    S2 [交替注视] → CU-A → CU-B → CU-A → CU-B
                    每个CU比前一个稍长（时间在膨胀）
                    面部微表情是唯一的"对白"
    S3 [环境间歇] → INSERT环境细节(时钟秒针/茶杯蒸汽/窗外风声)
                    功能: (a)给观众呼吸空间 (b)时间感可视化
                    (c)隐喻（水快烧开 = 张力快到极限）
    S4 [临界点]   → CU中出现明显变化（眼神移开/手指颤抖/嘴唇微动）
                    即将打破沉默的物理信号
    S5 [打破]     → 台词/动作/外部事件打破沉默
                    打破方式定义权力关系:
                      先说话的人 = 承受不住压力的人
                      外部打断 = 两人都在承受

  转换规则:
    S0→S1: 对话终止/事件发生 → 沉默降临
    S1→S2: MASTER已建立空间 → 进入面部特写交替
    S2→S3: 需要呼吸/需要展示时间流逝
    S3→S2: 回到面部（沉默还在继续）
    S2/S3→S4: 某人的微表情出现变化
    S4→S5: 沉默被打破

  时长规则:
    有效沉默的最小时长: 3秒（低于3秒只是"停顿"不是"沉默"）
    有效沉默的最大时长: 15-20秒（超过则观众从紧张转为困惑）
    最佳范围: 5-12秒

  退出条件:
    (a) 一方先开口 → 回到对话idiom（但权力已经转移）
    (b) 一方离开 → 门槛跨越idiom
    (c) 外部打断（电话/敲门/噪声）→ 张力被外力释放
    (d) 暴力爆发 → 追逐或冲突序列

  变体:
    A — 审讯沉默: 审讯者沉默 = 施压技巧（LOW ANGLE审讯者CU）
    B — 对等沉默: 两人都在等对方先说 → 心理博弈
    C — 悲伤沉默: 两人都知道真相但不愿说出 → CU中有泪光/颤抖
        （不是对峙而是共同承受）

  反面示例:
    沉默中只有一个角色的CU持续8秒不切
    → 不是"沉默对峙"而是"镜头忘了切"
    → 修复: 沉默中也需要切——切到另一个人、切到环境、回到MASTER
    每个切都在说"沉默还在继续，而且越来越难以承受"

  参考:
    Sergio Leone: 对峙沉默的极端时长运用
    Arijon (1976) 6.20-6.24: 沉默场景的镜头节奏
    Murch (2001): 沉默作为剪辑的"负空间"
```

### Idiom 17: 镜像/反射序列（Mirror/Reflection Sequence）

```
入口条件:
  角色面对镜子、水面、玻璃或其他反射面。
  反射不仅是物理现象——它是角色面对自我的视觉隐喻。

状态机:

  [角色面对镜子/反射面] →
    MS(角色接近镜子) → CU(镜中的脸) →
    INSERT(镜中看到不同于预期的东西) →
    CU(真实的脸 vs 镜中的脸) → 反应

  状态图:
    S0 [接近]     → MS角色走向镜子/反射面（水面/玻璃/屏幕）
    S1 [对视]     → CU镜中的角色面部
                    关键: 观众最初不确定看到的是"真实"还是"反射"
    S2 [差异]     → 镜中的反射与真实的角色出现差异
                    (a) 表情不同（镜中在笑但角色没笑）
                    (b) 动作延迟（镜中的反应比真实慢半拍）
                    (c) 细节不同（镜中看到的背景里有什么/没有什么）
                    (d) 纯粹的自我审视（角色在镜中看到自己不认识的面孔）
    S3 [质疑]     → CU角色真实面部 → 困惑/恐惧/自我怀疑
    S4 [确认/否认]→ 再次看镜子:
                    (a) 一切正常（是幻觉？）→ 不安留存
                    (b) 差异仍在 → 恐惧升级
                    (c) 镜子碎裂/角色转身离开 → 逃避

  转换规则:
    S0→S1: 角色的目光落到反射面上
    S1→S2: 反射出现异常（或角色在反射中看到不想面对的自己）
    S2→S3: 角色意识到异常
    S3→S4: 角色决定再次确认或逃避

  心理惊悚核心用法:
    镜子 = 自我对话的物理化
    角色看镜子 = 面对自己的另一面
    《镜中访客》特殊意义: "访客"就是镜中的自己
      → 镜像序列在此作品中是核心视觉母题
      → 每次镜像序列的差异程度 = 人格分裂的程度

  摄影机位置规则:
    拍镜中的脸: 摄影机正对镜子（可能看到摄影机 → 通常避免）
    拍真实的脸看着镜子: 摄影机在角色身后/侧面
    OTS看镜子: 角色的肩膀在前景，镜中的脸在焦点
    → OTS是最安全也最有效的选择（同时看到"看"和"被看"）

  退出条件:
    (a) 角色离开镜子 → 回到正常场景
    (b) 镜中出现明确的恐怖元素 → 跳吓/揭示
    (c) 角色摧毁镜子 → 拒绝面对自己 → 碎片中看到多个自己

  变体:
    A — 日常镜像: 简短的S0→S1(正常)→退出，但角色注视自己的
        时间"稍长了一点" → 微妙的不安信号
    B — 对话镜像: 角色在镜中与自己对话（S1时VO开始，
        镜中的嘴型与VO不同步 → 另一个人格在说话）
    C — 渐变镜像: 镜中的差异随故事进展加剧
        ch1: 多看了一眼   ch3: 表情不同
        ch5: 动作不同     ch7: 完全是另一个人

  反面示例:
    镜像场景中摄影机直接正对镜子 → 观众看到剧组的反射
    → 打破叙事幻觉
    → 修复: 使用OTS角度或微微偏转摄影机避开自身反射

  参考:
    Lacan: 镜像阶段理论——自我认同通过镜像构建
    Lynch "Lost Highway" / "Mulholland Drive": 镜像作为人格分裂的视觉符号
    Aronofsky "Black Swan": 镜像序列的渐变恐怖
```

### Idiom 18: 尾随/被跟踪序列（Being Followed / Tail Sequence）

```
入口条件:
  角色在移动中（步行/驾车/公共交通），感觉或确实被跟踪。
  核心张力来自不确定性——是真的还是偏执？

状态机:

  [角色感觉/确实被跟踪] →
    MS(角色正常行走) → 角色回头(OTS/POV) →
    LS(身后的街道——有人？没有？不确定) →
    MS(角色加快步伐) → 声音线索(SFX脚步声) →
    角色再次回头(CU恐惧 + POV) → LS(跟踪者的暗影/空无一人) →
    → 到达安全处/对峙/逃脱

  状态图:
    S0 [正常]     → MS/MLS角色行走，正常节奏
    S1 [第一信号] → 声音(SFX脚步/异响) 或 视觉(影子/反射)
                    角色微微停顿但继续走
    S2 [回头]     → OTS/POV角色回头看
                    → 身后画面: 空的/模糊的人影/明确的人
    S3 [加速]     → MS角色加快步伐，HANDHELD增加不稳定
                    节奏加快（步伐声频率增加）
    S4 [确认]     → CU角色恐惧 + 再次POV回头
                    → 跟踪者更近了/更明确了
    S5 [逃跑/对峙]→ 角色开始跑(TRACK跟拍) 或 角色停下面对
    S6 [结果]     → 到达安全处(门内) 或 对峙 或 跟踪者消失

  转换规则:
    S0→S1: 第一个异常信号（通常是声音——J-Cut先行）
    S1→S2: 信号足以让角色回头
    S2→S3: 回头的结果是不确定的（最大张力）
    S3→S4: 第二个信号，更强
    S4→S5: 角色的恐惧达到行动阈值
    S5→S6: 情况解决（暂时或永久）

  跟踪者可见度梯度:
    Level 0: 完全不可见（只有声音/角色的感觉）→ 最大不确定性
    Level 1: 模糊存在（远处的暗影/人群中的帽衫）
             → "是跟踪者还是路人？"
    Level 2: 明确存在但面目不清（帽衫+口罩/逆光剪影）
             → 确认被跟踪但不知是谁
    Level 3: 面目可见（观众看到跟踪者是谁）
             → 但角色可能没看到

  声画配合:
    脚步声先于画面出现（J-Cut）→ 角色先听到再看到
    环境音在S1后逐渐减少 → 世界缩小到角色和威胁之间
    角色的呼吸声在S3-S4中变得可闻 → 恐惧的声音化

  退出条件:
    (a) 角色进入安全空间(门槛跨越idiom) → 但跟踪者是否还在外面？
    (b) 对峙 → 转入正反打或沉默对峙
    (c) 跟踪者消失 → 最不安的结果（"是幻觉还是真的？"）
    (d) 第三方出现 → 社交安全（跟踪者撤退/角色被保护）

  变体:
    A — 主观体验: 全程角色POV + HANDHELD → 观众体验恐惧
    B — 上帝视角: 交替角色MS和跟踪者POV → 观众比角色知道更多
        （Hitchcock: 知道危险在那里但角色不知道 = 悬念）
    C — 虚假跟踪: 整个序列执行后S6揭示"跟踪者"其实是路人/巧合
        → 角色的偏执被展示 → 但观众也被骗了 → 不可靠叙述

  反面示例:
    跟踪者在S1就完全可见(LS明确展示跟踪者)
    → 之后反复切CU角色恐惧
    → 既然已经看到了，不确定性消失
    → 张力模式错误（这不是悬念而是恐惧）
    → 修复: 如果要制造悬念，跟踪者应该保持模糊
    → 如果要制造恐惧（观众和角色都知道），切换到追逐idiom

  参考:
    Hitchcock "Vertigo": 尾随序列的视觉语法典范
    Fincher "Zodiac": 被跟踪的主观恐惧构建
    Arijon (1976) 7.10-7.14: 追踪与被追踪的镜头逻辑
```

### 惯用语决策树（Idiom Decision Tree）

给定一个场景，按以下决策树选择首要惯用语:

```
这个场景中有几个角色在场？
|
+-- 0个角色（纯环境）
|   -> 内心独白可视化(Idiom 9) 配VO
|   -> 或 时间压缩蒙太奇(Idiom 12)
|
+-- 1个角色
|   +-- 角色在新空间？ -> 建立序列(Idiom 1) + 门槛跨越(Idiom 14)
|   +-- 角色在搜索/调查？ -> 独自调查(Idiom 8)
|   +-- 角色在日常中发现异常？ -> 异常发现(Idiom 15)
|   +-- 角色在思考/处理信息？ -> 内心独白可视化(Idiom 9)
|   +-- 角色在接电话？ -> 电话对话(Idiom 10)
|   +-- 角色面对镜子？ -> 镜像序列(Idiom 17)
|   +-- 角色感觉被跟踪？ -> 尾随序列(Idiom 18)
|   +-- 角色在时间流逝中重复动作？ -> 时间压缩蒙太奇(Idiom 12)
|
+-- 2个角色
|   +-- 在对话？
|   |   +-- 一方揭露秘密？ -> 告白对话(Idiom 7)
|   |   +-- 正常信息交换？ -> 正反打(Idiom 2)
|   |   +-- 沉默对峙？ -> 沉默对峙(Idiom 16)
|   +-- 一方在追另一方？ -> 追逐(Idiom 4)
|   +-- 一方在监视另一方？ -> 窥视(Idiom 5)
|   +-- 一方在跟踪另一方？ -> 尾随(Idiom 18)从被跟踪者视角
|
+-- 3+个角色
|   +-- 群体中的个体反应？ -> 反应链(Idiom 6)
|   +-- 群体动态/权力关系？ -> 群体扫描(Idiom 13)
|   +-- 两个子群在对话？ -> 正反打(Idiom 2)变体
|
+-- 跨时空
    +-- 两个同时进行的事件？ -> 平行剪辑(Idiom 11)
    +-- 压缩时间段？ -> 时间压缩蒙太奇(Idiom 12)

首次进入任何场景 -> 建立序列(Idiom 1)优先叠加
场景结束有空间转换 -> 门槛跨越(Idiom 14)封尾
```

### 惯用语优先级规则

当多个idiom适用于同一段落时，按以下优先级决定"外层"idiom:

```
优先级（高 -> 低）:
  1. 建立序列(Idiom 1) — 始终在场景开头执行，不可省略于首次空间
  2. 平行剪辑(Idiom 11) — 如果在交叉剪辑中，整体结构由它控制
  3. 场景核心idiom — 由决策树确定的主要idiom
  4. 门槛跨越(Idiom 14) — 空间转换时叠加
  5. 揭示序列(Idiom 3) — 当信息揭露是核心叙事动作时

优先级不是排斥——高优先级idiom可以包含低优先级idiom作为子序列
```

### 复合惯用语记法（Compound Idiom Notation）

实际场景经常需要多个idiom嵌套或衔接。用以下记法表达:

```
记法规则:

  顺序衔接: A -> B
    含义: idiom A完成后，过渡到idiom B
    示例: Establishing -> Shot-Reverse-Shot
          (建立空间后进入对话)

  嵌套包含: A [ B ]
    含义: idiom B作为idiom A的一个内部阶段执行
    示例: Confession [ Reveal(delayed) ]
          (告白对话中嵌套了一个延迟揭示)

  并行执行: A || B
    含义: 两个idiom在交叉剪辑框架中同时进行
    示例: SoloInvestigation || BeingFollowed
          (角色在调查的同时被跟踪——两个视角交替)

  中断切换: A >> B
    含义: idiom A被中断，强制切换到idiom B
    示例: InternalMonologue >> PhoneCall
          (内心独白被电话打断)

  循环: A *N
    含义: idiom A执行N次（通常带有变化）
    示例: AnomalyDiscovery *3 -> InternalMonologue
          (三次异常发现后进入内心处理)

  反idiom: !A
    含义: 故意违反idiom A的规则（必须有叙事理由）
    示例: !Establishing(reversed)
          (从CU开始逆向拉到ELS——延迟揭示空间)

完整场景标注示例:

  《镜中访客》ch1 咨询室场景:
    Establishing -> ShotReverseShot [ Confession(variant-A) ] ->
    Reveal(progressive) -> ReactionChain(minimal, 2人)

  《镜中访客》ch1 回家场景:
    ThresholdCrossing -> AnomalyDiscovery *2 ->
    InternalMonologue(variant-A) -> AnomalyDiscovery *1
```

### 《镜中访客》第一章惯用语标注

以下是ch1每个叙事段落的完整惯用语覆盖:


## 四、认知连贯性法则（神经科学验证）

以下规则不仅是"传统做法"，而是经过认知科学实验验证的知觉原理。

> **深度扩展**: 认知知觉系统的完整形式化规则（注意力聚光灯、认知负荷限制、
> 情感启动规则、注意力同步、Kuleshov 效应形式化、1/f 时长分布、注视预测模型、
> 知觉桥接规则）已独立为 `cognitive-perception.md`。
> 该文件包含 10 个神经科学来源、40+ 条 IF/THEN 规则、7 个可测试预测。
> 本节保留核心法则摘要；完整规则集和决策树请参照 cognitive-perception.md。

### 法则 1: 动作连贯优先（Action Continuity Primacy）

```
来源: Magliano et al., Cognitive Science, 2011

发现: 观众的事件分割（感知到"新事件开始了"）
  主要由动作的变化驱动，而非空间或时间的变化。

应用:
  ✓ 在动作进行中切 → 观众将两个镜头感知为同一事件
  ✓ 在动作完成时切 → 观众感知到事件边界（可以用于场景转换）
  ✗ 在动作完成后的静止中切 → 观众已经认为事件结束了，
     新镜头开始一个新事件

实践规则:
  同一事件内的镜头 → 在动作中切（匹配剪辑）
  跨事件的镜头 → 在动作完成后切（利用自然边界）
```

### 法则 2: 景别缩进无感，景别放大有感（Scale Asymmetry）

```
来源: Liao et al., Frontiers in Neuroscience, 2023

发现: ERP实验证明
  Scale-in（ELS→CU方向）: N300/N400波幅降低 → 观众几乎不注意到切
  Scale-out（CU→ELS方向）: N300/N400波幅增加 → 观众明确意识到切

应用:
  逐步推近（场景由远到近）→ 流畅、沉浸、无意识地深入
  突然拉远（从近景到远景）→ 明确的"打断"、揭示全貌、呼吸

实践规则:
  建立沉浸感时: 使用 scale-in 方向（ELS→LS→MS→CU）
  需要制造冲击时: 使用 scale-out 方向（CU→ELS，突然）
  "正常"的场景内推进: 总体 scale-in，偶尔 scale-out 做呼吸点
```

### 法则 3: 角度变化不触发事件感知

```
来源: Liao et al., Frontiers in Neuroscience, 2023

发现: 仅改变摄影机角度（不改变景别）
  不会产生显著的神经反应差异。

应用:
  纯角度变化（平视→俯拍）是"安静"的变化
  可以用来微妙地改变权力关系而不惊动观众
  配合景别变化时效果更显著

实践规则:
  需要微妙的权力转移: 只改变角度，不改变景别
  需要明确的权力转移: 同时改变角度和景别
  需要最大化冲击: 改变景别+角度+运动（三维突变）
```

### 法则 4: 运动跨切连续（Motion Carries）

```
来源: Continuity editing 经典规则 + 认知验证

发现: 当画面在运动中切，观众的视线追踪会自动延续到
  新画面中相同方向的运动 → 切本身"消失"了。

应用:
  运动中切到运动 → 最流畅的切（观众的眼睛一直在追踪）
  静止中切到静止 → 最明显的切（观众有机会"看到"切）
  运动中切到静止 → 急停效果（用于惊吓/强调）
  静止中切到运动 → 启动效果（用于行动开始）

实践规则:
  追逐场景: 始终在运动中切 → 无缝加速感
  审讯场景: 在静止中切 → 每个切都是一次"击打"
  揭示场景: 运动→静止 → 运动停止的瞬间就是揭示
```

## 五、形式化谓词表（Computable Predicate System）

> v2.1 核心升级: 所有验证规则编号为 P01-P20，
> 每个谓词有明确的 IF/THEN 形式，返回 PASS/FAIL/WARN。
> LLM 执行时应逐条检查，不得跳过或合并。
> 交叉引用: decomposition-algorithm.md VERIFY 步骤应引用此表。
> 交叉引用: evaluation.md LOGIC 维度的评分基于通过的谓词数量。

### 谓词命名规则

```
P01-P04: 切理由谓词（为什么切？）
P05-P10: 依赖谓词（信息流是否完整？）
P11-P14: 位置谓词（顺序是否正确？）
P15-P18: 参数谓词（镜头参数是否合理？）
P19-P20: 序列谓词（整体是否连贯？）
```

### Level 1: 切理由谓词（P01-P04）

```
P01 — INFORMATION CUT PREDICATE
  IF 镜头 S[n] 引入了至少一个信息单元 I，使得:
    (a) I 在 S[1]...S[n-1] 的任何通道 PROVIDES 中均未出现，且
    (b) I 所在的空间/角度在 S[n-1] 的取景范围外
  THEN REASON 包含 INFORMATION → PASS
  ELSE 该切不以 INFORMATION 为理由 → 继续检查其他理由

  Mirror Visitor 示例:
    S2 (林小曼进门): 林小曼是新角色，S1 未包含她 → P01 PASS
    反例: 如果 S1 已经是包含门口的宽景，林小曼进入时仍在画面内
          → 不需要切 → 延长 S1 即可

P02 — REACTION CUT PREDICATE
  IF 镜头 S[n-1] 或 S[n-k] (k≤3) 中发生了事件 E，且
    镜头 S[n] 展示了角色 C 对 E 的反应 R，使得:
    (a) R 携带叙事信息（不是空洞的"看了一眼"），且
    (b) C 在 S[n-1] 中不可见，或可见但景别不足以展示 R 的细节
  THEN REASON 包含 REACTION → PASS
  ELSE 该切不以 REACTION 为理由

  Mirror Visitor 示例:
    S5 (叶知秋反应): E = S4 的"有人跟踪我"，C = 叶知秋,
    R = 专业面具下的警觉，S4 是林小曼 CU → 叶知秋不可见 → P02 PASS
    反例: 如果 S4 是双人 MS 且叶知秋的反应已经可见 → P02 FAIL
          → 不需要单独切到叶知秋，除非需要更多细节 (→ 检查 P03)

P03 — EMPHASIS CUT PREDICATE
  IF 以下任一条件成立:
    (a) S[n-1] 到 S[n] 的叙事节拍重力变化 |ΔG| ≥ 2，且
        S[n] 的景别与 S[n-1] 不同（差 ≥ 1 级），或
    (b) 存在元素 E 在 S[n-1] 中可见但不突出，
        S[n] 通过更紧的景别使 E 成为视觉焦点
  THEN REASON 包含 EMPHASIS → PASS
  ELSE 该切不以 EMPHASIS 为理由

  Mirror Visitor 示例:
    S3→S4 (MCU→CU): 重力从 2 升到 4，ΔG = 2，景别推近 1 级 → P03 PASS
    反例: S3→S4 如果景别不变（都是 MCU），那这不是 EMPHASIS
          → 需要其他理由（P01 或 P02）

P04 — ORIENTATION CUT PREDICATE [v2.1 新增]
  IF 以下任一条件成立:
    (a) 角色在 S[n-1] 中移动，导致 S[n-1] 的取景范围无法展示
        角色之间的新空间关系，且 S[n] 使用更宽景别展示新空间格局
    (b) 新角色在 S[n-1] 到 S[n] 之间进入场景，且
        S[n] 展示新角色与现有角色的空间关系
    (c) 连续 ≥ 3 个 CU/ECU 镜头后，S[n] 回到 MS 或更宽景别
        重新建立空间定位
    (d) 时间跳跃后同一空间状态变化，S[n] 展示变化后的空间
  AND S[n] 不满足 P01 (没有叙事新信息)
  AND S[n] 不满足 P02 (不是反应)
  AND S[n] 不满足 P03 (不改变情感距离)
  THEN REASON = ORIENTATION → PASS
  ELSE 如果 S[n] 同时满足 P01/P02/P03 中至少一个 → 那不是纯 ORIENTATION
       (可以是复合理由，如 INFORMATION + ORIENTATION)

  ORIENTATION 频率约束:
    一个场景内，ORIENTATION 切占比应 ≤ 25%
    IF ORIENTATION 切 > 总切数 × 0.25 THEN WARN（可能过度重定向）

  Mirror Visitor 示例:
    假设在 S5-S6 之间，林小曼从沙发上站起来走到窗边
    → 需要一个 MS/MLS 展示她的新位置 → P04(a) PASS
    但在实际 ch1 中，两人始终坐着不动 → P04 在此场景不触发

P00 — CUT REASON COVERAGE (元谓词)
  FOR EACH 切 (S[n-1] → S[n]):
    IF P01 OR P02 OR P03 OR P04 → PASS (至少一个理由成立)
    ELSE → FAIL (无因之切)
  METRIC: cut_reason_coverage = PASS_count / total_cuts
  TARGET: = 1.0
```

### Level 2: 依赖谓词（P05-P10）

```
P05 — REQUIRES SATISFACTION PREDICATE
  FOR EACH 镜头 S[n], FOR EACH 通道 C ∈ {V, A, N}:
    FOR EACH 条目 R 在 S[n].C-REQUIRES 中:
      IF ∃ S[k] (k < n) 使得 R ∈ S[k].C-PROVIDES
         OR R ∈ S[k].C'-PROVIDES (跨通道: V↔A 可互相满足)
      THEN PASS
      ELSE FAIL — 信息缺口: S[n] 需要 R 但没有前置镜头提供
  特殊规则:
    N-REQUIRES 只能被 N-PROVIDES 满足，或被 V-PROVIDES + A-PROVIDES
    共同满足（即观众从看到的+听到的中推断出潜文本）

  Mirror Visitor 示例:
    S4.A-REQUIRES = "Shot 3 的沉默" → S3.A-PROVIDES = "沉默" → PASS
    反例: 如果删除 S1 直接从 S2 开始 → S2.V-REQUIRES = "空间定位"
          → 没有前置镜头提供 → P05 FAIL

P06 — PROVIDES NON-EMPTY PREDICATE
  FOR EACH 镜头 S[n]:
    IF S[n].V-PROVIDES 非空 OR S[n].A-PROVIDES 非空 OR S[n].N-PROVIDES 非空
    THEN PASS
    ELSE FAIL — 空转镜头，无存在理由
  TARGET: = 1.0 (100% 通过)

P07 — RAISES CONTINUITY PREDICATE
  FOR EACH 镜头 S[n]:
    LET active_questions = Σ(S[1]...S[n] 所有通道的 RAISES)
                         - Σ(S[1]...S[n] 所有通道回答的 RAISES)
    IF active_questions ≥ 1 THEN PASS
    ELSE WARN — 叙事死点（所有悬念都被解答且无新悬念）
  注意: WARN 而非 FAIL，因为场景结束前的最后一个镜头
        可以合法地达到 active_questions = 0（但极少见）

P08 — DAG ACYCLICITY PREDICATE
  构建依赖图 G = (S, E)，其中:
    S = 所有镜头
    E = { (S[i], S[j]) | S[j] 的某个 REQUIRES 被 S[i] 的 PROVIDES 满足 }
  IF G 是 DAG (无环)
  THEN PASS
  ELSE FAIL — 循环依赖
  实践: 循环依赖在线性叙事中几乎不应出现。
        在闪回结构中，闪回镜头应被标注为独立子图。

P09 — INFO DENSITY PREDICATE
  FOR EACH 镜头 S[n]:
    LET ID = |V-PROVIDES| + |A-PROVIDES| + |N-PROVIDES|
    IF ID ≤ 5 THEN PASS
    ELIF ID ≤ 7 AND S[n] 是建立镜头 THEN PASS (建立镜头豁免)
    ELSE WARN — 信息过载，考虑拆分

P10 — CHANNEL CONVERGENCE PREDICATE
  识别三通道汇聚点 = V-PROVIDES ≥ 2 AND A-PROVIDES ≥ 1 AND N-PROVIDES ≥ 1
  FOR EACH 汇聚点 S[n]:
    IF S[n] 对应场景的叙事高点（重力 ≥ 4）THEN PASS
    ELSE WARN — 汇聚点未对齐叙事高潮
  注意: 不是所有场景都需要三通道汇聚。
        但如果存在汇聚，它应该出现在有意义的位置。
```

### Level 3: 位置谓词（P11-P14）

```
P11 — CAUSALITY ORDER PREDICATE
  FOR EACH 镜头 S[n] 标记为 REACTION:
    LET S[k] 为触发反应的事件镜头
    IF k < n THEN PASS (原因在反应之前)
    ELIF 标注了"延迟揭示" THEN PASS (有意的叙事技法)
    ELSE FAIL — 反应先于原因

  Mirror Visitor 示例:
    S5 (反应) 的触发事件在 S4 (原因), 4 < 5 → P11 PASS

P12 — ORDER IRREVERSIBILITY PREDICATE
  FOR EACH 相邻镜头对 (S[n], S[n+1]):
    假设交换两者顺序:
    IF 交换后叙事意义改变或信息依赖被破坏 THEN PASS (顺序必要)
    ELSE WARN — 顺序可互换，因果关系不够强

  Mirror Visitor 示例:
    S4 和 S5 交换: 先看反应再听台词 → 叙事意义完全改变 → P12 PASS
    反例: 如果两个环境镜头（同样的空间、同样的信息）交换后无变化
    → P12 WARN → 应合并或给予不同信息

P13 — REDUNDANCY PREDICATE
  FOR EACH 镜头 S[n]:
    IF ∃ S[k] (k ≠ n) 使得 S[n].PROVIDES ⊆ S[k].PROVIDES (全通道)
    THEN WARN — S[n] 的所有信息已被 S[k] 覆盖，S[n] 可能冗余
    ELSE PASS
  注意: WARN 而非 FAIL，因为冗余有时用于强调（但需标注理由）

P14 — CANONICAL PROVIDER PREDICATE [v2.1 新增]
  当某个 REQUIRES 条目可被多个前置镜头满足时:
    标注第一个提供该信息的镜头为 canonical provider
    后续提供同一信息的镜头为 redundant provider
    IF redundant provider 存在且未标注强化/呼应理由 THEN WARN
    ELSE PASS
  目的: 区分"第一次建立"和"重复确认"的信息
```

### Level 4: 参数谓词（P15-P18）

```
P15 — SCALE MOTIVATION PREDICATE
  FOR EACH 镜头 S[n]:
    IF 景别选择有标注的叙事理由 THEN PASS
    ELSE FAIL — 景别选择缺乏理由
  理由模板:
    CU/ECU → "情感关键点" / "线索特写" / "反应细节"
    MS/MCU → "对话常规" / "动作展示"
    LS/ELS → "空间建立" / "孤独/渺小" / "环境压力"

P16 — SCALE TRANSITION PREDICATE
  FOR EACH 相邻镜头对 (S[n], S[n+1]):
    LET Δscale = |S[n].景别 - S[n+1].景别| (用景别序数差)
    IF Δscale ≤ 2 THEN PASS (正常过渡)
    ELIF Δscale > 2 AND 有标注的冲击/跳跃理由 THEN PASS
    ELSE WARN — 景别跳跃过大且无理由
  认知依据: Liao et al. (2023) — scale-out > 2 级会引发显著神经反应

P17 — ANGLE POWER PREDICATE
  FOR EACH 镜头 S[n] 中有角色交互:
    IF S[n].角度 反映了当前权力/心理关系的变化 THEN PASS
    ELIF S[n].角度 = EYE (平视) 且权力关系无变化 THEN PASS
    ELSE WARN — 角度与权力关系不匹配

P18 — DURATION DENSITY PREDICATE
  FOR EACH 镜头 S[n]:
    LET expected_duration = base_duration(景别) × gravity_factor(重力)
    IF |S[n].时长 - expected_duration| / expected_duration ≤ 0.5 THEN PASS
    ELIF 有标注的节奏理由（如"故意延长制造不安"）THEN PASS
    ELSE WARN — 时长与信息密度/重力不匹配
```

### Level 5: 序列谓词（P19-P20）

```
P19 — IDIOM COMPLETENESS PREDICATE
  FOR EACH 被识别的 idiom 模式:
    IF idiom 的所有必要阶段都存在 THEN PASS
    ELSE FAIL — idiom 不完整
  必要阶段:
    建立序列: 至少有 LS/ELS + MS
    正反打: 至少有 master + 2个反向镜头
    揭示序列: 至少有 准备 + 揭示 + 反应
    追逐序列: 至少有 追者 + 被追者 + 环境
    反应链: 至少有 事件 + 2个不同反应
    窥视序列: 至少有 POV(有遮挡) + 被窥视者 + 窥视者反应

P20 — NARRATIVE TRACTION PREDICATE
  在任意连续 5 个镜头的窗口内:
    LET questions = Σ(窗口内所有 RAISES) - Σ(窗口内已回答的 RAISES)
    IF questions ≥ 1 THEN PASS (观众脑中始终有悬念)
    ELSE WARN — 5 个镜头窗口内无活跃悬念
  注意: 场景最后 3 个镜头豁免（允许悬念收束）
```

### 谓词执行协议

```
执行顺序: P00 → P01-P04 → P05-P10 → P11-P14 → P15-P18 → P19-P20

统计输出:
  PASS: N 个
  WARN: M 个
  FAIL: K 个
  TOTAL: N + M + K

通过标准:
  FAIL = 0 且 WARN ≤ 3  → 序列通过验证
  FAIL = 0 且 WARN > 3   → 序列需要优化（可继续，但记录问题）
  FAIL ≥ 1               → 序列必须修复后重新验证

与 evaluation.md LOGIC 维度的映射:
  LOGIC 分数 = 10 × (1 - FAIL/total_predicates) × (1 - 0.1×WARN/total_predicates)
  即: 每个 FAIL 等比扣分，每个 WARN 扣 10% 的 FAIL 权重
```

## 七、常见逻辑错误词典

### 错误 1: 无因之切（Unmotivated Cut）

```
症状: 镜头切换没有叙事理由
现象: "为什么突然切到这个角度？"
原因: 追求视觉多样性而非叙事逻辑
检测: P00 FAIL — 切不满足 P01/P02/P03/P04 中任何一个
修复: 为每个切标注理由（信息/反应/强调/重定向），
      如果找不到理由就删除这个切
```

### 错误 2: 信息缺口（Information Gap）

```
症状: 观众突然看到一个没有前置信息的镜头
现象: "这个人是谁？" "这是什么地方？"
原因: 跳过了建立镜头或介绍镜头
检测: P05 FAIL — 某个通道的 REQUIRES 无前置满足
修复: 检查每个镜头每个通道的 REQUIRES，补充缺失的前置镜头
      注意跨通道满足: 如果观众已经听到了(A-channel)，
      视觉确认(V-channel)可以延迟但不能永远缺失
```

### 错误 3: 空转镜头（Empty Shot）

```
症状: 镜头不提供新信息（所有通道 PROVIDES 为空）
现象: "这个镜头看了和没看一样"
原因: 为了凑数量或节奏而加的填充镜头
检测: P06 FAIL — V/A/N 三通道 PROVIDES 全空
修复: 删除或合并；如果需要保留（节奏原因），
      至少在一个通道加入新信息
      注意: "设计过的沉默"(A-channel)和"空间确认"(V-channel)
      都是合法的 PROVIDES，不算空转
```

### 错误 4: 反应先于原因（Reaction Before Cause）

```
症状: 角色的反应镜头出现在引发反应的事件之前
现象: "她为什么哭？哦原来是因为后面那件事"
原因: 镜头顺序错误
检测: P11 FAIL — 反应镜头的索引 < 触发事件镜头的索引
修复: 调整顺序。除非是故意的倒叙/延迟揭示（需标注）
例外: 先给反应再给原因是合法的延迟揭示技巧，
      但必须标注"延迟揭示"以使 P11 通过
```

### 错误 5: 轴线跳跃（Axis Jump）

```
症状: 角色的左右关系在相邻镜头中突然颠倒
现象: 观众感到空间混乱
原因: 摄影机越过了180度轴线
修复: 重新标注摄影机位置；
      如果需要越轴，使用过渡镜头（沿轴线移动的镜头）
例外: 故意越轴表达混乱/权力反转
注意: 与 P04 ORIENTATION 的关系——越轴后通常需要
      ORIENTATION 切来重新建立空间关系
```

### 错误 6: 景别跳崖（Scale Cliff）

```
症状: 从ECU直接切到ELS（或反之），无过渡
现象: 观众的空间认知断裂
原因: 景别跳跃太大且没有叙事理由
检测: P16 WARN/FAIL — Δscale > 2 且无标注理由
修复: 加入中间景别（ECU→CU→MS→LS→ELS）
      或确认跳跃是故意的冲击效果并标注
认知依据: Liao et al. — scale-out会引发强烈神经反应，
          极端的scale-out应该是有目的的
例外: INSERT 镜头的前后过渡不受此规则约束 (R006 修正)
```

### 错误 7: 节奏谎言（Rhythm Lie）

```
症状: 镜头节奏暗示的情绪与叙事内容不一致
现象: 平静的对话用了追逐的节奏，或者紧张的追逐用了慢节奏
原因: 节奏设计没有跟随叙事意图
修复: 让节奏服务于叙事而非相反
例外: 故意的反差（平静画面+紧张节奏=不安感）
      → 但这应该是有意识的对位，不是失误
```

### 错误 8: 空间漂移（Spatial Drift）[v2.1 新增]

```
症状: 连续多个近景后，观众失去对角色空间位置的感知
现象: "他们现在在哪？" "他们之间多远？"
原因: 缺少 ORIENTATION 切（P04 触发条件已满足但未执行）
检测: P04(c) — 连续 ≥ 3 个 CU/ECU 后无 MS+ 景别
修复: 在近景序列后插入一个重定向镜头(MS或更宽)
      重新建立角色间的空间关系
区别于轴线跳跃: 轴线跳跃是空间关系被颠倒,
      空间漂移是空间关系被遗忘(更微妙，更常见)
群戏场景: 阈值降低到 ≥ 2 个 CU/ECU
```

### 错误 9: 全知摄影机（Omniscient Camera Without Purpose）

```
症状: 摄影机出现在角色不可能知道的位置
现象: "谁在从天花板上看？" "谁在壁橱里？"
原因: 追求"好看的角度"而忽略了视角逻辑
修复: 每个非常规角度问："谁的视角？"
  - 角色的POV → 合法
  - 全知叙述者 → 合法（但应该一致）
  - 没有理由 → 换回常规角度

在悬疑片中尤其重要:
  如果一直用全知视角 → 观众获得太多信息 → 没有悬念
  如果一直用限制视角 → 观众和角色一起发现 → 悬念最大化
  视角切换 → 应该是有意识的叙事决策
```

---

### 错误 10: 通道失衡（Channel Imbalance）[v2.1 新增]

```
症状: 某通道长期无信息更新，场景感觉"缺了什么"
现象:
  V-channel 长期静态 → "画面怎么一直没变？"
  A-channel 长期空白 → "怎么这么安静？"（非设计意图）
  N-channel 长期空白 → "表面上很热闹但感觉很空"
原因: 镜头设计过度依赖单一通道
检测: 连续 ≥ 4 个镜头中某通道 PROVIDES 全空
修复:
  V-channel 失衡: 加入 INSERT 或角色动作
  A-channel 失衡: 确认沉默是设计还是遗漏
  N-channel 失衡: 加入构图/并置暗示潜文本
```

## 八、视角逻辑系统

### 谁在看？

每个镜头都隐含一个"观察者"。明确这个观察者是镜头逻辑的关键。

```
观察者类型:

  1. 角色主观 (Character POV)
     摄影机 = 角色的眼睛
     限制: 只能看到角色能看到的
     效果: 最大代入感
     用途: 发现、搜索、恐惧体验

  2. 角色客观 (Character Adjacent)
     摄影机在角色旁边，像一个隐形的同伴
     限制: 看到角色周围的事，但不是全知的
     效果: 伴随感、轻度客观
     用途: 大部分叙事场景的默认选择

  3. 全知观察者 (Omniscient)
     摄影机可以去任何地方，看任何角度
     限制: 无
     效果: 上帝视角、命运感
     用途: 建立镜头、鸟瞰、平行蒙太奇

  4. 隐藏观察者 (Hidden Voyeur)
     摄影机通过遮挡物/缝隙/隐蔽处观看
     限制: 视线被部分遮挡
     效果: 偷窥感、被监视的不安
     用途: 悬疑/惊悚的标志性视角
```

### 视角一致性规则

```
一个场景内:
  选定一个主要视角类型 → 全场景保持
  视角切换 → 必须有叙事理由

跨场景:
  主人公的场景 → 角色客观 + 偶尔POV
  反派的场景 → 全知或隐藏观察者
  揭示场景 → 从限制视角到全知视角的转变本身就是叙事

悬疑的视角策略:
  前80% → 限制视角（和侦探一起不知道）
  揭示时 → 视角突然打开（全知展示真相）
  → 视角的扩大 = 真相的揭示
```
