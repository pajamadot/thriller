# 分镜拆解技能 — Script-to-Storyboard Decomposition

> 将叙事文本转化为专业分镜序列：镜头语言、蒙太奇节奏、场面调度、视觉叙事的完整方法论
> 融合 Eisenstein 蒙太奇理论、Hitchcock 悬疑影像学、Katz 逐镜设计法、Kurosawa 画面运动哲学

---

## 核心理念：三重翻译

```
文字叙事 ──→ 视觉节拍 ──→ 时间体验
(Script)     (Storyboard)   (Film)
```

分镜拆解的本质是一个**三重翻译过程**：

1. **语义翻译**：从文字的叙事意图中提取「视觉节拍」(visual beat)
2. **语法翻译**：用镜头语言的语法规则将节拍编码为具体的镜头设计
3. **韵律翻译**：将镜头序列组织为具有呼吸感和节奏的蒙太奇

> **Hitchcock 定律**：画面中物体的大小应等于它在那一刻对故事的重要性。
> **Spielberg 定律**：一个镜头一个想法——当想法改变时，切。
> **Kurosawa 定律**：先让画面内部运动，再运动摄影机。

## 元认知框架：五层分解模型

分镜拆解遵循由内而外的五层思考模型。每一层回答一个核心问题：

```
┌─────────────────────────────────────────────┐
│  Layer 5: SOUND（什么声音伴随画面？）          │
│  ┌─────────────────────────────────────────┐ │
│  │  Layer 4: MONTAGE（为什么是这个顺序？）    │ │
│  │  ┌─────────────────────────────────────┐ │ │
│  │  │  Layer 3: MISE-EN-SCÈNE（画面里有什么？）│ │ │
│  │  │  ┌─────────────────────────────────┐ │ │ │
│  │  │  │  Layer 2: VISUAL GRAMMAR        │ │ │ │
│  │  │  │  （如何用镜头说？）               │ │ │ │
│  │  │  │  ┌─────────────────────────┐    │ │ │ │
│  │  │  │  │  Layer 1: NARRATIVE      │    │ │ │ │
│  │  │  │  │  （在说什么？）            │    │ │ │ │
│  │  │  │  └─────────────────────────┘    │ │ │ │
│  │  │  └─────────────────────────────────┘ │ │ │
│  │  └─────────────────────────────────────┘ │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

### Layer 1: 叙事层 NARRATIVE（在说什么？）

从文本中提取叙事意图的最小单位——**节拍 (Beat)**。

**节拍识别规则**（McKee 价值变化法）：
- 每一次**价值状态的改变**是一个节拍
- 价值包括：权力关系、情感状态、信息量、威胁等级、角色距离
- 一个节拍 = 一个镜头的叙事基础

**节拍分类**：

| 节拍类型 | 定义 | 典型镜头倾向 |
|----------|------|-------------|
| 行动节拍 | 物理动作推进情节 | 中景-全景，跟拍 |
| 反应节拍 | 角色对事件的内在回应 | 特写-近景，静止 |
| 揭示节拍 | 新信息呈现给角色或观众 | POV / 插入镜头 / 推进运动 |
| 转折节拍 | 场景方向发生转变 | 反转切 / 角度突变 / 重新构图 |
| 张力节拍 | 悬念持续施压但无释放 | 长镜头 / 缓推 / 沉默 |
| 释放节拍 | 张力的暂时释放 | 拉远 / 切到全景 / 环境镜头 |
| 过渡节拍 | 时间/空间/情绪的桥接 | 空镜 / 叠化 / 蒙太奇 |

**节拍权重标注**：
```
每个节拍标注 1-5 的「叙事重力」：
  5 = 故事转折点（全剧转向的关键瞬间）
  4 = 场景转折点（本场戏的核心时刻）
  3 = 重要推进（情节/情感有实质进展）
  2 = 辅助建设（氛围/关系/世界的次要信息）
  1 = 过渡连接（纯粹的叙事胶水）
```

### Layer 2: 视觉语法层 VISUAL GRAMMAR（如何用镜头说？）

将节拍翻译为具体的镜头参数。

**景别选择矩阵**（情感距离 = 物理距离）：

| 景别 | 代码 | 情感效果 | 适用节拍 |
|------|------|---------|---------|
| 大远景 | ELS | 渺小、孤独、史诗感 | 建立、过渡 |
| 远景 | LS | 环境关系、自由/暴露 | 建立、释放 |
| 全景 | FS | 完整动作、社交距离 | 行动、过渡 |
| 中景 | MS | 对话常态、中性观察 | 行动、反应 |
| 中近景 | MCU | 亲密对话、轻度关注 | 反应、揭示 |
| 近景 | CU | 情感细节、关键反应 | 反应、揭示、转折 |
| 大特写 | ECU | 极端情感、微观细节 | 转折、张力 |
| 插入镜头 | INSERT | 关键物件/线索 | 揭示 |

**角度选择矩阵**（权力关系 = 垂直关系）：

| 角度 | 代码 | 心理效果 | 适用场景 |
|------|------|---------|---------|
| 平视 | EYE | 平等、客观 | 日常对话 |
| 俯拍 | HIGH | 弱势、被审视、全局 | 受害者、被控制者 |
| 仰拍 | LOW | 权威、威胁、崇高 | 反派、权力者 |
| 鸟瞰 | BIRD | 全知、命运感、脱离 | 全局揭示 |
| 虫视 | WORM | 极端弱势、压迫 | 恐惧、崩溃 |
| 荷兰角 | DUTCH | 不安、扭曲、疯狂 | 心理失衡 |
| 过肩 | OTS | 对话中的空间关系 | 审讯、对峙 |
| 主观 | POV | 代入、发现 | 调查、追踪 |

**摄影机运动矩阵**：

| 运动 | 代码 | 叙事效果 | 适用场景 |
|------|------|---------|---------|
| 静止 | STATIC | 观察、客观、压力 | 审讯、等待 |
| 推 | PUSH | 注意力聚焦、接近真相 | 发现、揭示 |
| 拉 | PULL | 疏远、揭示全貌 | 释放、建立 |
| 横移 | TRACK | 跟随、平行运动 | 行走、追逐 |
| 升 | CRANE UP | 解放、超越、告别 | 释放、过渡 |
| 降 | CRANE DN | 压迫、进入、聚焦 | 进入空间 |
| 环绕 | ORBIT | 审视、困惑、多角度 | 审讯、内心冲突 |
| 手持 | HANDHELD | 不稳定、临场感 | 追逐、恐慌 |
| 稳定器 | STEADY | 流畅跟随、超自然 | 跟踪、幽灵视角 |
| 变焦推拉 | ZOOM | 心理冲击（Vertigo效果） | 顿悟、恐惧 |
| Dolly Zoom | DZOOM | 空间扭曲、认知失调 | 关键揭示 |

### Layer 3: 场面调度层 MISE-EN-SCÈNE（画面里有什么？）

镜头内部的视觉叙事——不是「怎么拍」而是「拍什么」。

**深度分层**（必须为每个镜头标注）：
```
前景（Foreground）：遮挡物、框架元素、最近的角色
中景（Midground）：主要行动区域、对话双方
背景（Background）：环境信息、暗示、隐藏元素
```

**角色调度三要素**：
1. **位置** (Position)：角色在画面中的水平/垂直位置 → 权力与关系
2. **距离** (Proxemics)：角色间的物理距离 → Hall的四距离理论
   - 亲密区 (<45cm)：爱/威胁
   - 个人区 (45-120cm)：私密对话
   - 社交区 (120-360cm)：正式交流
   - 公共区 (>360cm)：演讲/孤立
3. **朝向** (Orientation)：面向/背对/侧身 → 亲近/拒绝/犹豫

**构图原则**（检查清单）：
- [ ] 三分法：关键元素在交叉点？
- [ ] 引导线：线条引向焦点？
- [ ] 框中框：门/窗/镜子作为画中画？
- [ ] 对称/破对称：秩序 vs 混乱？
- [ ] 负空间：空白传达什么情绪？
- [ ] 色彩主调：冷/暖/高对比/低饱和？

### Layer 4: 蒙太奇层 MONTAGE（为什么是这个顺序？）

镜头之间的关系——Eisenstein 碰撞理论的实际应用。

**Eisenstein 五级蒙太奇**（由简到复杂）：

| 级别 | 名称 | 组织原则 | 效果 | 适用场景 |
|------|------|---------|------|---------|
| 1 | 度量蒙太奇 | 镜头的绝对时长 | 节奏脉搏 | 追逐、倒计时 |
| 2 | 节奏蒙太奇 | 画面内运动决定剪辑点 | 动作流畅 | 动作、日常活动 |
| 3 | 调性蒙太奇 | 情绪基调的统一/对比 | 氛围沉浸 | 氛围建立、抒情 |
| 4 | 泛音蒙太奇 | 1-3的综合叠加 | 多层体验 | 高潮前建设 |
| 5 | 理性蒙太奇 | 符号并置产生概念 | 思想触发 | 主题揭示、反讽 |

**转场设计**：

| 转场 | 代码 | 效果 | 适用场景 |
|------|------|------|---------|
| 硬切 | CUT | 即时、直接 | 默认；同一场景内 |
| 跳切 | JCUT | 时间压缩、心理碎片 | 焦虑、不连贯 |
| 匹配切 | MATCH | 视觉/动作/主题延续 | 空间跳转 |
| 图形匹配 | GMATCH | 形状/线条的呼应 | 象征关联 |
| 淡出/淡入 | FADE | 时间流逝、章节感 | 大段时间跳跃 |
| 叠化 | DISSOLVE | 柔和过渡、记忆 | 回忆、梦境 |
| 划 | WIPE | 明确分隔 | 场景转换 |
| L/J切 | L/J | 声音先行或延续 | 心理延伸 |
| 虹膜 | IRIS | 聚焦/复古感 | 风格化表达 |
| 不可见切 | INVIS | 隐藏在动作中的切 | 伪长镜头 |

**视觉连贯性规则**（强制检查）：
- 180度轴线规则：摄影机不越过动作轴线
- 30度规则：相邻镜头角度变化≥30度
- 视线匹配：A看B → B的画面中应有A注视的方向空间
- 动作匹配：切在动作进行中（非完成后）
- 银幕方向一致：角色移动方向跨镜头保持一致

### Layer 5: 声音层 SOUND（什么声音伴随画面？）

声音设计是分镜的第五维度——不可分割的一部分。

| 声音元素 | 标注方式 | 功能 |
|----------|---------|------|
| 对白 | `DIA: "台词"` | 信息传递、性格表达 |
| 内心独白 | `VO: "内心"` | 内在思绪（画外音） |
| 环境音 | `AMB: 描述` | 空间感、真实感 |
| 音效 | `SFX: 描述` | 强调、惊吓、注意力引导 |
| 音乐 | `MUS: 描述/情绪` | 情绪操控、主题暗示 |
| 沉默 | `SIL: 时长` | 张力、空白的力量 |

**声画关系三种模式**：
- **同步 (Sync)**：声音与画面一致 → 强化
- **对位 (Counterpoint)**：声音与画面矛盾 → 反讽/不安
- **先行 (Advance)**：声音先于画面出现 → 预示/悬念

## `/decompose` 命令详解

### 输入

```
/decompose

输入方式（任选）：
  1. 直接粘贴场景文本
  2. 指定章节路径（如 projects/mirror-visitor/chapters/ch001.md）
  3. 描述场景概要（由AI展开）

可选参数：
  --style <导演名/类型>  视觉风格参考（默认：中性）
  --density <sparse|normal|dense>  镜头密度（默认：normal）
  --format <table|card|json>  输出格式（默认：table）
  --focus <tension|emotion|action|information>  侧重维度
```

### 分解算法（8步，含逻辑验证门禁）

```
Step 1: PARSE（解析）
  ├── 将场景文本按段落/对白切分为文本单元
  ├── 识别每个单元的叙事功能
  └── 标注隐含的视觉/听觉信息

Step 2: BEAT（节拍化）
  ├── 识别价值变化点 → 切分为节拍
  ├── 为每个节拍分类（行动/反应/揭示/转折/张力/释放/过渡）
  ├── 标注叙事重力（1-5）
  ├── 确认节拍序列的戏剧弧线
  └── 强制思维链: 每个节拍必须回答"什么价值变化了？"

Step 3: BLOCK（场面调度）
  ├── 确定角色在空间中的位置关系（A-I-L模型）
  ├── 设计关键时刻的角色走位
  ├── 标注角色间距离变化（Hall Proxemics 四距离）
  └── 设计前景/中景/背景的层次

Step 4: SHOOT（镜头设计）
  ├── 为每个节拍选择景别（情感距离法则）
  ├── 选择角度（权力关系法则）
  ├── 选择运动方式（能量传递法则）
  ├── 设计构图（叙事重心法则）
  ├── 标注声音设计
  └── 强制思维链: 每个镜头必须回答5个Q（见 llm-guidance.md）

Step 4.5: VERIFY（逻辑验证）[★ 核心门禁]
  ├── 为每个镜头标注 REQUIRES / PROVIDES / RAISES
  ├── 构建信息依赖图（DAG）
  ├── 验证每个切的合法理由（信息/反应/强调）
  ├── 检查因果顺序（原因→反应，不可互换）
  ├── 认知验证（scale-in/out方向、运动连贯性）
  ├── Idiom 识别与完整性检查
  └── FAIL → 返回 Step 4 修复 → 重新验证

Step 5: CONNECT（蒙太奇设计）
  ├── 设计镜头间转场（含认知效果验证）
  ├── 确定蒙太奇类型（Eisenstein五级）
  ├── 检查视觉连贯性（轴线/30度/匹配/运动传递）
  └── 设计跨场景的视觉主题线

Step 6: RHYTHM（节奏校准）
  ├── 标注每个镜头的建议时长
  ├── 计算整场的节奏曲线
  ├── 检查张力弧线与叙事弧线的一致性
  ├── 调整密度（加镜头/合并镜头/调整时长）
  └── 计算质量指标（时长方差σ、信息效率、景别熵）

Step 7: ANNOTATE（导演注释）
  ├── 为关键镜头添加意图说明
  ├── 标注参考片段（已知电影中的类似处理）
  ├── 记录替代方案（备选镜头设计）
  └── 标注技术难度和制作注意事项

Step 8: SELF-AUDIT（自审计）[★ 强制步骤]
  ├── 存在性审计（每个镜头有新信息？）
  ├── 依赖审计（信息依赖图有效？）
  ├── Idiom审计（模式完整？）
  ├── 认知审计（神经科学规则通过？）
  ├── 节奏审计（方差/倍率/呼吸点）
  └── 计算6项质量指标 → 输出审计报告
```

- **逻辑标注**:
  - 切理由: 反应→信息（叶知秋的视线引导观众到异常物品）
  - REQUIRES: Shot 3 建立了叶知秋在公寓（空间上下文）
  - PROVIDES: 帽衫的存在（异常信号——她不穿帽衫）
  - RAISES: "这件帽衫是谁的？"（线索C01的视觉植入）
  - 认知: scale-in（MS→CU），流畅无感地引导注意力
  - Idiom: 揭示序列-渐进模式（行动→视线→推近→物件）
- **导演注释**: 观众应在叶知秋之前注意到帽衫的异常——
  她的大衣是深色正装，帽衫是灰色休闲。对比制造不协调感。
  Fair-play线索设计：帽衫在画面中停留5秒，但叶知秋只是
  "大概是吧"地忽略——观众如果注意到了，在ch7就能提前推理。
- **蒙太奇**: 调性蒙太奇——前一镜的温暖（回家）与此镜的不安形成情绪碰撞
- **参考**: 《消失的爱人》开场——看似正常的家居中隐藏异常细节
```

## `/rhythm` 命令详解

分析并优化一组镜头的节奏。

### 节奏分析工具

```
/rhythm [镜头范围]

输出：
  1. 时长曲线图（ASCII）
  2. 景别变化图
  3. 运动能量图
  4. 张力弧线（与叙事弧线叠加对比）
  5. 节奏诊断（问题+修复建议）
```

**节奏曲线（ASCII可视化）**：

```
时长:  ■■■  ■■■■  ■■  ■■■■■■  ■■  ■  ■  ■■■  ■■■■■■■■■
       3s   4s   2s   6s    2s  1s 1s  3s    9s
景别:  ELS  MS   MCU  CU    CU  ECU CU  MS    LS
张力:  ─────╱──────╱───╱╲────╱──╱─────╲──────╲─────
       低    中     中高  高   高峰    中     低
```

**六种节奏病症**：

| 病症 | 特征 | 处方 |
|------|------|------|
| 单调症 | 连续5+镜头相同景别/时长 | 插入反差镜头 |
| 气喘症 | 全程短镜头，无喘息 | 加入长镜头呼吸点 |
| 昏睡症 | 全程长镜头，无加速 | 加入快切序列 |
| 心律不齐 | 无规律的时长变化 | 建立节奏模式再打破 |
| 高原症 | 张力长时间不变化 | 制造微起伏 |
| 过山车症 | 张力剧烈波动无逻辑 | 重新设计张力弧线 |

```

     ▲

  两人成90度，摄影机在对角
  ■ →
  ↓
  ■     CAM ↗
  适用：日常对话、会面

## `/visualize` 命令详解

将分镜序列转换为 AI 图像生成提示词。

```
/visualize [镜头范围]

输出模式:
  --format sdl    SDL中间表征（模型无关JSON，默认）
  --format nb2    Nano Banana 2 适配器（自然语言创意简报）
  --format sd     Stable Diffusion 适配器（关键词 + negative）
  --format dalle  DALL-E 适配器（简洁自然语言）
  --format mj     Midjourney 适配器（关键词 + --参数）
  --format fal    fal.ai 适配器

  架构: 分镜参数 → SDL(模型无关) → Adapter(薄层转换) → 模型格式
  换模型只需换适配器，SDL和分镜逻辑完全不变

范围控制:
  --scene <全场景>    每个镜头生成一条提示词
  --keyframes         只为重力≥3的关键帧生成
  --shot <N>          只为第N个镜头生成

一致性:
  --consistent        保持角色描述和环境描述跨镜头一致
  --resolve-refs      解析 @/# 引用为URL或文字描述

资产引用（双符号系统）:
  @ = 人物: @叶知秋  @林小曼.恐惧  @叶知秋.全身
  # = 非人物: #咨询室  #帽衫  #叶知秋公寓.夜

  解析链:
    有资产URL → 附加 reference_image（最佳一致性）
    无URL有描述 → 展开为文字（次佳）
    都没有 → 警告 + 建议先生成

提示词结构（6段式）:
  [镜头类型] [主体描述] [动作/姿态] [环境/背景] [光线/色彩/氛围] [风格/技术]

转换规则:
  景别 → 主体在画面中的占比（CU=面部填满，LS=全身+环境）
  角度 → 视角描述（LOW=仰视英雄感，HIGH=俯视脆弱感）
  色温 T_color() → 色彩关键词（3200K=warm golden，7500K=cool blue）
  深度分层 → 景深描述（浅焦=bokeh，深焦=everything sharp）
  隐喻 M01-M28 → 象征元素的可见呈现
  类型库 → 氛围词和负面提示词
```

详见 `references/image-prompt-gen.md`。

## 悬疑/惊悚类型的专项分镜策略

由于本技能包位于 `thriller` 仓库，以下策略针对悬疑/惊悚类型做专门优化：

### 线索植入的镜头技巧

```
Fair-Play 线索（让观众有机会发现）：
  - 将线索放在画面中但不用镜头强调
  - 使用 LS/FS 让线索存在于环境中而非特写
  - 角色视线不指向线索（但线索在画面三分点上）
  - 第二次出现时才用 CU/INSERT 揭示

误导线索（转移注意力）：
  - 用 CU/INSERT 强调非关键物品
  - 用角色反应镜头暗示错误方向
  - 用 SFX 引导注意力到错误区域
```

### 悬念维持的节奏模式

```
Hitchcock式悬念（观众知道角色不知道）：
  镜头节奏：CU(角色)-INSERT(威胁)-CU(角色)-INSERT(威胁)-ECU(角色反应)
  时长模式：3s - 2s - 4s - 2s - 6s（最后一个镜头延长 = 延迟释放）

追逐式悬念（角色和观众一起发现）：
  镜头节奏：POV-反应CU-POV(更近)-反应MCU-POV(揭示)
  时长模式：2s - 1s - 3s - 1s - 5s（递增发现，最终停留）

心理悬念（内在恐惧外化）：
  镜头节奏：MS(正常)-PUSH(逼近)-CU(面部)-DZOOM(空间扭曲)-STATIC CU(凝视)
  时长模式：4s - 3s - 3s - 2s - 8s（Dolly Zoom为转折点，之后静止延长）
```

### 真相揭示的镜头编排

```
线索回溯蒙太奇：
  采用理性蒙太奇——快速闪回之前的线索镜头（ECU级别）
  每个闪回 0.5-1.5s，以新的理解重新呈现
  音效：心跳声或钟表声作为节奏基底

渐进式揭示：
  景别从 LS 缓慢推到 ECU，一个长镜头完成
  观众随摄影机一起接近真相
  无音乐，环境音渐弱至沉默

瞬间揭示（twist）：
  前一镜头是完全不相关的内容（日常/平静）
  硬切到揭示镜头——ECU或INSERT
  声音设计：突然的沉默或单一音效（门声/呼吸声）
```
