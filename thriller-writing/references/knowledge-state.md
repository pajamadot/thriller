# 知识状态追踪系统

> 悬疑小说的核心引擎：谁在什么时刻知道什么
> 基于 Dynamic Epistemic Logic (Eger, AIIDE 2020)、NovelGenerator 角色知识追踪、Wagner et al. 双读者模型

---

## 一、为什么知识状态是悬疑的核心

```
悬疑 = 信息不对称的管理

所有悬疑技术的本质都是操控"谁知道什么"：
  - 线索 = 让读者获取信息的渠道
  - 红鲱鱼 = 让读者获取错误信息
  - 悬念 = 读者知道但角色不知道（Hitchcock）
  - 惊吓 = 读者和角色同时获知
  - 反转 = 读者以为知道的被推翻

因此：精确追踪知识状态是专业级悬疑写作的基础设施。
```

## 二、四层知识状态模型

### Layer 1: 事实层（Ground Truth）
作者知道的完整真相。

```
事实层清单（在 /trick 阶段完成）：
  F01: 凶手是 ___
  F02: 犯罪动机是 ___
  F03: 犯罪手法是 ___
  F04: 犯罪时间是 ___
  F05: 关键物证是 ___
  ...

所有其他层都是事实层的子集或扭曲。
```

### Layer 2: 角色知识层
每个角色在每个时间点知道什么。

```
知识矩阵（每章更新）：

| 事实 | 叶知秋 | 苏远 | 林小曼 | 陈雪莹 | 周明哲 |
|------|--------|------|--------|--------|--------|
| F01  | ✗      | ✗    | ✗      | 怀疑   | ✗      |  ← 第1章
| F01  | ✗      | 怀疑  | ✗      | 怀疑   | ✗      |  ← 第4章
| F01  | 面对   | 确认  | ✗      | ✗      | ✗      |  ← 第8章

标注系统：
  ✗ = 不知道
  怀疑 = 有直觉但无证据
  部分 = 知道部分事实
  误解 = 持有错误信息
  确认 = 知道且确定
  面对 = 知道但在否认
```

### Layer 3: 读者知识层
读者在每个章节末应该知道什么、推测什么、误信什么。

```
读者知识状态表：

| 章节 | 读者确知 | 读者推测 | 读者误信 | 读者不知 |
|------|---------|---------|---------|---------|
| ch1  | 有人被跟踪 | 周明哲可疑 | 叶知秋可靠 | 叶知秋=访客 |
| ch4  | 周明哲无辜 | 陈雪莹可疑 | 叶知秋可靠 | 叶知秋=访客 |
| ch7  | 监控身影像叶 | 叶知秋有问题 | — | 完整真相 |
```

### Layer 4: 叙述层
叙述者选择告诉读者什么（≠ 读者知识层，因为读者可以推理出叙述者未明说的事）。

```
叙述控制表：

| 事实 | 叙述者是否提及 | 提及方式 | 读者是否能推理出 |
|------|---------------|---------|-----------------|
| 叶知秋的空白时间 | 提及但淡化 | "我失眠了" | 二次阅读可以 |
| 鞋子上的泥 | 提及但自我解释 | "大概白天踩的" | 敏感读者可能注意 |
| 灰色帽衫 | 提及但不强调 | 混在场景描写中 | 几乎不可能注意到 |
```

## 三、知识状态操作

### 六种基本操作

| 操作 | 定义 | 悬疑效果 |
|------|------|----------|
| 揭示（Reveal） | 让角色/读者获取新事实 | 推进调查 |
| 隐藏（Conceal） | 阻止角色/读者获取事实 | 维持谜团 |
| 误导（Mislead） | 让角色/读者获取错误信息 | 制造红鲱鱼 |
| 纠正（Correct） | 推翻之前的错误信息 | 消解红鲱鱼 |
| 重新解读（Reinterpret） | 已知信息获得新含义 | 反转/中点 |
| 确认（Confirm） | 验证之前的推测为真 | 满足感 |

### 竞争叙事框架（引自 John Truby）

```
专业级悬疑中，侦探与凶手在争夺"哪个版本的现实被接受"：

凶手的叙事：事实 F01-F05 的伪装版本（"事情不是这样的"）
侦探的叙事：从碎片中重建事实（"事情一定是这样的"）
读者的叙事：在两个版本之间摇摆（"到底是哪个？"）

每个场景都是三个叙事的角力。
场景的核心冲突不仅是"发生了什么"，
而是"谁的版本更有说服力"。

设计每个场景时问自己：
  - 这个场景让凶手的叙事更强还是更弱？
  - 这个场景让侦探的叙事更强还是更弱？
  - 读者此刻更倾向相信谁？
```

## 四、双读者公平性验证

### Wagner et al. (2025) 的概率模型

```
一个好的悬疑需要同时对两个"理论读者"有效：

读者 A（天真读者）：
  - 按顺序阅读
  - 不会回溯验证
  - 会被红鲱鱼误导
  - 揭示时应该感到惊奇
  验证：读者 A 不应该在揭示前猜到真相

读者 B（侦探读者）：
  - 仔细阅读每个细节
  - 会回溯验证
  - 会质疑红鲱鱼
  - 揭示时应该感到"我早该想到"
  验证：读者 B 应该能通过线索推理出真相

公平性 = 读者 A 的惊奇 + 读者 B 的"原来如此"

验证方法：
  1. 以读者 A 的视角通读——是否被误导？揭示是否惊奇？
  2. 以读者 B 的视角通读——线索是否可见？逻辑链是否完整？
  3. 如果 A 不惊奇 → 误导不够
  4. 如果 B 无法推理 → 线索不够
  5. 两者同时满足 = 公平
```

## 五、知识状态与场景设计的关系

### 每场景知识状态检查

```
场景设计时填写：

进入场景时：
  侦探知道：___
  凶手知道：___
  读者知道：___
  本场景的其他角色知道：___

场景核心知识操作：
  □ 揭示 ___ 给 ___
  □ 隐藏 ___ 不让 ___ 知道
  □ 误导 ___ 让 ___ 以为 ___
  □ 重新解读 ___ 使其意义从 ___ 变为 ___

退出场景时：
  侦探知道：___（变化了什么？）
  凶手知道：___（ta知道侦探知道了什么？）
  读者知道：___（比侦探多还是少？）

核心检查：
  □ 至少一个知识状态发生了变化
  □ 变化推进了调查或加深了谜团
  □ 凶手的叙事和侦探的叙事至少有一个被强化或削弱
```

## 六、反派计划即情节骨架

### James Frey 的"反派引擎"原则

```
专业悬疑的情节设计从反派开始：

步骤 1：设计反派的完整计划
  - 反派要做什么？（dark mission）
  - 反派的计划分几步？
  - 每一步如何执行？
  - 反派预见到了哪些障碍？
  - 反派的后备方案是什么？

步骤 2：设计计划的"痕迹"
  - 每一步执行后会留下什么痕迹？
  - 哪些痕迹反派试图消除？
  - 哪些痕迹反派没有意识到？
  → 这些痕迹 = 线索

步骤 3：设计侦探如何发现痕迹
  - 侦探通过什么行动发现每条线索？
  - 发现的顺序如何影响侦探的推理？
  → 侦探的调查行动 = 情节

步骤 4：设计反派的反应
  - 反派如何得知侦探在调查？
  - 反派如何反击（销毁证据/制造混乱/威胁/新犯罪）？
  → 反派的反击 = 冲突升级

公式：反派计划 + 痕迹 + 侦探发现 + 反派反击 = 情节
```
