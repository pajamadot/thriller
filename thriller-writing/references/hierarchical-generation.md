# 层级级联生成模式

> 从一句话到完整作品的系统化展开路径
> 参考 Google DeepMind Dramatron 层级生成 + GOAT Story/Scene 分解

---

## 一、Dramatron 级联模型

```
Level 0: Log Line（一句话概要）
  ↓ 展开
Level 1: Characters（角色描述）
  ↓ 展开
Level 2: Plot Beats（故事节拍）
  ↓ 展开
Level 3: Locations（场景地点描述）
  ↓ 展开
Level 4: Dialogue（逐场景对话和叙述）

每一层的输出是下一层的输入上下文。
上层决定不改，下层才能开始。
```

### 应用于悬疑创作

```
Level 0: /start 的一句话描述
  "心理咨询师发现自己的来访者遭受跟踪，调查后发现跟踪者是自己。"

Level 1: /theme + /characters
  主题前提 + 角色系统（侦探、犯人、嫌疑人、受害者）

Level 2: /trick + /structure
  诡计设计 + 三幕结构 + 章节大纲

Level 3: 场景级规划
  每章的场景清单（地点、时间、角色、核心功能）

Level 4: /scene N
  逐章完整写作

关键原则：
  每一层都必须完成审核后再进入下一层。
  修改上层 → 下层全部需要重新验证。
  不要跳层——不要在角色没定好的时候开始写对话。
```

## 二、GOAT Story/Scene 两级分解

```
Story Level（宏观）：
  - 类型/子类型
  - 核心角色阵容
  - 世界设定
  - 主要情节弧线
  - 整体调性和风格

Scene Level（微观）：
  - 具体事件
  - 涉及角色
  - 场景情绪
  - 故事价值变化（McKee: 每场景必须有价值翻转）
  - 信息操作（线索/误导/揭示）

两级之间的映射：
  每个 Story Level 的情节弧线 → 分解为多个 Scene Level 事件
  每个 Scene Level 事件 → 必须推进至少一个 Story Level 弧线
  如果一个场景不推进任何弧线 → 删除它
```

## 三、动态上下文压缩

```
写到第 15 章时，不可能把全部前文放进上下文。

StoryWriter 的方法：根据当前章节动态压缩历史。

对第 N 章的写作，上下文应该包含：
  1. 项目摘要（theme.md, trick-design.md 的精简版）— 始终存在
  2. 角色状态快照（截至第 N-1 章的知识状态矩阵）
  3. 第 N-1 章全文（前一章细节完整保留）
  4. 第 N 章大纲（structure.md 中本章的计划）
  5. 线索追踪器（哪些线索已布设、哪些待布设）
  6. 章节摘要（第 1 到 N-2 章的一句话摘要）

不需要包含：
  ✗ 所有前文的完整正文
  ✗ 与当前章节无关的参考资料
  ✗ 已消解的红鲱鱼的详细信息
```

## 四、IBSEN 导演/演员分离模式

```
源自 ACL 2024 的 IBSEN 框架：

导演（Director）= 控制全局的系统层
  - 知道所有真相（完整的 trick-design）
  - 控制信息释放节奏（知识状态矩阵）
  - 决定每个场景的"任务"（本场景要完成什么）
  - 管理张力曲线

演员（Actor）= 扮演角色的执行层
  - 只知道自己角色知道的事情
  - 在限制信息下做出"真实"的反应
  - 角色的对话和行为由角色的知识状态决定

应用于写作：
  写每个场景时切换两种思维：
  1. 导演思维：这个场景要达成什么叙事目标？
  2. 演员思维：这个角色此刻知道什么、想要什么、会怎么做？

  两者冲突时 → 调整场景设计（导演思维），不要强行让角色做不自然的事（演员思维）
```
