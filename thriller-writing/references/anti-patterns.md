# 悬疑写作反模式

> 专业作者绝不会犯的错误——以及如何避免
> 从失败案例和编辑反馈中提取的负面清单

---

## 一、情节反模式

| 反模式 | 描述 | 修复 |
|--------|------|------|
| 天降解决（Deus Ex Machina） | 案件被偶然/外力解决，非侦探推理 | 所有解决方案必须来自已建立的线索和能力 |
| 白痴情节（Idiot Plot） | 故事只有在角色表现愚蠢时才成立 | 给每个角色的"失误"一个合理的心理原因 |
| 巧合破案 | 侦探碰巧在对的时间出现在对的地方 | 巧合可以制造问题，但不能解决问题 |
| 无限线索 | 每次卡住就出现新的证据/证人 | 所有关键线索在前 60% 必须已经存在 |
| 死人说话 | 受害者留下完美的线索指向凶手 | 受害者的"线索"应该是无意的痕迹，不是有意的指控 |
| 连环解释 | 揭示场景需要 20 分钟解释来龙去脉 | 分散揭示，高潮只确认读者已 80% 知道的事 |

## 二、角色反模式

| 反模式 | 描述 | 修复 |
|--------|------|------|
| 纸片嫌疑人 | 嫌疑人只有"嫌疑"功能，没有人格 | 每个嫌疑人有独立于案件的个人故事 |
| 万能侦探 | 侦探什么都懂，永不犯错 | 给侦探明确的知识边界和判断失误 |
| 工具受害者 | 受害者只是"引发案件的尸体" | 受害者是"最重要的不在场角色" |
| 动机不足 | 犯罪动机不足以支撑犯罪的严重程度 | 动机必须是犯人主观上"无法承受的压力" |
| 性格突变 | 角色在揭示时的行为与前文不符 | 揭示后回看，角色的所有行为在新信息下自洽 |

## 三、线索反模式

| 反模式 | 描述 | 修复 |
|--------|------|------|
| 知识作弊 | 依赖只有专家才知道的冷知识 | 关键线索必须基于常识或故事中已介绍的知识 |
| 隐形线索 | 线索存在但读者不可能注意到 | 线索必须被叙述提及（即使不强调） |
| 过度标记 | 线索出现时叙述明显"画重点" | 用功能遮蔽/情感遮蔽/列举遮蔽隐藏线索 |
| 红鲱鱼泛滥 | 误导太多，读者放弃推理 | 红鲱鱼数量 ≤ 关键线索数量 |
| 回溯植入 | 揭示时引用前文中不存在的线索 | 每条揭示中引用的线索都可以指出具体章节 |

## 四、叙事反模式

| 反模式 | 描述 | 修复 |
|--------|------|------|
| 信息灌输对话 | "正如你所知，我们是三年前认识的..." | 通过冲突和行动自然暴露信息 |
| 内心独白过载 | 侦探的推理占据大段篇幅 | 用对话和行动展示推理，内心独白只用于关键时刻 |
| 描写堆砌 | 每个场景都有长段环境描写 | 描写只在做功时出现（建立氛围/隐藏线索/反映心理） |
| 节奏单调 | 每章结构相同：调查→发现→反思 | 变化章节结构、长度、视角、张力级别 |
| 主题说教 | 角色直接讲述作品主题 | 主题通过行动和选择体现，不通过台词宣讲 |

## 五、互动小说特有反模式

| 反模式 | 描述 | 修复 |
|--------|------|------|
| 虚假选择 | 选项不同但结果完全一样 | 每个选择至少改变一个状态变量或叙述段落 |
| 惩罚好奇 | 探索性选择导致坏结果 | 探索可以有风险，但不应惩罚好奇心 |
| 信息锁死 | 关键线索只在一条路径上可得 | 关键信息至少 2 条路径可获取 |
| 选择轰炸 | 连续多个高风险选择 | 高负荷选择之间插入 2-3 个低负荷选择 |
| 最优路线 | 一条路线明显优于其他所有路线 | 每条路线有独特的叙事价值和收益 |
