# 聚合前提设计系统

> 为什么这些特定的人，在这个特定的时间，出现在这个特定的地点？
> 密室/暴风雪山庄/封闭社区等封闭型悬疑的第零层约束

---

## 一、为什么聚合前提是独立约束

```
聚合前提 ≠ 设定 (setting)
聚合前提 ≠ 角色动机 (character motive)
聚合前提 = 设定 × 角色动机 × 时间压力 × 社会逻辑 的交叉验证

它回答的不是"故事发生在哪"，也不是"角色想要什么"，
而是"为什么这些人此刻聚在一起"——
这个问题跨越了 /trick、/characters、/setting 三个步骤的边界。
```

---

## 二、聚合事件设计

### 聚合事件类型谱

| 类型 | 聚合力 | 封闭性 | 经典范例 |
|------|--------|--------|---------|
| 遗产/法律事件 | 极强（不来有法律后果） | 中 | 遗产继承、遗嘱公布、庭审 |
| 仪式/庆典 | 强（社会压力） | 中 | 婚礼、葬礼、毕业典礼、周年纪念 |
| 职业义务 | 强（职业后果） | 高 | 培训、团建、学术会议、船员值班 |
| 邀请/召集 | 中（可拒绝） | 需加强 | 聚会、旅行团、私人晚宴 |
| 困境/意外 | 极强（无法离开） | 极高 | 暴风雪、航班取消、封锁、灾难 |
| 共同利益 | 中（利益驱动） | 低 | 投资会、合作项目、寻宝 |
| 过去的约定 | 中（道义约束） | 低 | "十年之约"、赎罪之旅、和解会面 |

### 聚合事件设计检查单

```
□ 事件本身是否独立于犯罪有存在价值？（不能是"专门为了杀人才安排的聚会"）
□ 事件的发起人是谁？发起动机是否独立于犯罪可信？
□ 事件的表面目的与真实目的是否有差异？差异本身是否可以成为线索？
□ 事件的时间是否有紧迫性？（为什么是现在而不是去年/明年？）
□ 事件是否自然产生封闭性？还是需要外部事件（暴风雪等）叠加？
□ 事件是否为凶手提供了犯罪机会？（聚合前提与犯罪机会的因果链）
```

## 三、角色到场动机设计

### 三层动机模型

对每个角色，设计三层到场动机：

```
角色 X 的到场动机：

  Layer 1: 独立理由
    如果其他人都不来，X 自己还会来吗？为什么？
    → 这必须是一个与 X 的个人需求/义务/欲望直接相关的理由
    → 反例（弱动机）："被邀请了所以来了" — 这不是独立理由

  Layer 2: 关系理由
    X 来是否因为知道某个特定的人会来（或不会来）？
    → 这层理由强化角色关系网络
    → 例："如果 Y 不来，X 会来吗？" 的答案揭示 X-Y 关系的真实权重

  Layer 3: 不来的代价
    如果 X 不来，X 会面对什么后果？
    → 后果越严重，到场越可信
    → 后果类型：经济损失 / 社会声誉 / 法律义务 / 情感破裂 / 秘密暴露
```

### 反向验证

```
对每个角色执行：

  测试 1：减法测试
    "如果把这个角色从故事中完全移除，故事是否仍然成立？"
    → 如果成立 → 这个角色的存在缺乏叙事必要性 → 需要加强或合并

  测试 2：替换测试
    "这个角色能否被一封信/一个电话/一条短信替代？"
    → 如果可以 → 这个角色不需要在场，在场理由不充分

  测试 3：时间测试
    "这个角色能否迟到一天/提前一天离开？"
    → 如果可以且不影响情节 → 到场的时间约束不够强

  测试 4：拒绝测试
    "如果这个角色拒绝了邀请，会发生什么？"
    → 必须有一个可信的理由解释为什么 ta 没有拒绝
    → 最好的理由是：拒绝的代价高于参加的不便
```

## 四、聚合前提一致性矩阵

### 标准模板

```
| 角色 | 独立理由 | 关系依赖 | 不来的代价 | 与犯罪的隐藏关联 | 减法测试 |
|------|---------|---------|-----------|----------------|---------|
| A    |         |         |           |                | 通过/失败 |
| B    |         |         |           |                | 通过/失败 |
| C    |         |         |           |                | 通过/失败 |
| ...  |         |         |           |                |         |
```

### 一致性规则

```
规则 1：无循环依赖
  不允许：A 来是因为 B 来，B 来是因为 A 来
  → 至少一方必须有独立理由

规则 2：无纯工具人
  每个角色的到场动机至少满足三层中的两层
  → 仅有"被邀请"不够，必须有独立理由或不来的代价

规则 3：聚合事件的容量合理性
  聚合事件能容纳的人数是否与角色数量匹配？
  → 6 人的私人晚宴 vs 200 人的学术会议 → 角色密度不同

规则 4：到场动机与犯罪机会的因果性
  至少 1 个角色（凶手）的到场动机必须与犯罪有因果关系
  → 可以是：凶手主动制造了聚会以获取犯罪机会
  → 也可以是：凶手因为其他原因来了，发现机会后临时起意
  → 两者对嫌疑指数曲线的影响不同

规则 5：缺席角色的叙事价值
  "谁被邀请了但没来？为什么？"
  → 缺席本身可以是一条线索（缺席者知道会发生什么？）
  → 缺席的理由可以是红鲱鱼（"ta 没来是因为心虚？"）
```

## 五、聚合前提与子类型的适配

### 封闭型（密室/暴风雪山庄）

```
聚合前提是最关键的约束，必须在 /trick 之前完成。

  /start → /theme → /premise → /trick → /characters → /structure → /scene

原因：在封闭型故事中，"谁在场"直接决定了"犯罪如何发生"。
嫌疑人池 = 在场的所有人，因此每个人的到场理由都是故事的承重结构。

重点检查：
  □ 封闭的物理原因（暴风雪/岛屿/封锁）是否可信？
  □ 封闭的时间窗口是否足够犯罪+调查？
  □ 所有角色在封闭前到场的时间是否合理？
  □ 是否有角色"差点没来"？（制造张力）
```

### 开放型（社会派/犯罪程序/日常推理）

```
聚合前提相对灵活，可在 /characters 阶段处理。

  /start → /theme → /trick → /characters（含聚合前提）→ /structure → /scene

原因：开放型故事中角色可以自由进出，
聚合前提主要是"为什么侦探和嫌疑人会接触"而非"为什么大家都在"。

重点检查：
  □ 侦探为什么要管这件事？（超越"这是我的工作"）
  □ 嫌疑人为什么接受审问？（律师、社会压力、自信...）
  □ 信息源为什么提供/不提供信息？
```

### 心理悬疑型

```
聚合前提可能是诡计的一部分。

  /start → /theme → /trick（含聚合前提的扭曲版本）→ /characters → /structure → /scene

原因：在心理悬疑中，"为什么我在这里"本身可能是谜团的核心。
主角对自己到场理由的认知可能是错误的 → 这构成一层叙述诡计。

重点检查：
  □ 主角认为的到场理由 vs 真实的到场理由 → 差异本身是否构成反转？
  □ 其他角色知道真实理由吗？知道多少？
  □ 读者何时发现主角的到场理由有问题？
```

## 六、聚合前提与其他系统的接口

### → /trick（犯罪机会链）

```
聚合事件如何为犯罪创造机会：

  聚合事件 → 特定人群在特定空间 → 凶手与受害者接触 → 犯罪机会

  检查：
  □ 凶手是否利用了聚合事件本身的特征来犯罪？
    （例：利用晚宴的换座规则、利用暴风雪的停电、利用会议的时间安排）
  □ 没有聚合事件，凶手是否仍有犯罪机会？
    → 如果有 → 聚合事件与犯罪无因果关系 → 可以但需要有其他存在理由
    → 如果没有 → 聚合事件是犯罪的必要条件 → 这本身可以是一条线索
```

### → /characters（关系网络）

```
聚合前提提供角色关系的"前置真相"：

  到场动机揭示关系 →
    "A 来是因为 B" → A-B 有依附关系
    "C 来是因为不来会被开除" → C 与聚合发起人有权力不对等
    "D 来是因为听说 E 会来" → D-E 有未公开的关联

  这些关系真相在 /characters 阶段被深化，
  在 /scene 阶段被逐步揭示。
```

### → /structure（第一幕的开场逻辑）

```
聚合前提直接决定第一幕如何开场：

  第一幕的核心任务之一 = 自然地呈现所有角色的到场
  → 如果聚合前提薄弱，第一幕的"角色介绍"环节会显得生硬
  → 如果聚合前提强烈，角色的到场本身就是叙事动力

  检查：
  □ 第一幕的前 3 个场景能否自然完成所有角色的到场介绍？
  □ 到场过程中是否可以同时埋设第一批线索？
  □ 有没有角色的到场方式本身就制造了悬念？
```
