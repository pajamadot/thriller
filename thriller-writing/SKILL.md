# 悬疑小说剧本创作系统 — Thriller Screenplay Craft

> 基于专业编剧方法论的 AI 悬疑/推理小说剧本创作技能包
> 融合 Robert McKee《故事》、Syd Field《电影剧本写作基础》、Blake Snyder《救猫咪》、Raymond Chandler 推理写作法则、Alfred Hitchcock 悬念理论

---

## 系统定位

本技能专注于**悬疑/推理/惊悚**类型的小说与剧本创作，提供从构思到成稿的全流程系统化指导。覆盖：

- 本格推理（公平解谜型）
- 社会派推理（动机驱动型）
- 心理悬疑（不可靠叙事型）
- 惊悚悬疑（高压紧迫型）
- 密室推理（封闭空间型）
- 叙述性诡计（元叙事型）
- 犯罪悬疑（警探/侦探程序型）

## 命令体系

### `/开始` — 项目初始化

**触发条件**：新项目启动时
**必须收集的参数**：

| 参数 | 说明 | 选项示例 |
|------|------|----------|
| 类型 | 悬疑子类型 | 本格推理 / 社会派 / 心理悬疑 / 惊悚 / 密室 / 叙述诡计 / 犯罪程序 |
| 篇幅 | 目标字数/集数 | 短篇(3万字) / 中篇(10万字) / 长篇(25万字+) / 剧集(集数) |
| 叙事视角 | 视角选择 | 第一人称 / 第三人称有限 / 多视角交替 / 全知（慎用） |
| 核心诡计类型 | 谜团的本质 | 密室 / 不在场证明 / 身份替换 / 叙述陷阱 / 动机反转 / 连环杀人逻辑 |
| 风格调性 | 整体氛围 | 冷硬写实 / 哥特暗黑 / 日系清冷 / 社会批判 / 黑色幽默 |
| 目标读者 | 受众画像 | 硬核推理迷 / 大众悬疑读者 / 影视改编向 / 网文读者 |

**输出**：生成 `.thriller-state.json` 项目状态文件

**执行逻辑**：
1. 收集以上参数
2. 根据子类型加载对应参考资料：`references/mystery-structure.md`
3. 输出项目概要卡片
4. 生成状态文件，标记当前阶段为"已初始化"

### `/诡计设计` — 核心谜团架构

**触发条件**：`/开始` 完成后
**加载参考**：`references/clue-design.md`, `references/red-herring.md`, `references/twist-design.md`

**输出结构**（生成 `trick-design.md`）：

```
一、核心诡计概述
   - 谜面（读者看到的表象）
   - 谜底（真实发生的事情）
   - 诡计机制（表象如何形成）

二、公平性验证（Knox 十诫 / Van Dine 二十则 适用项检查）
   - 线索是否全部呈现给读者
   - 是否依赖未公开信息
   - 是否违反已建立的世界规则

三、线索布局
   - 关键线索清单（编号、出现章节、呈现方式、读者注意力预测）
   - 误导线索清单（编号、指向的错误结论、消解方式）
   - 线索-时间线交叉表

四、诡计可行性分析
   - 物理可行性（密室/不在场证明类必须）
   - 心理可行性（动机/行为逻辑）
   - 信息控制可行性（读者视角管理）
```

**设计原则**（引自 Raymond Chandler & Ellery Queen）：
- **解答唯一性**：同一组线索只能指向唯一真相
- **线索公平性**：至少 3 条关键线索在揭示前已呈现
- **误导合理性**：红鲱鱼必须有独立存在的叙事理由
- **逻辑闭环**：诡计揭示后，所有前文"不合理"之处获得解释

### `/人物` — 角色系统设计

**触发条件**：`/诡计设计` 完成后
**加载参考**：`references/character-archetype.md`

**输出结构**（生成 `characters.md`）：

```
一、侦探/主角
   - 人物档案（姓名、背景、性格、核心缺陷、侦探方法论）
   - Watson 角色（助手/叙述者/对照）
   - 侦探的个人案件（Personal Stakes）

二、犯人/反派
   - 第一印象设计（读者初见时的人设）
   - 真实身份与动机
   - 犯罪能力来源（技能、资源、机会）
   - 犯人的自洽叙事（ta 如何合理化自己的行为）
   - 暴露轨迹（从完美隐藏到逐步暴露的关键节点）

三、嫌疑人矩阵
   | 角色 | 动机 | 机会 | 能力 | 不在场证明 | 嫌疑指数曲线 |

四、受害者
   - 受害者不是 NPC：赋予完整人格与秘密
   - 受害者的 N 个面孔（不同角色眼中的受害者）
   - 受害者的秘密（与案件的真正关联）

五、关系网络图
   - 角色两两关系定义
   - 秘密链（A 知道 B 的秘密，B 知道 C 的秘密...）
   - 冲突矩阵
```

**角色设计原则**（引自 McKee《故事》）：
- **压力下的选择揭示人物**：每个关键角色至少有一个"两难选择"时刻
- **人物弧光与案件弧光交织**：主角在破案过程中完成内在转变
- **嫌疑人不是工具人**：每个嫌疑人都有独立于案件的个人故事线

### `/结构` — 叙事结构设计

**触发条件**：`/人物` 完成后
**加载参考**：`references/mystery-structure.md`, `references/pacing-tension.md`

**输出结构**（生成 `structure.md`）：

```
一、结构选择与论证
   - 所选结构模型（三幕式 / 英雄旅程 / 起承转合 / 倒叙 / 多线交织）
   - 为什么这个故事适合这个结构

二、三幕式（悬疑变体）适配
   第一幕·建立（全文 25%）
   - 日常→事件→卷入
   - 关键节点：尸体/案件发现点（Inciting Incident）
   - 完成条件：读者了解所有主要角色，第一个谜团建立

   第二幕·调查与复杂化（全文 50%）
   - 前半：表面调查，收集线索，建立错误假设
   - 中点逆转（Midpoint Reversal）：颠覆前半部分建立的认知
   - 后半：深层真相浮现，个人风险升级
   - 第二幕终点：最黑暗时刻 / 侦探陷入困境

   第三幕·揭示与解决（全文 25%）
   - 最终线索的获取
   - 推理/对决场景
   - 真相揭示
   - 余波与主题升华

三、章节大纲
   | 章节 | 标题 | 核心事件 | 线索操作 | 嫌疑转移 | 张力指数 |

四、时间线
   - 故事时间线（案件实际发生的顺序）
   - 叙事时间线（读者接收信息的顺序）
   - 两条时间线的错位设计（悬疑的核心来源）

五、悬念管理表
   | 悬念 | 建立章节 | 强化章节 | 揭示章节 | 类型(谜团/威胁/秘密) |
```

**结构原则**（融合 Syd Field 与 Blake Snyder）：
- **"承诺"原则**：第一幕末尾向读者做出的类型承诺，第三幕必须兑现
- **"进展复杂化"原则**：每个调查行动必须要么获得新线索，要么引发新问题
- **"不可逆转"原则**：中点之后，主角无法退出，stakes 只升不降

### `/场景 {章节号}` — 逐章创作

**触发条件**：`/结构` 完成后
**加载参考**：`references/suspense-technique.md`, `references/dialogue-interrogation.md`

**输出格式**（生成 `chapters/ch{NNN}.md`）：

```markdown
# 第 N 章：{章节标题}

## 场景信息
- POV：{视角人物}
- 地点：{场景地点}
- 时间：{故事内时间}
- 本章核心功能：{推进调查 / 人物塑造 / 线索埋设 / 转折 / 揭示}

## 正文
{800-3000字正文}

## 章末钩子
{下一章悬念设计}

---
### 创作备注（不进入正文）
- 本章埋设的线索：{编号列表}
- 本章的误导操作：{描述}
- 嫌疑指向变化：{角色A ↑, 角色B ↓}
- 读者此刻应有的推理状态：{描述}
```

**场景写作原则**（引自 Hitchcock 悬念理论）：
- **悬念 ≠ 惊吓**：观众知道桌下有炸弹（悬念）vs 炸弹突然爆炸（惊吓）。优先使用悬念
- **戏剧反讽**：当读者比角色知道更多时，张力最大化
- **场景内弧线**：每个场景都有自己的"期望→障碍→转折"微结构
- **"迟到的主角"原则**：让读者在主角到达前就感受到危险

### `/审线索` — 线索一致性审查

**触发条件**：任何创作阶段均可调用
**加载参考**：`references/clue-design.md`

**执行逻辑**：
1. 扫描所有已写章节，提取所有线索出现点
2. 与 `trick-design.md` 中的线索布局对照
3. 输出审查报告：

```
一、线索完整性
   - 已布设的关键线索：✅ / ❌
   - 未布设的关键线索：{列表，建议插入章节}

二、线索可见性
   - 各线索的"显眼程度"评估（过于明显 / 恰当 / 过于隐蔽）

三、红鲱鱼状态
   - 各误导线索是否已激活 / 是否已消解

四、公平性评分
   - Knox 十诫逐条检查
   - 总评：公平 / 基本公平 / 不公平（需修改）

五、漏洞检测
   - 逻辑矛盾
   - 时间线冲突
   - 角色行为不一致
```

### `/自检 {章节号}` — 单章质量评估

**触发条件**：章节写作完成后

**评分维度**（每项 1-10 分，满分 60 分）：

| 维度 | 评估要点 |
|------|----------|
| 悬念张力 | 信息不对称是否有效？读者是否有继续阅读的欲望？ |
| 线索技术 | 线索埋设是否自然？误导是否有说服力？ |
| 人物一致性 | 角色行为是否符合已建立的性格？对话是否有辨识度？ |
| 节奏控制 | 场景长度是否合理？张弛是否有度？ |
| 逻辑严密性 | 推理链条是否完整？有无漏洞？ |
| 文学质量 | 叙述是否有风格？意象是否有效？是否有"金句"？ |

**评级**：<30 需重写 → 30-40 可接受 → 40-50 良好 → 50-60 优秀

### `/揭示` — 真相揭示场景专项

**触发条件**：进入第三幕
**加载参考**：`references/twist-design.md`

**设计框架**：

```
一、揭示方式选择
   - 侦探推理展示（古典型：波洛式集合推理）
   - 犯人自白（心理型：动机驱动的告白）
   - 逐步剥离（层层反转，每次揭示推翻前一个认知）
   - 读者先于侦探（倒叙揭示，展示犯罪过程）
   - 对决中揭示（追逐/对抗中真相浮出）

二、揭示节奏设计
   - 预揭示（暗示，让聪明读者提前 0.5 秒猜到）
   - 核心揭示（真相公开的精确时刻）
   - 后揭示（揭示后的情感余波与意义升华）

三、反转层级（如适用）
   - 第一层反转：推翻表面假设
   - 第二层反转：推翻第一层反转建立的新认知
   - （慎用）第三层反转：终极真相

四、情感落点
   - 揭示后读者应有的情感反应
   - 主题在揭示中的体现
```

**揭示原则**（引自 Agatha Christie 与 东野圭吾 创作方法）：
- **"啊哈"时刻**：最佳揭示让读者发出"原来如此"而非"怎么可能"
- **回溯验证**：揭示后读者回想前文，每个"奇怪"之处都获得解释
- **情感优先**：技术性诡计揭示必须伴随情感冲击

## 工作流程序

```
/开始 → /诡计设计 → /人物 → /结构 → /场景 1..N → /审线索 → /自检 → /揭示 → /导出
                                            ↑                    |
                                            └────── 迭代修改 ─────┘
```

**严格顺序要求**：不可跳过前置阶段。每个阶段的输出是下一阶段的输入依赖。`/审线索` 和 `/自检` 可在任何创作阶段随时调用。
