# 悬疑小说剧本创作系统 — Thriller Screenplay Craft

> 基于专业编剧方法论的 AI 悬疑/推理小说剧本创作技能包
> 融合 Robert McKee《故事》、Syd Field《电影剧本写作基础》、Blake Snyder《救猫咪》、Raymond Chandler 推理写作法则、Alfred Hitchcock 悬念理论

---

## 系统定位

本技能专注于**悬疑/推理/惊悚**类型的小说与剧本创作，提供从构思到成稿的全流程系统化指导。覆盖：

- 本格推理（公平解谜型）
- 社会派推理（动机驱动型）
- 心理悬疑（不可靠叙事型）
- 惊悚悬疑（高压紧迫型）
- 密室推理（封闭空间型）
- 叙述性诡计（元叙事型）
- 犯罪悬疑（警探/侦探程序型）
- 倒叙推理（已知犯人型/Howcatchem）
- 日常推理（Cozy Mystery/轻推理型）

### `/theme` — 主题前提与立意

**触发条件**：`/start` 完成后（短篇模式下已在 `/start` 中完成，可跳过）
**加载参考**：`references/mystery-structure.md`

> "没有前提的悬疑只是谜题游戏；有前提的悬疑才是文学。" — 改编自 Lajos Egri《戏剧写作的艺术》

**输出结构**（生成 `theme.md`）：

```
一、主题前提（一句话）
   - 格式："{某种人性/社会现象} 导致 {某种结果}"
   - 例："对控制的执念最终摧毁了控制者自己"
   - 例："真相的代价有时高于谎言的代价"
   - 例："制度性的冷漠比个体的恶意更加致命"

二、主题在故事中的体现
   - 犯罪如何体现主题？（犯罪的本质是主题的极端表达）
   - 侦探的弧光如何回应主题？（破案过程即主题的论证过程）
   - 揭示如何升华主题？（真相揭示的那一刻，主题获得最终论证）

三、主题的三维验证（Egri 方法）
   - 生理维度：主题如何在角色的物理世界中体现？
   - 社会维度：主题如何反映社会结构/权力关系？
   - 心理维度：主题如何触及角色的内在冲突？

四、主题与子类型的适配
   - 本格推理：主题通过诡计机制隐喻表达
   - 社会派：主题即犯罪的社会根因
   - 心理悬疑：主题通过角色的心理困境表达
   - 惊悚：主题通过极端处境下的选择表达
```

**设计原则**（引自 Lajos Egri《戏剧写作的艺术》）：
- 前提必须是**可辩论的**——如果没有人会反对这个前提，它就不够有力
- 前提必须是**可论证的**——故事的结局是前提的"证明"
- 犯罪本身就是主题的极端表现形式

### `/trick` — 核心谜团架构

**触发条件**：`/theme` 完成后（本格/密室/叙述诡计型优先），或 `/characters` 完成后（社会派/心理悬疑型可后置）
**加载参考**：`references/clue-design.md`, `references/red-herring.md`, `references/twist-design.md`, `references/knowledge-state.md`

**输出结构**（生成 `trick-design.md`）：

```
一、核心诡计概述
   - 谜面（读者看到的表象）
   - 谜底（真实发生的事情）
   - 诡计机制（表象如何形成）

二、公平性验证（Knox 十诫 / Van Dine 二十则 适用项检查）
   - 线索是否全部呈现给读者
   - 是否依赖未公开信息
   - 是否违反已建立的世界规则

三、线索布局
   - 关键线索清单（编号、出现章节、呈现方式、读者注意力预测）
   - 误导线索清单（编号、指向的错误结论、消解方式）
   - 线索-时间线交叉表

四、诡计可行性分析
   - 物理可行性（密室/不在场证明类必须）
   - 心理可行性（动机/行为逻辑）
   - 信息控制可行性（读者视角管理）

五、反派计划骨架（James Frey 反派引擎法）
   - 反派的完整计划（分步骤）
   - 每步执行留下的痕迹 → 这些痕迹 = 线索
   - 侦探如何发现每条痕迹 → 发现过程 = 情节
   - 反派如何反击 → 反击 = 冲突升级

六、知识状态矩阵（参照 knowledge-state.md）
   - 角色知识矩阵：每角色每章知道什么
   - 读者知识层：读者确知/推测/误信/不知
   - 双读者验证：天真读者(惊奇) + 侦探读者(原来如此)
```

**设计原则**（引自 Raymond Chandler & Ellery Queen）：
- **解答唯一性**：同一组线索只能指向唯一真相
- **线索公平性**：至少 3 条关键线索在揭示前已呈现
- **误导合理性**：红鲱鱼必须有独立存在的叙事理由
- **逻辑闭环**：诡计揭示后，所有前文"不合理"之处获得解释

### `/characters` — 角色系统设计

**触发条件**：`/trick` 完成后
**加载参考**：`references/character-archetype.md`

**输出结构**（生成 `characters.md`）：

```
一、侦探/主角
   - 人物档案（姓名、背景、性格、核心缺陷、侦探方法论）
   - Watson 角色（助手/叙述者/对照）
   - 侦探的个人案件（Personal Stakes）

二、犯人/反派
   - 第一印象设计（读者初见时的人设）
   - 真实身份与动机
   - 犯罪能力来源（技能、资源、机会）
   - 犯人的自洽叙事（ta 如何合理化自己的行为）
   - 暴露轨迹（从完美隐藏到逐步暴露的关键节点）

三、嫌疑人矩阵
   标准模式（5-8人）：
   | 角色 | 动机 | 机会 | 能力 | 不在场证明 | 嫌疑指数曲线 |

   极简模式（2-3人，适用于心理悬疑/短篇/极简阵容）：
   | 角色 | 与主角的关系 | 表面嫌疑 | 真实角色 | 何时洗清/暴露 |
   注：极简模式下每个嫌疑人承担更多叙事功能（红鲱鱼+人物深化+主题映射）

四、受害者
   - 受害者不是 NPC：赋予完整人格与秘密
   - 受害者的 N 个面孔（不同角色眼中的受害者）
   - 受害者的秘密（与案件的真正关联）

五、关系网络图
   - 角色两两关系定义
   - 秘密链（A 知道 B 的秘密，B 知道 C 的秘密...）
   - 冲突矩阵
```

**角色设计原则**（引自 McKee《故事》）：
- **压力下的选择揭示人物**：每个关键角色至少有一个"两难选择"时刻
- **人物弧光与案件弧光交织**：主角在破案过程中完成内在转变
- **嫌疑人不是工具人**：每个嫌疑人都有独立于案件的个人故事线

### `/structure` — 叙事结构设计

**触发条件**：`/characters` 完成后
**加载参考**：`references/mystery-structure.md`, `references/pacing-tension.md`

**输出结构**（生成 `structure.md`）：

```
一、结构选择与论证
   - 所选结构模型（三幕式 / 英雄旅程 / 起承转合 / 倒叙 / 多线交织）
   - 为什么这个故事适合这个结构

二、三幕式（悬疑变体）适配
   第一幕·建立（全文 25%）
   - 日常→事件→卷入
   - 关键节点：尸体/案件发现点（Inciting Incident）
   - 完成条件：读者了解所有主要角色，第一个谜团建立

   第二幕·调查与复杂化（全文 50%）
   - 前半：表面调查，收集线索，建立错误假设
   - 中点逆转（Midpoint Reversal）：颠覆前半部分建立的认知
   - 后半：深层真相浮现，个人风险升级
   - 第二幕终点：最黑暗时刻 / 侦探陷入困境

   第三幕·揭示与解决（全文 25%）
   - 最终线索的获取
   - 推理/对决场景
   - 真相揭示
   - 余波与主题升华

三、章节大纲
   | 章节 | 标题 | 核心事件 | 线索操作 | 嫌疑转移 | 张力指数 |

四、时间线
   - 故事时间线（案件实际发生的顺序）
   - 叙事时间线（读者接收信息的顺序）
   - 两条时间线的错位设计（悬疑的核心来源）

五、悬念管理表
   | 悬念 | 建立章节 | 强化章节 | 揭示章节 | 类型(谜团/威胁/秘密) |
```

**结构原则**（融合 Syd Field 与 Blake Snyder）：
- **"承诺"原则**：第一幕末尾向读者做出的类型承诺，第三幕必须兑现
- **"进展复杂化"原则**：每个调查行动必须要么获得新线索，要么引发新问题
- **"不可逆转"原则**：中点之后，主角无法退出，stakes 只升不降

### `/scene {章节号}` — 逐章创作

**触发条件**：`/structure` 完成后
**加载参考**：`references/suspense-technique.md`, `references/dialogue-interrogation.md`

**输出格式**（生成 `chapters/ch{NNN}.md`）：

```markdown
# 第 N 章：{章节标题}

## 场景信息
- POV：{视角人物}
- 地点：{场景地点}
- 时间：{故事内时间}
- 本章核心功能：{推进调查 / 人物塑造 / 线索埋设 / 转折 / 揭示}

## 正文
{800-3000字正文}

## 章末钩子
{下一章悬念设计}

---
### 创作备注（不进入正文）
- 本章埋设的线索：{编号列表}
- 本章的误导操作：{描述}
- 嫌疑指向变化：{角色A ↑, 角色B ↓}
- 读者此刻应有的推理状态：{描述}
```

**场景写作原则**（引自 Hitchcock 悬念理论）：
- **悬念 ≠ 惊吓**：观众知道桌下有炸弹（悬念）vs 炸弹突然爆炸（惊吓）。优先使用悬念
- **戏剧反讽**：当读者比角色知道更多时，张力最大化
- **场景内弧线**：每个场景都有自己的"期望→障碍→转折"微结构
- **"迟到的主角"原则**：让读者在主角到达前就感受到危险

### `/audit` — 线索一致性审查

**触发条件**：任何创作阶段均可调用
**加载参考**：`references/clue-design.md`

**执行逻辑**：
1. 扫描所有已写章节，提取所有线索出现点
2. 与 `trick-design.md` 中的线索布局对照
3. 输出审查报告：

```
一、线索完整性
   - 已布设的关键线索：✅ / ❌
   - 未布设的关键线索：{列表，建议插入章节}

二、线索可见性
   - 各线索的"显眼程度"评估（过于明显 / 恰当 / 过于隐蔽）

三、红鲱鱼状态
   - 各误导线索是否已激活 / 是否已消解

四、公平性评分（按子类型过滤适用条目）
   - 本格/密室：Knox 全部 10 条 + Van Dine 核心条目
   - 犯罪程序：Knox #1,#2,#4,#6,#8 + 程序真实性检查
   - 心理悬疑：Knox #1,#8 + 不可靠叙述者公平性协议（参照 unreliable-narrator.md）
   - 叙述诡计：Knox #7,#8,#10 + 文字层面可追溯性检查
   - 社会派/惊悚/日常推理：Knox #1,#2,#6 + 动机合理性检查
   - 倒叙推理：Knox 不适用（读者已知真相），改为侦探推理链完整性检查
   - 双读者验证（参照 knowledge-state.md）：天真读者(惊奇) + 侦探读者(原来如此)
   - 总评：公平 / 基本公平 / 不公平（需修改）

五、漏洞检测
   - 逻辑矛盾
   - 时间线冲突
   - 角色行为不一致
```

### `/check {章节号}` — 单章质量评估

**触发条件**：章节写作完成后

**评分维度**（每项 1-10 分，满分 60 分，按子类型加权）：

| 维度 | 评估要点 | 本格 | 社会派 | 心理 | 惊悚 | 密室 | 犯罪 | 日常 |
|------|----------|------|--------|------|------|------|------|------|
| 悬念张力 | 信息不对称、阅读欲望 | ×1.0 | ×1.0 | ×1.2 | ×1.5 | ×1.0 | ×1.2 | ×0.8 |
| 线索技术 | 线索自然度、误导力 | ×1.5 | ×1.0 | ×1.0 | ×0.8 | ×1.5 | ×1.2 | ×1.0 |
| 人物一致性 | 行为/对话辨识度 | ×0.8 | ×1.2 | ×1.5 | ×1.0 | ×0.8 | ×1.0 | ×1.5 |
| 节奏控制 | 张弛、场景长度 | ×1.0 | ×1.0 | ×1.0 | ×1.2 | ×1.0 | ×1.0 | ×1.0 |
| 逻辑严密性 | 推理链、漏洞 | ×1.5 | ×1.0 | ×0.8 | ×0.8 | ×1.5 | ×1.2 | ×0.8 |
| 文学质量 | 风格、意象、金句 | ×0.8 | ×1.2 | ×1.5 | ×0.8 | ×0.8 | ×0.8 | ×1.5 |

加权后总分 = Σ(原始分 × 权重) / Σ(权重) × 10，仍按 60 分制评级

**评级**：<30 需重写 → 30-40 可接受 → 40-50 良好 → 50-60 优秀

### `/reveal` — 真相揭示场景专项

**触发条件**：进入第三幕
**加载参考**：`references/twist-design.md`

**设计框架**：

```
一、揭示方式选择
   - 侦探推理展示（古典型：波洛式集合推理）
   - 犯人自白（心理型：动机驱动的告白）
   - 逐步剥离（层层反转，每次揭示推翻前一个认知）
   - 读者先于侦探（倒叙揭示，展示犯罪过程）
   - 对决中揭示（追逐/对抗中真相浮出）

二、揭示节奏设计
   - 预揭示（暗示，让聪明读者提前 0.5 秒猜到）
   - 核心揭示（真相公开的精确时刻）
   - 后揭示（揭示后的情感余波与意义升华）

三、反转层级（如适用）
   - 第一层反转：推翻表面假设
   - 第二层反转：推翻第一层反转建立的新认知
   - （慎用）第三层反转：终极真相

四、情感落点
   - 揭示后读者应有的情感反应
   - 主题在揭示中的体现
```

**揭示原则**（引自 Agatha Christie 与 东野圭吾 创作方法）：
- **"啊哈"时刻**：最佳揭示让读者发出"原来如此"而非"怎么可能"
- **回溯验证**：揭示后读者回想前文，每个"奇怪"之处都获得解释
- **情感优先**：技术性诡计揭示必须伴随情感冲击

## 工作流程序

```
路线 A（本格/密室/叙述诡计——诡计优先）：
  /start → /theme → /trick → /characters → /structure → /scene 1..N → /revise → /reveal → /export

路线 B（社会派/心理悬疑——角色优先）：
  /start → /theme → /characters → /trick → /structure → /scene 1..N → /revise → /reveal → /export

路线 C（惊悚/犯罪程序——结构优先）：
  /start → /theme → /structure → /characters → /trick → /scene 1..N → /revise → /reveal → /export

通用规则：
  - /audit 和 /check 可在任何创作阶段随时调用
  - /revise 可在任何阶段反复调用
  - 每个阶段的输出是后续阶段的输入依赖
```

### `/revise` — 修改与打磨

**触发条件**：任何创作阶段均可调用，建议在全部初稿完成后系统执行
**加载参考**：根据改稿类型加载对应参考

**四种改稿模式**：

| 模式 | 关注点 | 检查内容 |
|------|--------|----------|
| 结构改稿 | 宏观节奏 | 三幕比例、中点位置、节拍缺失、张力曲线、MICE 嵌套闭合顺序 |
| 角色改稿 | 人物一致性 | 每人有独立声音？弧光完整？动机罗盘四维完整？嫌疑曲线合理？ |
| 线索改稿 | 推理逻辑 | 调用 `/audit` + 公平性验证 + 红鲱鱼五阶段生命周期 + 知识状态矩阵 |
| 文字改稿 | 逐章润色 | 删除冗余、强化意象、打磨对话、统一文风（参照 prose-craft.md） |

**自我编辑信法**（Elizabeth George 方法）：
在全稿初稿完成后，用 1-2 天快速通读全文，然后给自己写一封"编辑来信"：
- 结构层：哪里拖沓？哪里太急？中点是否有力？
- 角色层：哪个角色最弱？谁的声音和别人太像？
- 线索层：哪条线索太明显？哪条太隐蔽？红鲱鱼消解得是否满意？
- 情感层：哪里应该更痛？哪里应该更温暖？揭示的情感落点到位吗？
这封信成为改稿的路线图。

**输出**：修改建议清单 + 具体修改方案（逐章）
