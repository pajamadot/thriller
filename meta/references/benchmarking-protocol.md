# 外部方法论对标协议

> 如何系统地评估和整合外部知识源
> 不是盲目吸收，也不是傲慢拒绝——是批判性整合

---

## 一、外部知识源分类

### 按可靠性层级

| 层级 | 来源类型 | 可靠性 | 整合策略 |
|------|----------|--------|----------|
| A | 经典理论著作（McKee、Egri 等） | 高（经过时间验证） | 整合为原理 |
| B | 当代专业书籍/课程 | 中-高 | 整合为实践 |
| C | 成功创作者的经验分享 | 中（幸存者偏差） | 整合为实验 |
| D | 社交媒体/论坛讨论 | 低（未验证） | 仅作为信号 |
| E | 自己的直觉/灵感 | 不定 | 必须实践验证 |

### 按知识类型

```
声明性知识："X 是什么"
  例：McKee 定义的"场景"概念
  整合方式：直接纳入概念框架

程序性知识："怎么做 X"
  例：如何写一个审讯场景的具体步骤
  整合方式：测试后纳入技术工具箱

条件性知识："什么时候用 X"
  例：什么类型的故事适合倒叙开场
  整合方式：纳入决策矩阵

元认知知识："怎么思考 X"
  例：如何判断一个线索是否"公平"
  整合方式：最有价值，直接影响思维框架
```

## 二、对标流程

### Phase 1：理解外部方法

```
在评判之前先理解。

  1. 用外部方法的自己的术语描述它
     （不要急于翻译成我们的术语）

  2. 识别它的核心主张
     这个方法的 1-3 个最核心的观点是什么？

  3. 识别它的隐含假设
     这个方法假设了什么？
     （读者类型？创作语境？媒介？文化？）

  4. 识别它的适用范围
     这个方法在什么条件下最有效？
     在什么条件下可能失效？
```

### Phase 2：差异映射

```
逐条对比外部方法与本系统：

  | 主题 | 外部方法的观点 | 本系统的观点 | 关系 |

  关系类型：
  - 一致：双方观点相同 → 增加置信度
  - 互补：双方覆盖不同方面 → 有整合价值
  - 矛盾：双方观点冲突 → 需要辩论
  - 新增：外部方法覆盖了我们没有的领域 → 考虑填充
  - 多余：外部方法的内容我们已有更好的版本 → 可忽略
```

### Phase 3：批判性评估

```
对每个有价值的差异点：

  1. 证据评估
     外部方法的这个观点基于什么证据？
     - 理论推导？
     - 个人经验？
     - 多人验证？
     - 实验数据？
     - 历史案例？

  2. 语境评估
     外部方法的语境与我们是否匹配？
     - 语言/文化相同？
     - 媒介相同？（小说 vs 电影 vs 游戏）
     - 目标受众相同？
     - 创作模式相同？（AI辅助 vs 纯人工）

  3. 整合成本评估
     吸收这个观点需要修改多少已有内容？
     - 低成本：只需新增一段
     - 中成本：需要修改已有章节
     - 高成本：需要重构模块
```

### Phase 4：整合决策

```
决策矩阵：

                高价值
                  │
        吸收      │     深入研究
        (值得改)   │     (值得但成本高)
                  │
  ──────────────┼──────────────── 高成本
                  │
        快速纳入   │     记录备忘
        (容易改)   │     (不值得改)
                  │
                低价值

四种决策：
  A. 快速纳入：低成本 + 有价值 → 立即整合
  B. 吸收：中成本 + 高价值 → 安排在下次进化中
  C. 深入研究：高成本 + 高价值 → 需要更多证据
  D. 记录备忘：低价值 → 记录但不行动
```

## 三、常见对标来源及关注重点

### 编剧/写作类书籍

```
关注：结构方法、角色理论、对话技术
警惕：大多数基于电影/电视，可能不直接适用于小说/互动叙事
整合策略：提取原理，重新设计实践
```

### 游戏叙事设计

```
关注：互动叙事模式、玩家心理、状态管理
警惕：游戏玩家 ≠ 小说读者，互动节奏不同
整合策略：选择性吸收互动设计原理
```

### 认知心理学/行为经济学

```
关注：读者决策心理、注意力管理、记忆机制
警惕：实验室研究不等于自然阅读体验
整合策略：用来验证或质疑经验性规则
```

### 成功创作者的访谈/自述

```
关注：真实创作过程的洞见
警惕：事后合理化、幸存者偏差、不可复制的天才
整合策略：提取可复制的技术，忽略个人风格
```

## 四、对标输出模板

```markdown
# Benchmark: {来源名}

## 来源信息
- 类型：{书籍/课程/访谈/作品分析}
- 可靠性层级：{A/B/C/D/E}
- 语境匹配度：{高/中/低}

## 核心主张
1. {主张1}
2. {主张2}

## 差异映射
| 主题 | 外部观点 | 本系统观点 | 关系 | 决策 |

## 整合行动
| 行动 | 优先级 | 影响文件 | 状态 |

## 保留的冲突
| 冲突 | 我们的立场 | 理由 | 是否需要更多数据 |
```
