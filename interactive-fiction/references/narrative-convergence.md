# 叙事收敛与一致性

> 分支叙事中最难的问题：如何让不同路径的故事都讲得通
> 参考 Jon Ingold（inkle）的"涟漪"理论、Chris Crawford 互动叙事思想

---

## 一、收敛的核心挑战

### 分支叙事的根本矛盾

```
矛盾：
  读者期望 → 我的选择造成了真实的改变
  创作现实 → 无限分支在创作上不可行

解决思路：
  不是减少变化 → 而是让有限的变化感觉重大
  不是分支越多越好 → 而是让每条分支都有质量
```

### 三种收敛策略

| 策略 | 描述 | 适用 |
|------|------|------|
| 硬收敛 | 不同路径最终到达相同的叙事节点 | 瓶颈型拓扑 |
| 软收敛 | 不同路径到达"功能等价"的节点（位置不同但剧情功能相同） | 平行调查型 |
| 延迟分化 | 选择的影响不立即显现，在后期才分化 | 信任网络型 |

## 二、硬收敛技术

### 汇聚点写作技术

当多条路径汇聚到同一个节点时：

```
问题：路径 A 的读者经历了"仓库爆炸"
      路径 B 的读者经历了"安静的审讯"
      如何让他们在汇聚点的体验都自洽？

技术 1：事件驱动汇聚
  设计一个足够强大的事件，迫使所有路径的角色到达同一状态
  例：新的谋杀案发生 → 不管你之前在做什么，都必须到现场

技术 2：信息驱动汇聚
  不同路径获取了不同信息，汇聚点是信息整合
  例：团队会议 → 每人分享各自的发现

技术 3：时间驱动汇聚
  外部时间节点迫使汇聚
  例：嫌疑人的保释听证 → 不管查到什么，这个日期到了

技术 4：条件文本汇聚
  同一节点用条件判断呈现不同的文本片段
  核心叙事相同，但细节根据路径变化
```

### 汇聚点的"记忆"处理

```
读者的选择不应在汇聚点被"遗忘"

方法 1：引用过去
  "你想起了在仓库看到的那些文件。"（如果走了仓库路线）
  "你想起了陈某在审讯室里不安的表情。"（如果走了审讯路线）

方法 2：态度差异
  走过危险路线的读者 → 角色更谨慎
  走过信息路线的读者 → 角色更自信

方法 3：NPC 记忆
  "上次你帮了我，所以这次我告诉你一件事..."
  "你上次没有理会我，现在凭什么来问？"
```

## 三、角色一致性管理

### 跨路径角色一致性

```
核心原则：角色的基本人格在所有路径中不变
          变化的是：角色对读者的态度、角色展现出的面向

一致层（永远不变）：
  - 核心价值观
  - 基本性格特质
  - 背景故事
  - 与案件的真实关系

可变层（随路径变化）：
  - 对读者角色的信任程度
  - 展示出的信息量
  - 情感反应的强度
  - 最终命运

绝对禁止：
  ✗ 同一角色在不同路径中有矛盾的背景故事
  ✗ 角色在某条路径中做出违反基本人格的行为（无合理原因）
  ✗ 角色的秘密在不同路径中不同
```

### 角色的"冰山一致性"

```
冰山上方（读者可见）：可以因路径不同而不同
  - 角色说了什么
  - 角色做了什么
  - 角色对读者的态度

冰山下方（角色本质）：在所有路径中一致
  - 角色的真实动机
  - 角色的秘密
  - 角色的恐惧和欲望
  - 角色的过去

这意味着：不同路径中角色的不同表现，
都可以被理解为"同一个人在不同压力下的不同反应"
```

## 四、线索公平性跨路径保证

### 推理类互动叙事的特殊要求

```
公平性原则：
  每个可达结局所需的关键线索，必须在到达该结局的路径上可获取

验证方法：
  对每个结局 E：
    列出理解 E 所需的关键信息：[I1, I2, I3, ...]
    对每条到达 E 的路径 P：
      检查 P 上是否可以获取 [I1, I2, I3, ...]
      如果缺少 I_n → 需要在该路径上增加获取点

线索冗余设计：
  关键线索不应只有一个获取途径
  至少 2 条路径可以获取同一条关键线索
  （获取方式可以不同：直接发现 / 角色告知 / 推理得出）
```

### 信息不对称的叙事价值

```
不同路径获取不同信息 → 不同的推理角度 → 不同但都合理的结论

这是互动推理的核心魅力：
  路径 A 的读者：看到的信息导向结论 X
  路径 B 的读者：看到的信息导向结论 Y
  两者都是"合理的推理"，但可能只有一个是"完整的真相"

设计目标：让读者在完成一条路线后想"如果我选了另一条会怎样"
```

## 五、叙事一致性检查流程

### 系统化检查方法

```
Step 1：路径枚举
  列出所有可能的路径（或抽样关键路径）
  重点检查：
  - 最短路径
  - 最长路径
  - "全左"路径和"全右"路径
  - 随机路径 × 5

Step 2：逐路径阅读
  从每条路径的视角完整阅读
  标记：
  - 逻辑断裂点（"我怎么知道这个？"）
  - 角色不一致（"ta 上次不是这个态度吗？"）
  - 信息缺口（"这个名字/事件没有被提及过"）

Step 3：跨路径对比
  比较同一节点在不同路径到达时的体验
  检查：
  - 汇聚节点是否所有路径都通顺
  - 分支后的文本是否引用了只在某条路径中存在的事件
  - 角色是否记住了不应该记住的事情

Step 4：结局验证
  对每个结局：
  - 结局的情感是否匹配路径的体验？
  - 揭示的信息读者是否有足够的上下文理解？
  - 结局是否引用了路径中确实发生的事件？
```

## 六、"涟漪"模型

### Jon Ingold 的涟漪理论

```
选择的影响像水面的涟漪：

  投入石子（做出选择）
       ↓
  第一圈涟漪（即时影响：下一段文本不同）
       ↓
  第二圈涟漪（短期影响：角色态度变化）
       ↓
  第三圈涟漪（中期影响：后续可用选项不同）
       ↓
  第四圈涟漪（长期影响：影响结局）

涟漪的衰减：
  大部分选择的涟漪在 2-3 个节点后衰减
  少数关键选择的涟漪贯穿全文

设计建议：
  - 每个选择至少有"第一圈涟漪"（即时可感知的差异）
  - 20% 的选择应有"第三圈涟漪"（影响后续可用选项）
  - 3-5 个核心选择有"第四圈涟漪"（影响结局）
```

### 涟漪的实现方式

```
轻量实现：条件文本
  在后续节点中插入 if-then 文本块
  成本低，但只改变表面文字

中量实现：路径分化
  选择后进入不同的子路径（3-5个节点后汇合）
  成本中等，但差异明显

重量实现：永久分支
  选择后永久进入不同的故事线
  成本高，但选择感最强
  仅用于核心关键选择
```
