# 互动叙事节奏

> 当读者控制节奏时，如何仍然保持戏剧张力
> 参考 inkle 叙事设计、Choice of Games 节奏指南

---

## 一、互动节奏 vs 线性节奏

```
线性小说：作者 100% 控制节奏
互动小说：作者控制结构节奏，读者控制体验节奏

结构节奏（作者控制）：
  - 决策点之间的叙事节点数量
  - 高张力/低张力段落的安排
  - 强制性剧情事件的位置

体验节奏（读者控制）：
  - 在决策点停留多久
  - 是否探索可选内容
  - 阅读速度
```

## 二、决策密度管理

```
决策密度 = 每千字中的决策点数量

过密（每 200-300 字一次选择）：
  ✗ 读者无法沉浸在叙事中
  ✗ 选择变成机械操作
  ✗ 没有足够上下文做有意义的判断

过疏（每 2000+ 字一次选择）：
  ✗ 读者失去参与感
  ✗ 感觉像在读普通小说
  ✗ 选择来临时已经遗忘了上下文

理想密度（按场景类型）：
  调查/探索段落：每 500-800 字一次（策略选择）
  对话段落：每 300-500 字一次（回应选择）
  高张力段落：每 200-400 字一次（紧急决策）
  揭示/结局段落：每 800-1500 字一次（沉浸体验）
```

## 三、张力在互动中的维持

### 强制高潮节点

```
某些节点不受分支影响——所有路径必须经过：

  中点反转节点：不管之前怎么选，这个事件都会发生
  危机节点：所有路径汇聚到最黑暗时刻
  揭示节点：真相在此公开（细节可因路径不同而变化）

这些"强制节点"保证了宏观张力曲线不被分支破坏。
```

### 选择后果的延迟

```
最好的互动悬念 = 选择的后果不立即显现

  做出选择 → 2-3 个节点后后果显现
  → 读者在等待中想"我选对了吗？"
  → 这就是互动独有的悬念形式

实现：用状态变量记录选择，在后续节点用条件文本反映后果
```

## 四、"穷尽探索者"问题

```
部分读者会在每个分支点保存，探索所有选项后再推进。

应对策略：
  1. 接受它——设计时考虑这类读者的体验
  2. 信息累积型选择：探索越多获得越多信息（不惩罚穷尽探索）
  3. 互斥信息：某些选择获得 A 信息就不能获得 B（维持重玩价值）
  4. 时间压力叙事（不是真实计时）：用叙事语言制造紧迫感
     "你只有时间做一件事" → 暗示不能两全
```

## 五、交替选择权重

```
决策负荷应该交替轻重：

  轻 → 轻 → 重 → 轻 → 轻 → 重 → 轻 → 极重（高潮选择）

  轻选择：调查方向、对话语气、观察哪个细节
  重选择：信任/背叛、冒险/安全、指控/沉默
  极重选择：最终揭示、关键牺牲、道德抉择

  连续两个重选择 = 决策疲劳
  连续四个轻选择 = 读者觉得选择不重要
```
