# Convergence Discipline Checklist

Use this checklist when multiple routes should merge without feeling flattened or contradictory.

## Shared Node Contract

- Every merged node declares `routeMemory` for the prior play it needs to remember.
- Every merged node with `routeMemory` also declares `mergeCallbacks`.
- Merge callback variables are a subset of the node's `routeMemory`.

## Reader Legibility

- Shared body text does not assume facts absent on some incoming paths.
- Route history is preserved through callback pressure, altered access, or visible posture shifts.
- The merge keeps a shared dramatic question without pretending the routes were identical.

## Structural Discipline

- Do not duplicate whole scenes when a callback contract can preserve the route texture.
- Do not merge too early if the route still needs one more local consequence beat.
- If a merged node forgets meaningful prior play, either widen its callback layer or delay the merge.

## Red-Team Layer

- A hostile reading can name which merge currently washes prior play away.
- The route can point to one callback or altered access beat that now survives the merge.
- The next band can name which convergence is still too blunt.
