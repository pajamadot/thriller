# 选择设计系统

> 互动叙事的核心：如何设计让读者感到有意义的选择
> 参考 Sam Kabo Ashwell 的选择分类法、Jon Ingold（inkle）的叙事选择理论

---

## 一、选择的七种类型

### 1. 策略选择（Strategic Choice）
读者基于已知信息做出最优判断

```
场景：你有一个小时的调查时间
选项 A：去搜查受害者的办公室
选项 B：去审问目击证人
选项 C：去查看监控录像

特征：
  - 每个选项有明确的信息收益
  - 不同选择获得不同线索
  - 没有"正确"答案，但有权衡取舍

最佳实践：让读者事后觉得另一个选择"也想试试"
```

### 2. 道德选择（Moral Choice）
读者面临价值观的冲突

```
场景：你发现朋友与案件有关
选项 A：如实向警方报告
选项 B：先找朋友私下了解
选项 C：暂时隐瞒这个发现

特征：
  - 没有客观最优解
  - 反映读者的价值判断
  - 选择后的后果应验证选择的重量

最佳实践：后果不应简单地"惩罚"或"奖励"任一选择
```

### 3. 性格选择（Personality Choice）
定义/强化读者扮演的角色性格

```
场景：嫌疑人在审讯中崩溃哭泣
选项 A：温柔地递上纸巾（同理心路线）
选项 B：继续施压提问（效率路线）
选项 C：给 ta 独处的时间（观察路线）

特征：
  - 不直接影响剧情走向
  - 影响关系状态和对话变体
  - 累积形成角色"画像"

最佳实践：让性格选择在后续场景中被角色记住和引用
```

### 4. 信息选择（Information Choice）
决定获取哪些信息

```
场景：你在犯罪现场
选项 A：仔细检查尸体（获得法医线索）
选项 B：检查门窗和出入口（获得物理线索）
选项 C：查看受害者的手机（获得社交线索）

特征：
  - 最适合推理类型
  - 选择决定读者掌握的信息
  - 影响后续推理的可能性

最佳实践：每个选项都应提供有价值的信息，但各有侧重
```

### 5. 风险选择（Risk Choice）
高风险高回报 vs 低风险低回报

```
场景：你知道嫌疑人今晚会去某个地方
选项 A：跟踪 ta（高风险：可能被发现，但能获取关键证据）
选项 B：通知警方（低风险：安全但可能打草惊蛇）

特征：
  - 读者评估风险收益比
  - 高风险选择可能导致重大好处或重大危险
  - 制造真正的紧张感

最佳实践：高风险不应总是导致坏结果（否则读者学会永远选安全项）
```

### 6. 时间压力选择（Timed Choice）
在有限时间/资源下做出选择

```
场景：炸弹将在 5 分钟后爆炸
选项 A：尝试拆除（技术挑战）
选项 B：疏散人群（放弃线索但保住生命）
选项 C：搜索现场获取最后的证据（与时间赛跑）

特征：
  - 紧迫感增加情感投入
  - 不可能"两全"
  - 展示优先级价值观

最佳实践：在纯文本中可通过叙述制造紧迫感，不一定需要真实计时
```

### 7. 叙事选择（Narrative Choice）
选择了解故事的角度/顺序

```
场景：回忆闪回
选项 A：回忆与受害者的最后一次见面
选项 B：回忆第一次遇到嫌疑人
选项 C：回忆案发前的异常事件

特征：
  - 不改变"发生了什么"
  - 改变读者了解信息的顺序
  - 影响读者的情感体验和推理方向

最佳实践：每个叙事角度都揭示不同的情感和信息层面
```

## 二、选择设计原则

### 核心原则

```
1. 知情选择原则
   读者在选择时必须有足够的信息来做出有意义的判断
   ✗ 选项是两扇门但不知道门后是什么
   ✓ 选项有清晰的意图方向，但后果不完全可预期

2. 两难原则
   好的选择没有"正确答案"
   如果一个选项明显优于其他选项 → 不是真正的选择

3. 后果可感知原则
   读者应该能感受到选择的影响
   - 即时反馈：选择后的场景直接不同
   - 延迟反馈：后续角色/对话引用你的选择
   - 累积反馈：多个选择的累积影响在结局显现

4. 角色一致性原则
   所有选项都应该是"读者的角色可能会做的事"
   ✗ "帮助老太太过马路" vs "踢猫"
   ✓ "报警" vs "自己调查"（都是合理反应）

5. 不惩罚好奇心
   探索性选择不应导致不可逆的坏结果
   可以有风险，但不应惩罚读者尝试不同路线
```

## 三、选择文案写作

### 选项文案的 Do / Don't

```
DO:
  ✓ 明确表达意图（"告诉她真相" vs "说实话"）
  ✓ 使用角色的声音（第一人称时）
  ✓ 暗示但不剧透后果
  ✓ 保持选项间长度一致

DON'T:
  ✗ 使用"好"/"坏"这样的价值判断词
  ✗ 透露后果（"选这个会导致 X 死亡"）
  ✗ 一个选项明显比其他选项更详细/吸引人
  ✗ 使用模糊到无法判断的措辞
```

### 选项数量

```
2 个选项：二元对立，最清晰的取舍
  适合：道德困境、是/否决定、立场选择

3 个选项：最常用，平衡了复杂度和可管理性
  适合：策略选择、性格表达
  通常模式：积极 / 消极 / 中立 或 方法 A / B / C

4+ 个选项：慎用，决策疲劳
  适合：信息收集类（选择查看什么）、一次性的大决策

推荐：80% 的决策点使用 2-3 个选项
```

## 四、选择后果设计

### 后果类型

| 类型 | 描述 | 示例 |
|------|------|------|
| 即时变化 | 下一段文本直接不同 | 选择不同的调查方向 → 看到不同的场景 |
| 状态变化 | 修改隐藏的状态变量 | 信任度 +10、线索收集 +1 |
| 对话变体 | 后续对话中的差异 | NPC 记住你的选择并提及 |
| 路线分化 | 后续可用的选项不同 | 某条路线只有之前做了特定选择才能访问 |
| 结局影响 | 影响最终结局的条件 | 累积的信任度决定结局品质 |

### 后果的"冰山"原则

```
读者能看到的后果（冰山上方）：
  - 不同的场景描述
  - 角色的不同反应
  - 不同的对话

读者感受到但看不清的后果（冰山下方）：
  - 状态变量变化
  - 解锁/锁定后续路线
  - 影响结局条件

设计比例：上方 30% / 下方 70%
读者不需要看到所有后果才觉得选择有意义
```

## 五、悬疑特有的选择设计

### 调查选择的设计

```
原则：让读者感觉自己在做真正的推理

  1. 每个调查选择都应提供不同类型的信息
  2. 没有"浪费"的调查——即使选了"错"的方向也能获得角色/世界信息
  3. 关键线索应该可以通过多条路径获得（至少 2 条）
  4. "运气"不应决定成败——好的推理应该由策略决定
```

### 推理选择（指控/结论）

```
高风险选择：指控某人是凶手

设计要点：
  - 只有在读者有足够信息时才提供这个选择
  - 如果选错，应该有戏剧性的后果但不是"游戏结束"
  - 选对时的揭示场景应该令人满意
  - 考虑"部分正确"的情况（猜对了人但原因错了）
```

### 信任/怀疑选择

```
场景：角色告诉你一些信息
选项 A：相信 ta 的说法
选项 B：质疑 ta 的说法
选项 C：假装相信但暗中求证

这类选择影响：
  - 你与该角色的关系
  - 你获取信息的来源和质量
  - 该角色在后续场景中对你的态度
  - 可能影响到该角色是否会帮助你/背叛你
```

## 六、选择平衡测试

### 对每个选择点执行

```
□ 每个选项都有至少 20% 的选择率预期
   （如果某选项预期选择率 < 10%，说明它不够吸引人或太明显是"坏选择"）

□ 每个选项后的内容量大致相当
   （差异不超过 2 倍，否则某些读者体验会明显更短/更长）

□ 选项之间没有明显的"最优解"

□ 每个选项都推进了故事（没有"原地踏步"的选项）

□ 选项之间的差异在读者做选择时就能感知到
   （不是选完才发现"原来这两个一样"）
```
