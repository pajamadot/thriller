# 互动叙事文本写作

> 互动小说的文字不是线性小说的文字——屏幕阅读、选择上下文、重复到达，每一条都改变写作方式
> 参考 Emily Short 的 Craft of Interactive Fiction、Jon Ingold 的 ink 写作指南

---

## 一、互动小说 vs 线性小说的写作差异

| 维度 | 线性小说 | 互动小说 |
|------|----------|----------|
| 人称 | 第一/第三人称均可 | 第二人称最自然 |
| 段落长度 | 可以很长 | 宜短（150-300字/段） |
| 节奏控制 | 作者完全控制 | 读者点击/选择控制 |
| 重复 | 不会读到同一段两次 | 汇聚节点可能被多次到达 |
| 上下文 | 前文必定存在 | 前文取决于路径 |
| 信息密度 | 可以慢慢展开 | 每个节点必须自足 |

---

## 二、人称选择

### 第二人称（推荐默认）

```
"你推开门。空气中弥漫着一股甜腻的气味。"

优势：
  - 最强代入感——读者就是主角
  - 选择感最自然（"你"做了这个选择）
  - 互动小说的行业标准

劣势：
  - 长篇容易产生距离感（读者觉得"我不会这样做"）
  - 角色塑造受限（"你"的性格取决于读者选择）

适用：侦探视角、密室逃脱、生存恐怖
```

### 第一人称

```
"我推开门。空气中那股味道让我皱起了眉头。"

优势：
  - 角色有更强的个性（"我"有自己的想法和声音）
  - 不可靠叙述者更容易实施
  - 文学性更强

劣势：
  - 读者可能觉得"我只是在看别人做选择"
  - 选择与角色性格不一致时会违和

适用：心理悬疑、有强角色设定的故事、叙述性诡计
```

### 第三人称

```
"林探长推开门。那股味道让她停住了脚步。"

优势：
  - 可以展示角色看不到的东西
  - 多视角切换更自然
  - 不会产生"我不会这样做"的抵触

劣势：
  - 代入感最弱
  - 选择感不够直接

适用：上帝视角/导演模式、多角色切换、电影化叙事
```

## 三、节点文本长度指南

### 每节点字数建议

| 节点类型 | 建议字数 | 原因 |
|----------|---------|------|
| 开场节点 | 500-800字 | 建立氛围，读者有耐心 |
| 叙事节点（标准） | 300-600字 | 保持推进感 |
| 决策前文 | 200-400字 | 提供做选择需要的上下文 |
| 高张力节点 | 150-300字 | 短文本加速节奏 |
| 揭示/结局节点 | 600-1200字 | 需要足够空间承载情感 |
| 对话节点 | 根据对话长度 | 对话天然适合屏幕阅读 |

### "滚动疲劳"规则

```
在手机端：
  超过 800 字 → 需要滚动 → 读者注意力下降
  超过 1200 字 → 大量滚动 → 显著流失风险

解决方案：
  1. 拆分长节点为自动推进的连续节点（"继续" 按钮）
  2. 将描述性段落转化为可选的探索（"仔细看看书桌"）
  3. 用选择打断长文（每 300-500 字一个选择点）
```

## 四、汇聚节点的文本技巧

### 问题：多条路径到达同一节点

```
情景：
  路径 A 的读者刚经历了追逐场景，气喘吁吁
  路径 B 的读者刚完成了平静的对话，心态平稳
  两条路径到达同一个节点

如果写"你平复了一下心情"——
  路径 A：合理（刚追逐完）
  路径 B：莫名其妙（刚才很平静啊）
```

### 解决技术

**1. 情绪中性开头**

```
用不假设前一情绪状态的叙述开头：

  ✓ "天已经黑了。你站在咖啡馆门口。"（中性的时空定位）
  ✗ "你终于安全了。"（假设了之前有危险）
  ✗ "事情越来越清晰了。"（假设了之前有进展）
```

**2. 条件性开头句**

```
根据来源路径切换开头 1-2 句：

  {从追逐场景来}: 你的心跳还在加速。推开咖啡馆的门时，空调的冷气让你打了个寒颤。
  {从对话场景来}: 你走进咖啡馆。角落里的位置还空着。
  {通用}: 服务员递来菜单。你没接。

只需要 1-2 句过渡，后面的内容可以完全共享。
```

**3. "当下时刻"锚定**

```
直接从一个新的、共享的"此刻"开始：

  "手机震了一下。来电显示：未知号码。"

  不管之前发生了什么，这个新事件把所有路径拉回同一个"现在"。
```

## 五、选择文案写作

### 选项应该是"意图"而非"行动"

```
✗ 差的选项文案：
  "打开门"
  "不打开门"
  → 只描述了物理动作，没有透露意图/性格

✓ 好的选项文案：
  "推开门——不管里面有什么，你必须知道。"
  "后退一步。你还没准备好面对门后的东西。"
  → 传递了角色的心理状态和动机
```

### 选项长度

```
理想：10-25 字（中文）
最短：5 字（紧急情况下的快速选择）
最长：40 字（复杂决策需要更多信息）

多个选项之间长度应大致一致
最长选项不超过最短选项的 3 倍
```

### 选项排序

```
默认排序原则：
  1. 最符合当前角色性格的选项在前
  2. 如果是道德选择，温和→激进的顺序
  3. 如果是策略选择，安全→冒险的顺序

永远不要：
  ✗ 把"最好"的选项永远放在第一个（读者会学到"选第一个就好"）
  ✗ 用选项顺序暗示"正确"答案
```

## 六、重复到达的处理

### 当读者重新来到同一节点

```
场景：自由探索型拓扑中，读者可能多次访问同一地点

策略 1：首次/非首次变体
  {first time}: 你第一次走进这间书房。灰尘在空气中漂浮。
  {not first time}: 你又回到了书房。

策略 2：渐进揭示
  每次访问揭示一层新内容
  第一次：整体描述
  第二次：注意到上次没注意的细节
  第三次：结合新线索重新解读旧细节

策略 3：状态感知描述
  {found_knife}: 你的目光不由自主地落在抽屉上。你知道里面有什么。
  {not found_knife}: 和上次一样。你总觉得少看了什么。
```

## 七、对话在互动小说中的写法

### 互动对话格式

```
标准互动小说对话（不是电影剧本格式）：

  张警探靠在桌边。"你昨晚在哪里？"

  > "在家。一个人。"（坦白路线）
  > "你凭什么问我？"（对抗路线）
  > 沉默不语。（观察路线）

对话不需要对白标签（"XX说"）——互动格式中说话者很清晰。
但如果多人对话，需要标明说话者。
```

### NPC 对话的"记忆感"

```
让 NPC 对话反映读者之前的选择：

  {之前帮助过她}:
    "谢谢你上次帮我。" 她的眼神比上次温暖多了。
  {之前拒绝了她}:
    她看到你，表情冷了下来。"又有什么事？"
  {第一次见面}:
    "你是谁？" 她防备地看着你。

这种小细节是互动小说代入感的最大来源。
```

## 八、悬疑特有的互动文本技巧

### 线索的交互化呈现

```
不要直接告诉读者"这是一条线索"。
让读者自己发现和判断。

  方法 1：可选检查
    你注意到桌上有一个信封。
    > [打开信封]  → 获取线索
    > [先检查其他地方]  → 不获取这条线索（但后面还有机会）

  方法 2：注意力测试
    把线索嵌入描述性段落中，不特别强调：
    "房间里有一张床、一个衣柜、地上散落着几本杂志。
     墙上挂着一幅风景画，画框有点歪。"
    > [仔细看看那幅画]  → 发现画框后面的保险柜
    > [检查衣柜]  → 走另一条线索链

  方法 3：推理选择
    给读者现有信息，让ta自己推理：
    "目前你知道：A在9点离开，B说自己在家，C的车在停车场。"
    > "A有嫌疑——ta离开的时间和案发时间吻合"
    > "B在说谎——ta的手机定位不在家"
    > "C才是重点——ta说没来过这里但车在这"
```

### 推理揭示的互动节奏

```
在揭示场景中，不要一次性给出所有真相。
让读者通过选择"拼出"真相。

  "你准备好了。所有线索都指向一个方向。"

  > [从时间线开始分析]
    "首先，案发时间不是我们以为的凌晨2点..."
    → 揭示时间线真相
    > [继续]

  > [从动机开始分析]
    "关键是：谁有足够的理由这么做？"
    → 揭示动机真相
    > [继续]

两条路最终到达同一个完整揭示，但顺序不同 → 体验不同。
```
