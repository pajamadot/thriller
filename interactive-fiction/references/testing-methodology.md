# 互动叙事测试方法论

> 确保每条路径都讲得通、每个选择都有意义、每个结局都令人满意
> 参考 ChoiceScript quicktest/randomtest、inkle 内部测试流程、Failbetter QA 协议

---

## 一、三层测试框架

### Layer 1: 结构测试（自动化）

```
可自动验证的项目：

路径完整性：
  □ 所有节点至少有一条路径可达
  □ 所有非结局节点有至少一个后续节点
  □ 不存在无限循环（除非是有意设计的时间循环）
  □ 所有结局都可以通过合理选择到达

状态一致性：
  □ 所有使用的状态变量已被定义
  □ 所有条件检查在至少一条路径上为真
  □ 没有矛盾的状态组合（alive_X = false 但后续节点需要 X 在场）
  □ 数值变量不会超出定义范围

引用完整性：
  □ 所有节点跳转目标存在
  □ 所有条件引用的变量存在
  □ 无孤立节点（不被任何其他节点引用且非起始节点）
```

### Layer 2: 叙事测试（人工阅读）

```
需要人工验证的项目：

路径体验（对每条关键路径）：
  □ 故事从头到尾通顺
  □ 角色行为在此路径下自洽
  □ 信息揭示的顺序产生有效的戏剧效果
  □ 汇聚节点不会引用此路径中不存在的事件

选择体验：
  □ 每个选择点都有做选择的上下文
  □ 没有"盲选"（不知道在选什么）
  □ 选择后的文本回应了选择的意图
  □ 没有"后悔陷阱"（看似合理但必然导致坏结果的选项）

结局体验：
  □ 每个结局在情感上有交代
  □ 结局与此路径的选择历史呼应
  □ 非最佳结局也是"好故事"（不只是 game over）
```

### Layer 3: 平衡测试（数据驱动）

```
需要多人/多次测试后分析：

选择分布：
  理想：每个选项被 20-40% 的测试者选择
  问题：某选项 > 70% → 其他选项不够吸引人
  问题：某选项 < 10% → 可能是"蠢选项"

结局分布：
  理想：最常见结局 30-40%，最不常见 10-15%
  问题：某结局 > 60% → 路径太收敛
  问题：某结局 < 5% → 触发条件太苛刻

路径长度分布：
  理想：最短和最长路径的差异不超过 2 倍
  问题：某条路径明显更短 → 内容不均衡

完成率：
  理想：> 80% 的测试者到达某个结局
  问题：< 60% → 中途流失，可能有无聊/困惑的段落
```

## 二、关键路径测试协议

```
定义 5-8 条关键路径，必须全部测试：

  Path 1: 最短路径（最少选择到达结局）
  Path 2: 最长路径（最多探索到达结局）
  Path 3: "全左"路径（总是选第一个选项）
  Path 4: "全右"路径（总是选最后一个选项）
  Path 5: 最佳结局路径
  Path 6: 最差结局路径
  Path 7-8: 随机选择路径 × 2

对每条路径记录：
  - 总字数 / 预估阅读时间
  - 获得的线索数量
  - 经历的关键剧情事件
  - 到达的结局及其品质
  - 任何不通顺/不一致的地方
```

## 三、平台特定测试

### Twine
```
  □ 在浏览器中可正常运行
  □ 使用"后退"按钮不会破坏状态
  □ 保存/加载功能正常（如果使用 SugarCube）
  □ 长文本段落有适当的滚动
```

### ink
```
  □ inkle 编辑器中无编译错误
  □ 所有 divert 目标存在
  □ 变量逻辑无运行时错误
  □ 在 Inky 测试器中走完所有路径
```

### ChoiceScript
```
  □ quicktest 通过（快速语法检查）
  □ randomtest 通过 10000 次（随机路径无崩溃）
  □ 所有 *goto 目标存在
  □ 不存在无限 *goto 循环
```

## 四、悬疑特有的测试项

```
线索可达性测试：
  对每个结局 E，列出理解 E 所需的关键线索 [I1, I2, ...]
  对每条到达 E 的路径 P，验证 P 上可以获得 [I1, I2, ...]
  如果某路径缺少关键线索 → 增加获取途径

推理公平性测试：
  以"侦探读者"视角走最佳结局路径
  在揭示前暂停：此时的线索是否足以推理出真相？
  如果不足 → 增加线索或调整线索分布

重玩价值测试：
  完成最佳结局后，是否有动力再玩一次？
  第二次游玩是否发现新信息/新视角？
  是否有"只有知道真相后才能理解"的二周目内容？
```
