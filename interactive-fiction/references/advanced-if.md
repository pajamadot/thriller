# 高级互动叙事设计

> 超越基础分支：Quality-Based Narrative、垂直钻探、专业对话设计
> 基于 Emily Short (QBN理论)、Sam Barlow (Her Story/Immortality)、Jon Ingold (inkle)、Failbetter Games 写作指南

---

## 一、Accept / Reject / Deflect 对话设计

### Jon Ingold 的三元选择模型

```
每个对话选择点应提供三种回应姿态：

Accept（接受）：同意/顺从/配合对方
  "你说得对，我们应该去查看现场。"
  → 获取信息，建立信任，推进对话

Reject（拒绝）：反对/质疑/对抗对方
  "你在撒谎。监控显示你当时根本不在那里。"
  → 制造冲突，暴露矛盾，可能关闭信息渠道

Deflect（转移）：回避/转移话题/用幽默化解
  "先不说这个——你注意到窗户上的指纹了吗？"
  → 保持关系中立，控制对话方向，避免正面冲突

为什么是三个：
  - 两个选项 = 二元对立，太简单
  - 四个以上 = 决策疲劳
  - 三个 = 恰好覆盖所有社交姿态
```

### Loops 和 Trapdoors

```
Loop（循环）：对话在当前强度级别循环
  当对话还没有足够的戏剧张力来推进时，
  让玩家在同一话题上继续探索。

  例：审讯嫌疑人时，可以反复追问不同细节，
  直到获得足够信息或关系发生变化。

Trapdoor（出口）：退出循环推进场景的选项
  总是提供一个"离开这个对话"的选择。
  玩家不应被困在对话循环中。

  例："[我已经听够了。去检查另一个房间。]"

节奏规则：
  - 循环内的每次对话必须提供新信息或新反应
  - 循环最多 3-4 轮，之后必须有 trapdoor
  - Trapdoor 的时机决定了对话的节奏和张力
```

## 二、Quality-Based Narrative (QBN)

### Emily Short 的超越分支结构

```
传统分支叙事：节点 → 选择 → 节点 → 选择...（树状）
QBN：大量 Storylet 漂浮在池中，根据"品质值"浮现

Storylet 定义：
  一段内容 + 出现条件（prerequisites）+ 效果（effects）

  例：
  Storylet: "陈雪莹主动找你谈话"
  Prerequisites: trust_chen > 40 AND chapter >= 3 AND NOT talked_chen_today
  Effects: know_chen_secret = true, trust_chen += 10

优势（对比传统分支）：
  - 不需要预设所有路径
  - 内容可以自由组合
  - 同一 storylet 可在多个情境下出现
  - 更接近"世界模拟"而非"故事树"

何时使用 QBN vs 传统分支：
  QBN 适合：开放调查、自由探索、社交互动
  传统分支适合：关键剧情节点、揭示场景、结局分化
  最佳实践：QBN 处理日常/调查段落 + 传统分支处理关键时刻
```

### Failbetter 的精简原则（Parsimony）

```
"用尽可能少的品质值。"

品质值（Quality）过多的危害：
  - 难以预测状态组合
  - Bug 率指数增长
  - 叙事一致性无法保证

Failbetter 的硬性限制：
  - 每个委托最多 3-5 个核心品质值
  - 品质值的交互关系必须是线性的（A 影响 B，不是 A 和 B 互相影响）
  - "结构应该是河流，不是树" — 所有读者都在往同一个方向走，只是走的路不同

字数限制（Fallen London 标准）：
  - 场景入口描述：≤ 30 字
  - 选项描述：≤ 20 字
  - 结果描述：≤ 100 字
  - 仅极少数例外

这些限制逼出了精炼的散文。
对于悬疑互动小说：每一个字都必须做功。
```

## 三、垂直钻探叙事

### Sam Barlow 的非线性叙事方法

```
传统互动叙事 = 横向移动（A → B → C，选择不同的路径到达不同的 C）

Barlow 的方法 = 纵向钻探（同一事件从不同深度/角度理解）

Her Story 的设计：
  - 读者通过关键词搜索数据库
  - 每次搜索揭示同一故事的不同碎片
  - 叙事不是"向前推进"而是"向下挖掘"
  - 每个新碎片重新解读已知的碎片

Immortality 的 match-cut 导航：
  - 点击画面中的元素 → 跳转到包含相同元素的另一个场景
  - 叙事通过"视觉联想"而非"情节逻辑"导航
  - 同一个物品/姿态/构图在不同时间线中的出现 = 主题关联

应用于悬疑互动小说：
  不是"选择去哪里调查"（横向）
  而是"选择从什么角度理解这件事"（纵向）

  例：一段证词
  → 第一次阅读：了解事实
  → 选择关注"情感"角度 → 发现说话人在隐藏悲伤
  → 选择关注"矛盾"角度 → 发现时间线不一致
  → 选择关注"缺席"角度 → 发现有一个人从未被提及

  同一段文本，不同的"钻探方向"产生不同的理解
  → 这比选择不同的路径更深刻
```

### 设计垂直钻探的方法

```
步骤 1：创建"信息晶体"
  一个核心事件/证据包含多层信息：
  - 表层：表面事实
  - 情感层：情感含义
  - 逻辑层：因果关系
  - 隐藏层：只有结合其他信息才能解读的含义

步骤 2：设计"钻探路径"
  读者可以从不同方向"进入"同一信息晶体
  每个方向揭示不同的层

步骤 3：设计"共振"
  当读者从两个不同信息晶体中获取的信息
  突然产生关联时 → "啊哈"时刻

  这是悬疑互动小说的最高形式：
  不是作者告诉你真相，
  而是你自己在碎片之间发现了真相。
```

## 四、Storylet 与悬疑的结合

### 调查阶段的 Storylet 池设计

```
将调查阶段设计为 Storylet 池而非固定路径：

池中 Storylet 类型：
  [线索发现] — 在特定条件下可发现新证据
  [角色互动] — 与 NPC 的对话（受信任度影响）
  [环境探索] — 地点检查（受已知信息影响）
  [事件触发] — 时间/条件触发的剧情事件
  [内心独白] — 侦探的推理和反思

出现条件示例：
  "在酒吧遇到证人"
  → requires: know_witness_identity AND NOT talked_witness
     AND current_location == "downtown"
  → effects: clue_alibi = true, suspicion_chen += 10

关键节点仍用传统分支：
  当 clue_count >= threshold → 触发中点反转
  当 trust_ally >= 70 AND found_key_evidence → 解锁揭示路径
```

## 五、互动悬疑的读者心理

### 选择的认知负荷管理

```
调查阶段：
  低认知负荷选择（信息收集）
  "先检查书桌还是衣柜？" — 策略性但不焦虑

关键时刻：
  高认知负荷选择（道德/信任/风险）
  "告发朋友还是帮他隐瞒？" — 需要深思

节奏规则：
  - 高负荷选择之间至少间隔 2-3 个低负荷选择
  - 连续两个高负荷选择 = 读者决策疲劳
  - 高负荷选择前需要"呼吸空间"（纯叙事段落）
```

### 损失厌恶在互动悬疑中的利用

```
读者更害怕失去已有的东西，而非获取新东西。

设计策略：
  - 让读者先拥有（信任、盟友、安全），再威胁失去
  - "如果你去调查仓库，陈某可能不再信任你"
    比 "去调查仓库可能获得新线索" 更有张力
  - 选择的框架应强调"代价"而非"收益"

BUT：不要每次都是损失——
  偶尔的纯获取选择（发现宝贵线索、赢得新盟友）
  提供必要的正反馈，防止读者疲惫
```

## 六、自然发现行为设计

### Barlow 的"预判玩家行为"原则

```
Her Story 的设计秘密：
  Barlow 预判了大多数玩家会搜索的关键词，
  精心安排了每个关键词会返回哪些视频片段。

  结果：玩家的"自然搜索行为"恰好产生戏剧化的信息揭示顺序。

应用于互动悬疑：
  不要假设读者会按你预期的顺序探索。
  而是设计内容，使得——
  无论读者选择什么顺序，信息的累积都能产生有效的叙事。

  方法：
  1. 列出 5-10 种最可能的探索顺序
  2. 对每种顺序，检查信息揭示是否有戏剧效果
  3. 如果某种顺序"太早"揭示关键信息 → 增加前置条件
  4. 如果某种顺序"错过"必要信息 → 增加冗余获取途径
```
