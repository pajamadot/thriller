# Choice Tradeoff Checklist

Use this checklist when an interactive thriller should feel playable rather than merely branch-shaped.

## Choice Contract

- Each option has a distinct `thriller.intent`.
- Each option pays a visible price in trust, time, safety, certainty, control, or moral position.
- Each option names both an `immediateOutcome` and a `delayedRisk`.

## Visibility

- Every meaningful choice points to `visibleWithinNodes` that exist in the current package.
- At least one visible consequence appears within one to three subsequent nodes.
- A choice set with multiple options does not route every option to the same emotional effect.

## Fairness

- A strong option is not simply free information plus low cost.
- A risky option does not secretly yield the same payoff as a patient option without new downside.
- If an option narrows the suspect field, that narrowing is visible as a cost as well as a benefit.

## Replay Value

- Replaying the node makes different strategies feel different before the route ends.
- At least one option preserves ambiguity while another trades ambiguity for pressure.
- The next band can name which tradeoff is still under-modeled instead of repeating the same fix.
