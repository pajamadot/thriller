# 互动悬疑小说创作系统 — Interactive Thriller Fiction

> 将悬疑小说转化为互动叙事体验——支持静态分支和 LLM 动态运行时生成两种模式
> 融合 Choice Architecture、Narrative Design、Dynamic Narrative Generation（Dramamancer/Drama Llama/StoryVerse）

---

## 系统定位

本技能支持**两种互动模式**：

### Mode A: 静态分支（Static Branching）
预先编写所有节点和选择。适用于：
- 互动小说平台（Twine / ink / ChoiceScript / 橙光）
- 互动影视脚本（Netflix 交互式）
- EPUB 互动增强版

### Mode B: 动态叙事（Dynamic Narrative）
作者定义 Story Schema + Drama Manager 约束，LLM 在运行时根据玩家自由输入实时生成。适用于：
- AI 驱动的文字冒险游戏
- TTRPG AI Game Master
- 对话式互动小说
- 实时互动直播/体验

### Mode C: 混合模式（Hybrid，推荐）
关键剧情节点 = 静态（精确控制），节点之间 = 动态（自由探索）。

---

## 核心理念

### 三大设计支柱

1. **有意义的选择**：每个分支选择必须让读者感受到"我的决定改变了什么"
2. **叙事完整性**：无论走哪条路线，故事都必须在逻辑和情感上完整
3. **可控复杂度**：分支数量在创作可行性范围内，避免指数爆炸

### 悬疑类型的特殊优势

悬疑是最适合互动化的文学类型：
- **调查天然是选择**：先查哪条线索？先审谁？
- **信息即权力**：不同路线获取不同信息，影响推理
- **多结局有天然锚点**：真凶已定，但读者能否找到 ta？
- **失败有叙事价值**：推理错误不是"game over"而是"另一个故事"

### `/branches` — 叙事拓扑设计

**加载参考**：`references/branch-architecture.md`

**输出结构**（生成 `branch-map.md`）：

```
一、分支拓扑选择

   可选拓扑模型：

   A. 瓶颈型（Bottleneck）
      分支 → 汇聚 → 分支 → 汇聚 → 结局分化
      适合：中等复杂度，创作成本可控

   B. 平行调查型（Parallel Investigation）
      共同起点 → 多条调查线并行 → 信息汇总节点 → 结局
      适合：侦探类，不同线索链通向不同真相片段

   C. 时间循环型（Time Loop）
      同一事件反复经历 → 每次携带前次信息 → 最终完美通关
      适合：密室、"土拨鼠之日"式悬疑

   D. 信任网络型（Trust Network）
      选择信任/怀疑不同角色 → 获取不同信息 → 影响同盟和结局
      适合：社会派推理，角色关系驱动

   E. 自由探索型（Open Investigation）
      场景/线索节点自由访问 → 收集足够信息后触发结局 → 结局品质取决于信息量
      适合：经典解谜，高自由度

二、关键节点图

   [Mermaid 流程图：展示所有分支节点、汇聚点、结局点]

三、分支统计
   - 总节点数：N
   - 决策点数：N
   - 唯一路径数：N
   - 最短路径：N 个节点
   - 最长路径：N 个节点
   - 总文本量估算：N 字
```

### `/choices` — 选择点设计

**加载参考**：`references/choice-design.md`, `references/advanced-if.md`

**为每个决策点输出**（生成 `choices/decision-{NNN}.md`）：

```
决策点 #{N}：{名称}

一、上下文
   - 此刻的叙事状态
   - 读者已知信息
   - 情感温度

二、选项设计
   选项 A：{文案}
   - 读者选择动机：{为什么有人会选这个}
   - 隐含信息：{这个选择暗示读者的推理倾向}
   - 导向：{下一个节点}
   - 后果预览等级：{明确 / 模糊 / 隐藏}

   选项 B：{文案}
   - （同上结构）

   选项 C（可选）：{文案}
   - （同上结构）

三、选择质量检查
   □ 没有"正确答案"——每个选项都有合理理由
   □ 选项之间有真实的取舍（trade-off）
   □ 选项反映不同的价值观或推理策略
   □ 选项文案不会泄露后续剧情
   □ 不存在"陷阱选项"（看起来合理实则必死）
```

**选择设计原则**：
- **两难优于多选**：2-3 个选项最佳，超过 4 个造成决策瘫痪
- **后果有延迟**：最好的选择在做出时看不出好坏，数个节点后才显现影响
- **情感与逻辑并重**：理性最优解 ≠ 情感最优解时，产生最好的选择体验
- **"可辩护"原则**：每个选项都应该能被合理辩护，没有"蠢选项"

### `/consistency` — 全局审查

**加载参考**：`references/narrative-convergence.md`

**执行逻辑**：

```
一、路径完整性
   - 所有节点是否可达？
   - 是否存在死胡同（无后续节点的非结局节点）？
   - 是否存在孤立节点？

二、状态一致性
   - 所有使用的状态变量是否已定义？
   - 状态检查条件是否可能为真？
   - 是否存在不可能到达的结局？

三、叙事连贯性
   - 从每条可能路径阅读，故事是否通顺？
   - 汇聚节点是否能兼容所有来源路径？
   - 角色是否在不同路径中行为自洽？

四、线索公平性（悬疑特有）
   - 每个结局所需的关键线索，是否在该路径上可获取？
   - 是否存在"读者做了正确选择却无法获得真相"的情况？

五、平衡性
   - 各路线的内容量是否大致均衡？
   - 是否存在"最优路线"使其他路线失去吸引力？
   - 不同结局的品质差异是否会让部分读者感到被惩罚？
```

## 从线性小说到互动小说的转化工作流

当输入源是已完成的线性悬疑小说时：

```
步骤 1：分析原作结构
  - 识别原作中的关键决策时刻（角色做出选择的地方）
  - 标记"如果ta做了不同选择会怎样"的分歧点
  - 识别读者会想"我会怎么做"的时刻

步骤 2：选择互动化策略
  - 策略 A：决策点互动化（在角色决策处给读者选择权）
  - 策略 B：视角互动化（读者选择跟随哪个角色）
  - 策略 C：信息互动化（读者选择调查哪条线索）
  - 策略 D：混合策略

步骤 3：设计分支差异
  - 最小差异：同一结局不同路径
  - 中等差异：次要情节不同，主线不变
  - 最大差异：不同结局，完全分化的第三幕

步骤 4：处理原作内容
  - 保留：可直接复用的叙事段落（通常 30-50%）
  - 改写：需要适配分支上下文的段落（通常 20-30%）
  - 新增：分支独有的新内容（通常 20-30%）
  - 删除：线性叙事中的过渡，互动版中由选择替代（通常 10-20%）

步骤 5：处理揭示场景的多路径适配
  - 原作的单一揭示 → 根据读者信息状态分化为多个揭示变体
  - 完全信息路径：最接近原作的揭示（完美结局）
  - 部分信息路径：简化版揭示 + 遗留谜团（好结局）
  - 缺失信息路径：错误推理 / 无法揭示（差结局）
```
