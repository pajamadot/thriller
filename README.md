# Thriller — 悬疑小说创作与互动化技能包

> 面向 AI 辅助创作的专业悬疑/推理小说剧本创作系统
> 融合经典编剧方法论 + 互动叙事设计，从线性故事到分支体验

---

## 两个技能

### 1. 悬疑剧本创作 (`thriller-writing/`)

系统化的悬疑/推理小说创作流程，基于专业编剧方法论：

| 命令 | 功能 | 输出 |
|------|------|------|
| `/开始` | 项目初始化（类型、篇幅、视角、诡计类型、风格） | `.thriller-state.json` |
| `/诡计设计` | 核心谜团架构、线索布局、公平性验证 | `trick-design.md` |
| `/人物` | 角色系统（侦探、犯人、嫌疑人矩阵、受害者） | `characters.md` |
| `/结构` | 三幕结构适配、章节大纲、悬念管理 | `structure.md` |
| `/场景 {N}` | 逐章创作（正文 + 线索追踪 + 嫌疑指向） | `chapters/ch{N}.md` |
| `/审线索` | 线索一致性审查、Knox十诫检查 | 审查报告 |
| `/自检 {N}` | 六维质量评估（悬念/线索/人物/节奏/逻辑/文学） | 评分报告 |
| `/揭示` | 真相揭示场景专项设计 | 揭示方案 |
| `/导出` | 完整作品导出 | `export/` |

**支持 7 种悬疑子类型**：本格推理 · 社会派 · 心理悬疑 · 惊悚 · 密室 · 叙述诡计 · 犯罪程序

### 2. 互动悬疑小说 (`interactive-fiction/`)

将悬疑小说转化为带分支选择的互动叙事体验：

| 命令 | 功能 | 输出 |
|------|------|------|
| `/互动初始化` | 项目设置（来源、分支深度、结局数、平台） | `.interactive-state.json` |
| `/分支架构` | 叙事拓扑设计（5种模型选择） | `branch-map.md` |
| `/选择设计` | 每个决策点的选项设计与质量检查 | `choices/` |
| `/角色分支` | 角色命运矩阵、跨路径一致性 | `character-branches.md` |
| `/状态系统` | 变量设计、条件逻辑、结局触发矩阵 | `state-system.md` |
| `/节点 {N}` | 逐节点创作（叙事 + 选择 + 状态变更） | `nodes/` |
| `/结局 {N}` | 结局创作（触发条件 + 正文 + 重玩引导） | `endings/` |
| `/一致性检查` | 路径完整性、状态一致性、叙事连贯性审查 | 审查报告 |
| `/导出 {格式}` | 格式化导出（Twine/ink/ChoiceScript/JSON/Mermaid） | `export/` |

**5 种分支拓扑模型**：瓶颈型 · 平行调查型 · 时间循环型 · 信任网络型 · 自由探索型

## 工作流

### 线性创作流

```
/开始 → /诡计设计 → /人物 → /结构 → /场景 1..N → /审线索 → /揭示 → /导出
```

### 互动化流

```
/互动初始化 → /分支架构 → /选择设计 → /状态系统 → /节点 1..N → /结局 → /导出
```

### 线性→互动转化流

```
线性完稿 → /互动初始化(来源:已有小说) → 识别决策点 → /分支架构 → 扩展分支 → /导出
```
